## 一、批次概述

### 1.1 目标

本批次实现图片多模态描述生成功能，使用 Ollama 的视觉模型（qwen2.5-ocr/llava）生成图片描述，包括：

1. **图片场景描述**：自动识别图片内容并生成文字描述
2. **图表类型识别**：区分图表类型（折线图、柱状图、饼图、散点图等）
3. **图表数据提取**：从图表图片中提取关键数据摘要
4. **语义标签生成**：为图片生成相关语义标签

### 1.2 技术方案

```
图片文件 → ImageParser → VisionClient → Ollama 视觉模型 → ImageDescription
                                      ↓
                              - description (场景描述)
                              - chart_type (图表类型)
                              - chart_data_summary (数据摘要)
                              - semantic_tags (语义标签)
```

### 1.3 图表类型支持


| 图表类型 | 英文名           | 说明     |
| ---- | ------------- | ------ |
| 折线图  | line_chart    | 展示数据趋势 |
| 柱状图  | bar_chart     | 展示数据对比 |
| 饼图   | pie_chart     | 展示占比关系 |
| 散点图  | scatter_chart | 展示数据分布 |
| 雷达图  | radar_chart   | 展示多维数据 |
| 热力图  | heatmap       | 展示密度分布 |
| 其他图表 | other_chart   | 其他图表类型 |
| 普通图片 | null          | 非图表图片  |

## 二、详细设计

### 2.1 配置扩展

#### 2.1.1 VisionConfig 配置类

```python
class VisionConfig(BaseModel):
    """视觉模型配置"""
    model_name: str = "qwen2.5-ocr"  # 视觉模型名称
    ollama_host: str = "http://localhost:11434"  # Ollama 服务地址
    timeout: int = 180  # 请求超时时间(秒)
    retry_times: int = 3  # 重试次数
    enabled: bool = True  # 是否启用视觉描述生成
```

#### 2.1.2 YAML 配置示例

```yaml
# 视觉模型配置
vision:
  model_name: qwen2.5-ocr
  ollama_host: http://localhost:11434
  timeout: 180
  retry_times: 3
  enabled: true
```

### 2.2 数据模型

#### 2.2.1 ImageDescription 模型

```python
class ImageDescription(BaseModel):
    """图片描述模型"""
    description: str = ""           # 场景描述
    chart_type: Optional[str] = None  # 图表类型
    chart_data_summary: Optional[str] = None  # 图表数据摘要
    alt_text: Optional[str] = None  # 替代文本
    semantic_tags: List[str] = []   # 语义标签
    confidence: float = 0.9         # 置信度
```

### 2.3 视觉模型客户端

#### 2.3.1 VisionClient 核心方法

```python
class VisionClient:
    """Ollama 视觉模型客户端"""

    def describe(self, image_bytes: bytes) -> ImageDescription:
        """生成图片描述"""

    def describe_chart(self, image_bytes: bytes) -> ImageDescription:
        """专门用于图表图片的描述生成"""

    def health_check(self) -> bool:
        """检查视觉模型服务健康状态"""
```

## 四、使用说明

### 4.1 前置条件

1. **安装 Ollama**：确保 Ollama 服务已安装并运行
2. **拉取视觉模型**：选择以下任一模型并拉取
  ```bash
   # 推荐使用 qwen2.5-ocr（支持中文）
   ollama pull qwen2.5-ocr

   # 或使用 llava
   ollama pull llava
  ```
3. **启动 Ollama 服务**：
  ```bash
   ollama serve
  ```

### 4.2 配置说明

在 `application-local.yml` 中配置：

```yaml
vision:
  model_name: qwen2.5-ocr  # 视觉模型名称
  ollama_host: http://localhost:11434  # Ollama 服务地址
  timeout: 180  # 请求超时（秒）
  retry_times: 3  # 重试次数
  enabled: true  # 是否启用
```

### 4.3 API 调用示例

图片解析会自动调用视觉模型生成描述：

```python
from app.parsers.image_parser import ImageParser

# 初始化解析器
parser = ImageParser()

# 解析图片
elements = parser.parse(
    file_path="/path/to/image.png",
    version_id=1,
    document_id=1
)

# 获取图片描述
if elements:
    element = elements[0]
    description = element.image_description

    print(f"描述: {description.description}")
    print(f"图表类型: {description.chart_type}")
    print(f"数据摘要: {description.chart_data_summary}")
    print(f"语义标签: {description.semantic_tags}")
    print(f"置信度: {description.confidence}")
```

## 六、错误处理

### 6.1 错误场景


| 错误场景         | 处理方式                                              |
| ------------ | ------------------------------------------------- |
| 视觉模型未启用      | 返回默认描述 `{"description": "图片", "confidence": 0.5}` |
| Ollama 服务不可用 | 返回默认描述，记录警告日志                                     |
| 请求超时         | 重试 `retry_times` 次后返回降级描述                         |
| JSON 解析失败    | 使用原始文本作为描述，降低置信度                                  |
| 模型响应异常       | 返回空描述，记录错误日志                                      |


### 6.2 日志记录

所有操作都会记录详细日志：

```json
{
    "event": "图片描述生成成功",
    "model": "qwen2.5-ocr",
    "cost_ms": 2500,
    "chart_type": "柱线图",
    "confidence": 0.95
}
```

## 七、性能考虑

### 7.1 响应时间


| 图片大小          | 预期响应时间 |
| ------------- | ------ |
| < 100KB       | < 3 秒  |
| 100KB - 500KB | < 5 秒  |
| > 500KB       | < 10 秒 |


### 7.2 并发限制

- 建议配置 Ollama 服务的并发连接数
- 可通过调整 `httpx` 的 `max_connections` 控制

### 7.3 缓存策略

- 相同图片的描述可缓存（基于图片哈希）
- 缓存 TTL 可配置，默认 24 小时

## 二、需求概述

### 2.1 背景

随着企业知识资产的不断积累，如何高效地管理和利用这些知识成为一个重要课题。传统的关键词检索难以理解语义，而纯向量检索对专有名词和精确匹配不够友好。本项目旨在构建一个基于 RAG（Retrieval-Augmented Generation）技术的智能知识库系统，实现文档智能解析、混合检索与智能问答。

### 2.2 目标

1. **文档智能解析**：支持多格式文档（Word、PDF、图片、表格）的自动解析、结构提取和 OCR 识别
2. **语义向量化存储**：基于 Qwen3-Embedding 实现文本向量化，存入 Milvus 向量数据库
3. **混合检索能力**：融合向量语义检索与关键词精确检索，通过 RRF 融合和重排序提升结果质量
4. **智能问答服务**：基于检索增强生成，为用户提供准确的答案和引用来源
5. **持续优化闭环**：通过用户反馈不断优化清洗规则和检索策略

### 2.3 范围

**包含：**

- 多格式文档上传与解析
- 文档清洗与语义切片
- 向量与关键词双通道存储
- 混合检索与重排序
- 智能问答生成
- 用户反馈与系统优化

**不包含：**

- 前端界面开发（预留接口）
- 多语言支持
- 文档协作编辑

### 2.4 假设与依赖

### 3.2 功能详细描述

#### F-001 文档上传

**功能描述：**

用户通过 API 上传文档文件，系统自动识别文件类型、计算文件 Hash、检查重复，并创建文档记录。

**业务流程：**

```
用户上传文件 → 计算文件 Hash → 检查重复 → 存储文件 → 创建 Document → 创建 DocumentVersion → 创建 ImportTask
```

**输入：**

- file: 文件数据（二进制）
- business_id: 业务归属 ID（可选）
- overwrite: 是否覆盖同名文档（可选）

**输出：**

- document_id: 文档 ID
- version_id: 版本 ID
- task_id: 导入任务 ID
- is_duplicate: 是否重复文件

**处理规则：**

1. 支持的文件类型：docx、doc、pdf、txt、md、html、xlsx、xls、csv、png、jpg、jpeg
2. 单文件大小限制：默认 100MB
3. 批量上传限制：默认最多 20 个文件
4. 相同 Hash 文件判定为重复

#### F-002 文档解析

**功能描述：**

对上传的文档进行结构化解析，提取标题、段落、表格、图片等元素，支持 OCR 识别。

**业务流程：**

```
文件类型判断 → 选择解析器 → 结构提取/OCR → 版面分析 → 结构还原 → DocumentElement
```

**处理规则：**

1. Word 文档：优先使用原生结构
2. 电子版 PDF：提取文本层和坐标
3. 扫描版 PDF：OCR 识别
4. 图片：OCR 识别 + 多模态描述
5. 表格：保留表头和合并单元格关系

#### F-003 文本清洗

**功能描述：**

对解析后的文本进行清洗，提升质量。

**处理规则：**

1. 乱码修复：识别并修复编码问题
2. 噪声过滤：删除页眉、页脚、水印
3. 敏感脱敏：手机号、身份证、邮箱脱敏
4. 质量评分：0-1 分质量评分

#### F-004 语义切片

**功能描述：**

将清洗后的文档按语义边界切分为适合检索的 Chunk。

**配置参数：**


| 参数             | 建议值 | 说明   |
| -------------- | --- | ---- |
| target_tokens  | 600 | 目标长度 |
| max_tokens     | 900 | 最大长度 |
| min_tokens     | 120 | 最小长度 |
| overlap_tokens | 100 | 重叠长度 |


**处理规则：**

1. 有章节结构：按标题层级优先切分
2. 无章节结构：按语义边界切分
3. 长表格：按行块切分，保留表头
4. 图片：图题+描述合并

#### F-005 向量存储

**功能描述：**

将 Chunk 文本向量化并存入 Milvus。

**配置参数：**


| 参数         | 建议值             | 说明   |
| ---------- | --------------- | ---- |
| model_name | Qwen3-Embedding | 模型名称 |
| dimension  | 1024            | 向量维度 |
| batch_size | 32              | 批量大小 |


**处理规则：**

1. 支持批量向量化
2. 支持 Embedding 缓存
3. 向量归一化处理

#### F-006 关键词索引

**功能描述：**

对 Chunk 内容构建关键词倒排索引。

**处理规则：**

1. 使用 jieba 分词（中文）
2. 支持同义词扩展
3. 记录词频和位置信息

#### F-007 混合检索

**功能描述：**

融合向量检索和关键词检索，返回最优结果。

**配置参数：**


| 参数            | 建议值 | 说明     |
| ------------- | --- | ------ |
| vector_top_k  | 100 | 向量召回数  |
| keyword_top_k | 100 | 关键词召回数 |
| rrf_k         | 60  | RRF 参数 |


**处理规则：**

1. 查询改写：多查询生成、HyDE
2. 向量检索：Milvus ANN TopK
3. 关键词检索：BM25 评分
4. 结果融合：RRF 或加权融合
5. 权限过滤：按业务和文档过滤

#### F-008 重排序

**功能描述：**

使用 Cross-Encoder 对候选结果进行精细排序。

**处理规则：**

1. 构造 query-chunk pairs
2. Cross-Encoder 打分
3. 按分数排序
4. 过滤低分结果

#### F-009 智能问答

**功能描述：**

基于检索结果和 LLM 生成答案，并提供引用来源。

**处理规则：**

1. 上下文组装：按 Token 约束拼接
2. Prompt 构造：包含参考信息和要求
3. LLM 生成：调用通义千问 API
4. 引用绑定：标注信息来源

#### F-010 用户反馈

**功能描述：**

记录用户对问答结果的反馈，用于系统优化。

**处理规则：**

1. 反馈类型：满意/不满意
2. 反馈原因：文字说明
3. 负面反馈分析：检索问题/生成问题

#### F-011 异步处理

**功能描述：**

使用 RabbitMQ 实现任务异步处理。

**队列设计：**


| 队列名称            | 任务类型      | 说明    |
| --------------- | --------- | ----- |
| parse_queue     | parse     | 解析任务  |
| clean_queue     | clean     | 清洗任务  |
| chunk_queue     | chunk     | 切分任务  |
| embedding_queue | embedding | 向量化任务 |
| index_queue     | index     | 索引任务  |


**处理规则：**

1. 消息持久化
2. 失败重试（最大 3 次）
3. 死信队列处理

#### F-012 版本管理

**功能描述：**

管理文档的多个版本，支持历史回溯。

**处理规则：**

1. 相同 Hash 复用版本
2. 不同 Hash 创建新版本
3. 旧版本标记 inactive
4. 检索默认查 active 版本

#### F-013 质量控制

**功能描述：**

监控和记录解析质量指标。

**质量指标：**


| 指标                     | 说明             |
| ---------------------- | -------------- |
| parse_success_rate     | 解析成功率          |
| ocr_confidence_avg     | OCR 平均置信度      |
| low_quality_page_count | 低质页面数量         |
| chunk_count            | 切分数量           |
| avg_chunk_tokens       | 平均 Chunk Token |

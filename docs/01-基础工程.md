## 二、批次概述

### 2.1 目标

本批次目标是建立 RAG 知识库系统的后端基础工程规范，包括：

1. **项目结构规范**：建立清晰的模块化项目结构，符合路由→服务→模型的层级调用
2. **配置管理规范**：实现 yml 配置管理和多环境（local/dev/prod）支持
3. **基础设施**：实现统一响应、统一异常、统一日志、数据库连接、向量库连接、消息队列连接
4. **接口文档**：配置 Swagger/OpenAPI 文档
5. **测试基础**：提供基础测试用例和测试配置

### 2.2 范围

**包含：**

- 项目目录结构搭建
- 配置文件创建（application-local.yml、application-dev.yml、application-prod.yml）
- 数据库连接配置（SQLAlchemy）
- Redis 连接配置
- Milvus 连接配置
- RabbitMQ 连接配置
- 统一响应模块
- 统一异常模块
- 日志中间件
- 追踪 ID 中间件

- 基础测试用例

**不包含：**

- 具体业务模块实现
- 数据库表创建
- API 接口实现

### 3.2 核心模块说明

#### 3.2.1 配置模块 (core/config.py)

配置类需支持：

- 从 yml 文件加载配置
- 支持环境变量覆盖
- 支持多环境切换（local/dev/prod）
- 类型安全（使用 Pydantic BaseModel）

#### 3.2.2 数据库模块 (core/database.py)

需实现：

- SQLAlchemy 引擎创建
- Session 管理
- 自动建表（开发环境）
- 连接池配置

#### 3.2.3 Redis 模块 (core/redis.py)

需实现：

- Redis 连接池管理
- 连接健康检查
- 基本的 GET/SET 操作封装

#### 3.2.4 Milvus 模块 (core/milvus.py)

需实现：

- Milvus 连接管理
- Collection 管理
- 连接健康检查

#### 3.2.5 RabbitMQ 模块 (core/mq.py)

需实现：

- 连接管理
- Channel 管理
- 发布/订阅基础封装

#### 3.2.6 统一响应 (common/response.py)

统一响应格式：

```python
class ResponseCode:
    SUCCESS = 0
    PARAM_ERROR = 1001
    SYSTEM_ERROR = 1000
    BUSINESS_ERROR = 2000
```

响应工具函数：

```python
def success_response(data: Any, message: str = "success") -> dict
def error_response(code: str, message: str, data: Any = None) -> dict
```

#### 3.2.7 统一异常 (common/exception.py)

自定义异常类：

- BusinessException（业务异常）
- ValidationException（参数校验异常）
- DatabaseException（数据库异常）
- ExternalServiceException（外部服务异常）

全局异常处理器：

```python
@app.exception_handler(BusinessException)
async def business_exception_handler(request, exc)
```

#### 3.2.8 日志中间件 (common/logging.py)

日志格式（JSON）：

```json
{
  "time": "2026-05-22T12:00:00.123+08:00",
  "level": "INFO",
  "traceId": "202605221200000001",
  "userId": null,
  "method": "POST",
  "uri": "/api/v1/documents/upload",
  "requestParams": {},
  "responseCode": 200,
  "costMs": 156,
  "clientIp": "127.0.0.1",
  "message": "文件上传成功"
}
```

#### 3.2.9 中间件 (common/middleware.py)

- 追踪 ID 中间件：自动生成或传递 traceId
- 请求日志中间件：记录请求和响应信息
- 跨域中间件：CORS 配置

```python
# tests/test_basic.py
import pytest
from httpx import AsyncClient
from app.main import app

class TestHealth:
    """健康检查测试"""

    @pytest.mark.asyncio
    async def test_health_check(self):
        """测试健康检查接口"""
        async with AsyncClient(app=app, base_url="http://test") as client:
            response = await client.get("/health")
            assert response.status_code == 200
            data = response.json()
            assert data["code"] == 0

class TestConfig:
    """配置测试"""

    def test_config_load(self):
        """测试配置加载"""
        from core.config import get_settings
        settings = get_settings()
        assert settings.app.name == "rag-knowledge-base"
        assert settings.server.port == 8011
```

### Q2: 什么是HyDE（Hypothetical Document Embeddings）？

**题目类型**：技术原理类

**问题描述**：HyDE是一种查询改写技术。什么是HyDE？它的工作原理是什么？

**答案要点：**

**HyDE原理：**

HyDE的核心思想是"让LLM先猜测答案，再用答案去检索"：

```
用户Query → LLM生成假设答案 → 用假设答案生成Embedding → 向量检索
```

**工作流程：**

```python
class HyDE:
    def __init__(self, llm_client):
        self.llm = llm_client
    
    def search(self, query):
        # 1. LLM生成假设答案
        hypothetical_answer = self.llm.generate(
            f"假设你是专家，请简洁回答：{query}"
        )
        
        # 2. 用假设答案生成向量
        answer_vector = self.embedding_model.encode(hypothetical_answer)
        
        # 3. 向量检索
        results = self.vector_db.search(answer_vector)
        
        return results
```

**为什么有效：**

1. **生成式偏置**
   - LLM生成的文本更接近知识库中的正式表达
   - 假设答案和真实文档风格更接近

2. **语义增强**
   - 生成答案时会补充相关背景
   - 扩展了原始查询的语义

**HyDE vs 普通检索：**

| Query | 普通检索 | HyDE |
|-------|----------|------|
| "embedding维度怎么选" | 匹配包含维度的文档 | 生成答案后检索，可能包含更多技术细节 |
| "RAG怎么优化" | 匹配"优化"相关文档 | 生成优化建议后检索，可能更全面 |

**注意事项：**
- 增加LLM调用延迟
- 假设答案质量影响效果
- 不适合事实性查询

### Q3: 什么是后退提示（Step-Back Prompting）？

**题目类型**：技术原理类

**问题描述**：后退提示是一种查询改写技术。它与普通查询改写有什么区别？如何实现？

**答案要点：**

**后退提示的核心思想：**

后退提示（Step-Back）将具体问题抽象为宏观问题：

```
具体问题: "如何优化RAG系统的检索召回率？"
    ↓ 后退
宏观问题: "RAG系统有哪些优化方法？"
```

**适用场景：**

| 场景 | 具体问题 | 后退问题 |
|------|----------|----------|
| 太具体难以召回 | "RAG在医疗QA的具体应用" | "RAG有哪些应用领域" |
| 意图模糊 | "transformer注意力机制" | "注意力机制是什么" |
| 跨领域问题 | "金融风控中的RAG应用" | "RAG有哪些行业应用" |

**实现代码：**

```python
class StepBackRewriter:
    def __init__(self, llm_client):
        self.llm = llm_client
    
    def rewrite(self, query):
        # 判断是否需要后退
        if self.should_step_back(query):
            # 生成后退问题
            step_back_prompt = f"""
从以下问题中抽象出一个更general的问题：
问题：{query}

请输出一个更general、更abstract的问题，
它可以帮助理解原问题的背景和上下文。

格式：后退问题是什么
"""
            step_back = self.llm.generate(step_back_prompt)
            return {
                'original': query,
                'step_back': step_back,
                'use_both': True
            }
        else:
            return {
                'original': query,
                'step_back': None,
                'use_both': False
            }
    
    def should_step_back(self, query):
        # 太具体或太窄的问题需要后退
        specific_keywords = ['具体', '哪个', '第几个', '某公司的']
        return any(kw in query for kw in specific_keywords)
```

**检索策略：**

```python
def step_back_search(query):
    rewriter = StepBackRewriter()
    rewritten = rewriter.rewrite(query)
    
    results = []

        background_results = vector_search(rewritten['step_back'])
        results.extend(background_results)
    
    # 2. 用原问题检索具体答案

    results.extend(specific_results)
    
    # 3. 合并结果

```

### Q4: 如何实现多查询生成（Multi-Query Generation）？

**题目类型**：技术原理类

**问题描述**：多查询生成是查询改写的重要技术。如何实现？从哪些角度扩展查询？

**答案要点：**

**多查询生成原理：**

一个用户问题可以转化为多个不同角度的查询：

```
原问题: "如何部署RAG系统？"
    ↓ 扩展
查询1: "RAG系统部署步骤"
查询2: "RAG系统有哪些部署方式"
查询3: "RAG部署需要哪些配置"
查询4: "RAG系统搭建教程"
查询5: "RAG生产环境部署"
```

**实现方式：**

```python
class MultiQueryGenerator:
    def __init__(self, llm_client):
        self.llm = llm_client
    
    def generate(self, query, num_queries=5):
        prompt = f"""
用户问题：{query}

请从不同角度生成{num_queries}个搜索查询，
这些查询应该：
1. 覆盖问题的不同方面
2. 使用不同的表达方式
3. 包含同义词替换

格式要求：
每个查询占一行，直接输出，不要编号
"""
        response = self.llm.generate(prompt)
        
        # 解析生成的查询
        queries = [q.strip() for q in response.split('\n') if q.strip()]
        return queries[:num_queries]
```

**查询扩展方向：**

| 方向 | 示例 |
|------|------|
| 同义词 | "部署" → "安装" "搭建" |
| 上下位 | "RAG" → "RAG检索" "RAG生成" |
| 场景 | "部署" → "生产部署" "本地部署" |
| 问题类型 | "怎么做" → "步骤" "方法" "教程" |

**检索合并策略：**

```python
def multi_query_search(query, num_queries=5):
    generator = MultiQueryGenerator()
    
    # 1. 生成多个查询

    # 2. 并行检索

    for q in queries:
        results = vector_search(q)
        all_results.extend(results)
    
    # 3. 去重合并
    merged = deduplicate_by_id(all_results)
    
    # 4. 排序返回
    return sorted(merged, key=lambda x: x.score, reverse=True)
```

### Q5: 查询改写如何平衡改写质量和延迟？

**题目类型**：性能优化类

**问题描述**：查询改写需要调用LLM，增加系统延迟。如何平衡改写质量和响应速度？

**答案要点：**

**延迟来源分析：**

| 组件 | 延迟 | 占比 |
|------|------|------|
| LLM调用 | 500-2000ms | 60-80% |
| 向量检索 | 50-100ms | 10-20% |
| 结果处理 | 10-50ms | 5-10% |

**优化策略：**

1. **缓存改写结果**
```python
class CachedQueryRewriter:
    def __init__(self, cache):
        self.cache = cache
    
    def rewrite(self, query):
        cache_key = hashlib.md5(query.encode()).hexdigest()
        
        # 缓存命中
        if self.cache.exists(cache_key):
            return self.cache.get(cache_key)
        
        # 缓存未命中，调用LLM
        rewritten = self.llm.rewrite(query)
        
        # 写入缓存
        self.cache.set(cache_key, rewritten, ttl=86400)
        
        return rewritten
```

2. **异步改写**
```python
async def async_rewrite_search(query):
    # 并行执行改写和普通检索
    rewrite_task = asyncio.create_task(rewrite_query(query))
    search_task = asyncio.create_task(vector_search(query))
    
    # 先用原查询快速返回
    quick_results = await search_task
    
    # 改写完成后再扩展结果
    rewritten = await rewrite_task
    
    if rewritten != query:
        rewritten_results = await vector_search(rewritten)
        results = merge_results(quick_results, rewritten_results)
    else:
        results = quick_results
    
    return results
```

3. **轻量级改写**
```python
# 使用小模型或规则进行轻量改写
def lightweight_rewrite(query):
    # 规则改写
```

**策略选择：**

| 场景 | 策略 | 延迟 | 质量 |
|------|------|------|------|
| 高频查询 | 缓存 | <10ms | 高 |
| 低延迟要求 | 异步改写 | ~100ms | 中 |

### Q6: 如何设计查询改写的质量评估体系？

**题目类型**：技术原理类

**问题描述**：查询改写的效果如何量化评估？有哪些指标？

**答案要点：**

**评估指标体系：**

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| 召回提升率 | (改写后召回-改写前召回)/改写前召回 | 改写带来的召回提升 |
| 答案质量 | 人工评分1-5 | 答案完整性、准确性 |
| 改写一致率 | 人工认为改写合理/总改写数 | 改写质量 |
| 意图保留率 | 改写后意图与原意一致/总改写数 | 意图保持 |

**评估方法：**

```python
class RewriteEvaluator:
    def __init__(self):
        self.ground_truth = {}
    
    def evaluate_single(self, original, rewritten, retrieval_results):
        """
        评估单个改写效果
        """
        # 1. 召回提升
        original_recall = self.calculate_recall(original)
        rewritten_recall = self.calculate_recall(rewritten)
        recall_gain = (rewritten_recall - original_recall) / original_recall
        
        # 2. 结果重叠度
        overlap = len(set(original_results) & set(rewritten_results)) / len(set(original_results) | set(rewritten_results))
        
        # 3. 新增结果质量
        new_results = set(rewritten_results) - set(original_results)
        new_quality = self.evaluate_new_results_quality(new_results)
        
        return {
            'recall_gain': recall_gain,
            'overlap': overlap,
            'new_quality': new_quality,
            'overall_score': recall_gain * 0.4 + new_quality * 0.6
        }
    
    def evaluate_new_results_quality(self, new_results):
        """
        评估新增结果的质量
        """
        if not new_results:
            return 0.0
        
        relevant_count = sum(1 for r in new_results if self.is_relevant(r))
        return relevant_count / len(new_results)
```

### Q7: 如何处理查询改写中的意图偏移问题？

**题目类型**：场景解决类

**问题描述**：查询改写可能导致意图偏移。如何检测和处理意图偏移？

**答案要点：**

**意图偏移的类型：**

| 类型 | 原意 | 改写后 | 原因 |
|------|------|--------|------|
| 范围偏移 | "RAG的部署" | "RAG和Agent" | 扩展过度 |
| 语义偏移 | "如何调参" | "调参的原理" | 侧重点变化 |
| 领域偏移 | "Python的list" | "list接口设计" | 领域理解错误 |

**检测方法：**

```python
class IntentShiftDetector:
    def __init__(self, embedding_model):
        self.model = embedding_model
    
    def detect_shift(self, original, rewritten):
        """
        检测意图偏移
        """
        # 1. 语义相似度检测
        orig_vec = self.model.encode(original)
        rew_vec = self.model.encode(rewritten)
        semantic_sim = cosine_similarity(orig_vec, rew_vec)
        
        # 2. 关键词重叠检测
        orig_keywords = extract_keywords(original)
        rew_keywords = extract_keywords(rewritten)
        keyword_overlap = len(orig_keywords & rew_keywords) / len(orig_keywords)
        
        # 3. 实体一致性检测
        orig_entities = extract_entities(original)
        rew_entities = extract_entities(rewritten)
        entity_preserved = orig_entities == rew_entities
        
        # 综合判断
        shift_score = (
            (1 - semantic_sim) * 0.4 +
            (1 - keyword_overlap) * 0.3 +
            (0 if entity_preserved else 0.3)
        )
        
        return {
            'shift_score': shift_score,
            'has_shift': shift_score > 0.3,
            'semantic_sim': semantic_sim,
            'keyword_overlap': keyword_overlap,
            'entity_preserved': entity_preserved
        }
```

**处理策略：**

```python
def safe_rewrite(query, rewrite_threshold=0.3):
    """
    安全改写：检测到偏移则回退
    """
    rewriter = MultiQueryRewriter()
    rewritten = rewriter.generate(query)
    
    detector = IntentShiftDetector()
    shift_info = detector.detect_shift(query, rewritten)
    
    if shift_info['has_shift']:
        # 意图偏移，使用轻量级改写
        lightweight = rule_based_rewrite(query)
        return {
            'rewritten': lightweight,
            'method': 'rule',
            'shift_detected': True
        }
    else:
        return {
            'rewritten': rewritten,
            'method': 'llm',
            'shift_detected': False
        }
```

### Q8: 查询改写如何与不同检索策略配合？

**题目类型**：技术原理类

**问题描述**：不同的查询改写技术适用于不同的检索场景。如何根据场景选择合适的改写策略？

**答案要点：**

**策略匹配矩阵：**

| 查询类型 | 适用改写 | 说明 |
|----------|----------|------|
| 口语化问题 | 规范化 | 转换为正式表达 |
| 复杂多意图 | 分解+后退 | 分而治之 |
| 概念理解类 | 后退提示 | 需要背景知识 |
| 模糊问题 | HyDE | 猜测式扩展 |
| 简单问题 | 多查询 | 同义词扩展 |

**自适应选择：**

```python
class AdaptiveRewrite:
    def __init__(self):
        self.query_classifier = load_classifier()
    
    def rewrite(self, query):
        # 1. 分类查询类型
        query_type = self.query_classifier.predict(query)
        
        # 2. 根据类型选择策略
        if query_type == 'factual':
            # 事实查询：轻量改写
            return self.lightweight_rewrite(query)
        
        elif query_type == 'complex':
            # 复杂查询：分解+后退
            return self.decompose_and_stepback(query)
        
        elif query_type == 'vague':
            # 模糊查询：HyDE
            return self.hyde_rewrite(query)
        
        elif query_type == 'simple':
            # 简单查询：多查询扩展
            return self.multi_query_rewrite(query)
        
        else:
            return {'queries': [query], 'method': 'original'}
```

```python
def combined_rewrite(query):
```

**问题描述**：用户的查询意图不同，检索策略也应该不同。如何识别用户意图？

**答案要点：**

**意图分类体系：**

| 意图类型 | 特征词 | 检索策略 |
|----------|--------|----------|
| 事实查询 | 是什么、多少、定义 | 精确匹配优先 |
| 方法查询 | 如何、怎么办、步骤 | 过程文档优先 |
| 列表查询 | 有哪些、哪些、列举 | 列表文档优先 |
| 对比查询 | 区别、比较、差异 | 对比文档优先 |
| 原因查询 | 为什么、原因 | 解释文档优先 |
| 闲聊 | 你好、在吗 | 直接回复 |

**实现方法：**

```python
class IntentClassifier:
    def __init__(self):
        self.patterns = {
            'factual': ['是什么', '定义', '多少', '哪个', '什么是'],
            'method': ['如何', '怎么', '怎么办', '步骤', '方法'],
            'list': ['有哪些', '哪些', '列举', '列表'],
            'comparison': ['区别', '比较', '差异', '不同'],
            'reason': ['为什么', '原因', '原理']
        }
    
    def classify(self, query):
        scores = {}
        
        for intent, keywords in self.patterns.items():
            score = sum(1 for kw in keywords if kw in query)
            scores[intent] = score
        
        # 选择得分最高的意图
        if max(scores.values()) == 0:
            return 'unknown'
        
        return max(scores.items(), key=lambda x: x[1])[0]
```

### Q13: 如何处理查询中的否定语义？

**题目类型**：技术原理类

**问题描述**：用户查询可能包含否定语义，如"不包括"、"不是"、"区别于"等。如何正确处理？

**答案要点：**

**否定语义类型：**

| 类型 | 示例 | 检索策略 |
|------|------|----------|
| 排除 | "RAG但不包括Agent" | 排除Agent相关内容 |
| 对比 | "RAG和Agent的区别" | 两者都检索 |
| 否定的确认 | "不是RAG是什么" | 明确意图后检索 |
| 双重否定 | "不得不了解" | 等同于肯定 |

**处理方法：**

```python
class NegationHandler:
    def __init__(self):
        self.negation_words = ['不', '没', '无', '非', '别', '莫']
        self.exclusion_patterns = [
            '不包括', '不含', '除了', '排除',
            '区别于', '不同于', '不是', '不包括'
        ]
    
    def handle(self, query):
        """处理否定语义"""
        # 1. 检测否定模式
        if self.is_exclusion_query(query):
            return self.handle_exclusion(query)
        elif self.is_comparison_query(query):
            return self.handle_comparison(query)
        else:
            return {'type': 'normal', 'query': query}
    
    def is_exclusion_query(self, query):
        """是否为排除查询"""
        return any(pattern in query for pattern in self.exclusion_patterns)
    
    def handle_exclusion(self, query):
        """
        处理排除查询
        例如："RAG但不包括Agent"
        """
        # 解析排除关系
        parts = query.split('不包括')
        main_topic = parts[0].replace('但', '').strip()
        exclude_topic = parts[1].strip() if len(parts) > 1 else None
        
        return {
            'type': 'exclusion',
            'main_query': main_topic,
            'exclude_query': exclude_topic,
            'action': '检索main_query，排除exclude_query'
        }
```

**检索实现：**


### Q16: 如何实现查询的相似度计算？

**题目类型**：技术原理类

**问题描述**：如何判断两个查询的相似度？有哪些计算方法？

**答案要点：**

**相似度计算方法：**

1. **编辑距离**
```python
def edit_distance(s1, s2):
    """编辑距离"""
    if len(s1) < len(s2):
        return edit_distance(s2, s1)
    
    if len(s2) == 0:
        return len(s1)
    
    previous_row = range(len(s2) + 1)
    for i, c1 in enumerate(s1):
        current_row = [i + 1]
        for j, c2 in enumerate(s2):
            insertions = previous_row[j + 1] + 1
            deletions = current_row[j] + 1
            substitutions = previous_row[j] + (c1 != c2)
            current_row.append(min(insertions, deletions, substitutions))
        previous_row = current_row
    
    return previous_row[-1]
```

2. **Jaccard相似度**
```python
def jaccard_similarity(s1, s2):
    """Jaccard相似度"""
    set1 = set(s1)
    set2 = set(s2)
    return len(set1 & set2) / len(set1 | set2)
```

3. **向量相似度**
```python
def vector_similarity(s1, s2, model):
    """向量余弦相似度"""
    vec1 = model.encode(s1)
    vec2 = model.encode(s2)
    return cosine_similarity(vec1, vec2)
```

**综合相似度：**

```python
def comprehensive_similarity(s1, s2, model):
    """综合相似度"""
    # 编辑距离归一化
    edit_sim = 1 - edit_distance(s1, s2) / max(len(s1), len(s2))
    
    # Jaccard
    jaccard_sim = jaccard_similarity(s1, s2)
    
    # 向量相似度
    vector_sim = vector_similarity(s1, s2, model)
    
    # 综合
    return 0.2 * edit_sim + 0.3 * jaccard_sim + 0.5 * vector_sim
```

### Q18: 如何实现查询的纠错功能？

|------|------|------|

**纠错方法：**

```python
class QueryCorrector:
    def __init__(self):
        self.dictionary = load_dictionary()  # 正确词汇词典
        self.similar_words = load_similar_words()  # 相似词表
    
    def correct(self, query):

        tokens = jieba.cut(query)

        has_correction = False

            if token not in self.dictionary:
                # 尝试纠错

                if correction:
                    corrected_tokens.append(correction)
                    has_correction = True
                else:
                    corrected_tokens.append(token)
            else:
                corrected_tokens.append(token)

        return {
            'original': query,
            'corrected': corrected_query,
            'has_correction': has_correction
        }

        # 编辑距离纠错
        candidates = []
        for correct_word in self.dictionary:
            distance = edit_distance(word, correct_word)
            if distance <= 2:  # 允许2个编辑距离
                candidates.append((correct_word, distance))
        
        if candidates:
            # 返回距离最小的
            return min(candidates, key=lambda x: x[1])[0]
        
        return None
```

```python
class QueryUnderstandingPipeline:
    def __init__(self):
        self.tokenizer = ChineseTokenizer()
        self.intent_classifier = IntentClassifier()
        self.entity_extractor = EntityExtractor()
        self.negation_handler = NegationHandler()
        self.domain_classifier = DomainClassifier()

    def process(self, query):

        完整的查询理解Pipeline
        """
        result = {
            'original_query': query,
            'steps': []
        }
        
        # 1. 纠错
        correction = self.corrector.correct(query)
        processed_query = correction['corrected']
        result['steps'].append({
            'step': 'correction',
            'input': query,
            'output': processed_query,
            'has_correction': correction['has_correction']
        })
        
        # 2. 分词
        tokens = list(self.tokenizer.tokenize(processed_query))
        result['steps'].append({
            'step': 'tokenization',
            'output': tokens
        })
        
        # 3. 意图识别
        intent = self.intent_classifier.classify(processed_query)
        result['steps'].append({
            'step': 'intent',
            'output': intent
        })
        
        # 4. 实体抽取
        entities = self.entity_extractor.extract(processed_query)
        result['steps'].append({
            'step': 'entities',
            'output': entities
        })
        
        # 5. 否定处理
        negation = self.negation_handler.handle(processed_query)
        result['steps'].append({
            'step': 'negation',
            'output': negation
        })
        
        # 6. 领域识别
        domain = self.domain_classifier.classify(processed_query)
        result['steps'].append({
            'step': 'domain',
            'output': domain
        })
        
        # 7. 综合理解结果
        result['understanding'] = {
            'processed_query': processed_query,
            'tokens': tokens,
            'intent': intent,
            'entities': entities,
            'negation': negation,
            'domain': domain
        }
        
        return result
```

```python
class OnlineRewriteLearner:
    def __init__(self):
        self.feedback_buffer = []
        self.update_threshold = 100
    
    def record_feedback(self, original, rewritten, retrieval_result, user_action):
        """记录用户反馈"""
        self.feedback_buffer.append({
            'original': original,
            'rewritten': rewritten,
            'result_useful': user_action == 'click' and user_action.duration > 30,
            'timestamp': now()
        })
        
        # 达到阈值则更新
        if len(self.feedback_buffer) >= self.update_threshold:
            self.incremental_update()
    
    def incremental_update(self):
        """增量更新改写策略"""
        # 1. 分析反馈
        good_rewrites = []
        bad_rewrites = []
        
        for fb in self.feedback_buffer:
            if fb['result_useful']:
                good_rewrites.append((fb['original'], fb['rewritten']))
            else:
                bad_rewrites.append((fb['original'], fb['rewritten']))
        
        # 2. 提取好的改写模式
        good_patterns = self.extract_patterns(good_rewrites)
        bad_patterns = self.extract_patterns(bad_rewrites)
        
        # 3. 更新策略
        self.update_strategy(good_patterns, bad_patterns)
        
        # 4. 清空缓冲区
        self.feedback_buffer = []
    
    def update_strategy(self, good_patterns, bad_patterns):
        """更新改写策略"""
        # 强化好的模式
        for pattern in good_patterns:
            self.pattern_weights[pattern] *= 1.1
        
        # 弱化差的模式
        for pattern in bad_patterns:
            self.pattern_weights[pattern] *= 0.9
        
        # 归一化
        self.normalize_weights()
```

```python
class RewriteABTest:
    def __init__(self):
        self.experiments = {}
    
    def create_experiment(self, name, strategy_a, strategy_b):
        """创建改写策略对比实验"""

            'A': metrics_a,
            'B': metrics_b,
```

### Q28: 如何优化查询改写的批量处理？

**题目类型**：性能优化类

**问题描述**：大量查询需要改写时如何优化？

**答案要点：**

**批量优化策略：**

```python
class BatchRewriter:
    def __init__(self):
        self.batch_size = 10
        self.max_concurrency = 5
    
    async def batch_rewrite(self, queries):
        """
        批量改写优化
        """
        # 1. 去重
        unique_queries = list(set(queries))
        
        # 2. 检查缓存
        uncached = []
        cached_results = {}
        
        for query in unique_queries:
            cached = self.cache.get(query)
            if cached:
                cached_results[query] = cached
            else:
                uncached.append(query)
        
        # 3. 批量LLM调用
        if uncached:
            # 分批处理
            batches = [uncached[i:i+self.batch_size] 
                      for i in range(0, len(uncached), self.batch_size)]
            
            # 并发执行批次
            semaphore = asyncio.Semaphore(self.max_concurrency)
            
            async def process_batch(batch):
                async with semaphore:
                    return await self.llm.batch_rewrite(batch)
            
            batch_results = await asyncio.gather(*[
                process_batch(batch) for batch in batches
            ])
            
            # 合并结果
            for batch_result in batch_results:
                for query, rewritten in batch_result.items():
                    cached_results[query] = rewritten
                    self.cache.set(query, rewritten)
        
        # 4. 返回原始顺序
        return [cached_results[q] for q in queries]
```

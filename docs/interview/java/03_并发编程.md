### Q1: 什么是进程和线程？它们有什么区别？

**题目类型**：基础概念类

**问题描述**：进程和线程是什么？它们有什么区别？Java中线程与进程的关系是什么？

**答案要点**：

**基本概念：**

| 概念 | 定义 | 特点 |
|------|------|------|
| 进程(Process) | 程序的一次执行实例 | 独立的内存空间，资源分配基本单位 |
| 线程(Thread) | CPU调度的最小单位 | 共享进程资源，轻量级 |

**核心区别：**

| 维度 | 进程 | 线程 |
|------|------|------|
| 资源拥有 | 独立内存空间 | 共享进程资源 |
| 创建开销 | 大（需分配独立空间） | 小（共享资源） |
| 通信方式 | 管道、消息队列、共享内存 | 直接读写共享变量 |
| 切换开销 | 大 | 小 |
| 独立性 | 互不影响 | 一个线程崩溃可能影响其他线程 |

**Java中线程与进程：**

```java
// Java中的进程
public class ProcessDemo {
    public static void main(String[] args) {
        // 获取当前进程
        ProcessHandle current = ProcessHandle.current();
        System.out.println("进程ID: " + current.pid());
    }
}

// Java中的线程
public class ThreadDemo {
    public static void main(String[] args) {
        Thread mainThread = Thread.currentThread();
        System.out.println("主线程: " + mainThread.getName());
        
        // 创建新线程
        Thread t = new Thread(() -> {
            System.out.println("新线程执行");
        });
        t.start();
    }
}
```

**Java线程的特点：**
- Java默认2个线程：main线程和GC线程
- Java不能直接操作硬件，通过操作系统内核线程实现
- HotSpot VM将Java线程直接映射为操作系统线程

### Q2: 创建线程有哪几种方式？各有什么优缺点？

**题目类型**：基础概念类

**问题描述**：Java中创建线程有哪些方式？它们各自的优缺点是什么？

**答案要点**：

**四种创建方式：**

```java
// 方式1：继承Thread类
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("继承Thread方式");
    }
}
new MyThread().start();

// 方式2：实现Runnable接口
class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("实现Runnable方式");
    }
}
new Thread(new MyRunnable()).start();

// 方式3：实现Callable接口 + FutureTask
class MyCallable implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        return 42;
    }
}
FutureTask<Integer> futureTask = new FutureTask<>(new MyCallable());
new Thread(futureTask).start();
Integer result = futureTask.get();  // 获取返回值

// 方式4：线程池创建
ExecutorService executor = Executors.newFixedThreadPool(4);
executor.submit(() -> {
    System.out.println("线程池方式");
});
executor.shutdown();
```

**方式对比：**

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 继承Thread | 编写简单 | Java单继承，扩展性差 | 简单任务 |
| 实现Runnable | 解耦，扩展性好 | 无法获取返回值 | 需执行一次 |
| 实现Callable | 可获取返回值/异常 | 需FutureTask包装 | 需要返回值 |
| 线程池 | 复用线程，控制资源 | 使用稍复杂 | 频繁创建线程 |

**最佳实践：**
```java
// 推荐：使用线程池
ExecutorService executor = Executors.newCachedThreadPool();
// 或自定义线程池
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    corePoolSize, maxPoolSize, 
    keepAliveTime, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(100),
    new ThreadPoolExecutor.AbortPolicy()
);
```

### Q3: 线程的状态有哪些？状态之间是如何转换的？

**题目类型**：基础概念类

**问题描述**：Java线程有哪些状态？状态之间是如何转换的？

**答案要点**：

**六种线程状态：**

```java
public enum Thread.State {
    NEW,           // 新建
    RUNNABLE,      // 可运行
    BLOCKED,       // 阻塞
    WAITING,       // 等待
    TIMED_WAITING, // 计时等待
    TERMINATED     // 终止
}
```

**状态转换图：**

```
         ┌─────────┐
         │   NEW   │  new Thread()
         └────┬────┘
              │ start()
              ↓
    ┌─────────────────┐
    │                 │
    │    RUNNABLE     │◄────────────────┐
    │                 │                 │
    └────────┬────────┘                 │
       │     │     │                    │
       │     │     │                    │
   run() │  sleep() │ wait() │  notify()
   完成  │  超时   │   被唤醒           │
       │     │     │                    │
       ↓     ↓     ↓                    │
 ┌──────────┐  ┌──────────┐  ┌──────────┐
 │TERMINATED│  │TIMED_    │  │ WAITING  │
 │  终止    │  │WAITING   │  │  等待    │
 └──────────┘  └──────────┘  └──────────┘
                           sleep(n)/wait(n)/join(n)
```

**状态详解：**

```java
public class ThreadStateDemo {
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        
        System.out.println(thread.getState());  // NEW
        
        thread.start();
        System.out.println(thread.getState());  // RUNNABLE
        
        Thread.sleep(100);
        System.out.println(thread.getState()); // TIMED_WAITING
        
        thread.join();
        System.out.println(thread.getState()); // TERMINATED
    }
}
```

**阻塞场景：**
- BLOCKED：等待获取对象锁
- WAITING：wait(), join(), LockSupport.park()
- TIMED_WAITING：sleep(), wait(timeout), join(timeout)

### Q4: 什么是线程优先级？它是如何工作的？

**问题描述**：Java线程优先级是什么？如何设置？它真的有效吗？

**线程优先级：**

```java
public class Thread implements Runnable {

    public static final int MIN_PRIORITY = 1;
    public static final int NORM_PRIORITY = 5;
```

```java
// 线程优先级只在多线程竞争CPU时起作用
```

### Q5: 什么是守护线程？它有什么特点？

**题目类型**：基础概念类

**问题描述**：什么是守护线程？Java中的守护线程有什么特点？

**答案要点**：

**守护线程的特点：**

```java
// 设置守护线程
Thread daemonThread = new Thread(() -> {
    while (true) {
        System.out.println("守护线程运行中");
    }
});
daemonThread.setDaemon(true);  // 必须在start()前设置
daemonThread.start();

// 特性：所有用户线程结束后，JVM会自动退出
public class DaemonDemo {
    public static void main(String[] args) {
        Thread daemon = new Thread(() -> {
            int count = 0;
            while (true) {
                System.out.println("守护线程: " + ++count);
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    break;
                }
            }
        });
        
        daemon.setDaemon(true);
        daemon.start();
        
        // 主线程sleep 3秒后结束
        try {
            Thread.sleep(3000);
        } catch (InterruptedException e) {}
        
        System.out.println("主线程结束，程序退出");
    }
}
// 输出3-4次后，守护线程被JVM终止
```

**JVM中的守护线程：**

| 守护线程 | 作用 |
|----------|------|
| GC线程 | 垃圾回收 |
| 编译线程 | JIT编译 |
| 信号分发线程 | 处理信号 |
| 心跳检测线程 | 监控线程 |

**注意事项：**
- setDaemon(true)必须在start()之前调用
- finally块中的代码可能不会执行
- 不适合执行业务逻辑

### Q6: Thread.yield()有什么作用？它和sleep()有什么区别？

**题目类型**：技术对比类

**问题描述**：Thread.yield()方法有什么作用？它和sleep()有什么区别？

**答案要点**：

**yield()方法：**

```java
public static native void yield();

// 作用：提示线程调度器放弃当前CPU执行权
// 让同优先级或更高优先级的线程有机会运行
// 但不保证其他线程一定会获得CPU

Thread t = new Thread(() -> {
    for (int i = 0; i < 5; i++) {
        System.out.println("线程执行: " + i);
        Thread.yield();  // 让步
    }
});
```

**yield vs sleep：**

| 特性 | yield() | sleep() |
|------|---------|---------|
| 作用 | 让出CPU，当前线程可重新竞争 | 暂停一段时间 |
| 状态变化 | RUNNABLE → RUNNABLE | RUNNABLE → TIMED_WAITING |
| 异常 | 不抛出InterruptedException | 抛出InterruptedException |
| 阻塞时间 | 不确定 | 精确指定 |
| 释放锁 | 不释放 | 不释放（sleep不释放synchronized） |

**实际使用场景：**

```java
// yield - 避免CPU占用过高（很少使用）
public void run() {
    while (hasWork()) {
        Work work = getWork();
        if (work == null) {
            Thread.yield();  // 没有工作时让出CPU
        } else {
            process(work);
        }
    }
}

// sleep - 暂停执行
try {
    Thread.sleep(1000);  // 暂停1秒
} catch (InterruptedException e) {
    e.printStackTrace();
}

// 两者都不释放锁
synchronized void demo() {
    doSomething();
    Thread.sleep(1000);  // 锁仍被持有
    // Thread.yield();    // 锁仍被持有
}
```

### Q7: 什么是线程中断？如何优雅地中断线程？

**题目类型**：技术原理类

**问题描述**：线程中断是什么？如何正确地中断一个线程？

**答案要点**：

**中断机制：**

```java
public class Thread {
    private volatile boolean interrupted = false;
    
    public void interrupt() {
        interrupted = true;
    }
    
    public boolean isInterrupted() {
        return interrupted;
    }
    
    public static boolean interrupted() {
        boolean old = currentThread().interrupted;
        currentThread().interrupted = false;  // 重置
        return old;
    }
}
```

**中断的三种方式：**

```java
// 方式1：使用中断标志
Thread t = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        try {
            doWork();
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            // 重新设置中断标志
            Thread.currentThread().interrupt();
            break;
        }
    }
});
t.start();
t.interrupt();

// 方式2：使用volatile变量
volatile boolean running = true;
Thread t2 = new Thread(() -> {
    while (running) {
        doWork();
    }
});
running = false;

// 方式3：使用共享队列
BlockingQueue<Task> queue = new LinkedBlockingQueue<>();
Thread t3 = new Thread(() -> {
    while (true) {
        Task task = queue.poll(1, TimeUnit.SECONDS);
        if (task == null) continue;
        process(task);
    }
});
```

**最佳实践：**

```java
public class GracefulShutdown {
    public static void main(String[] args) throws InterruptedException {
        Thread worker = new Thread(() -> {
            try {
                while (!Thread.currentThread().isInterrupted()) {
                    // 阻塞方法会清除中断标志
                    doWork();
                    Thread.sleep(1000);  // InterruptedException
                }
            } catch (InterruptedException e) {
                // 收到中断信号
                // 清理资源
            } finally {
                // 必要的清理
            }
        });
        
        worker.start();
        Thread.sleep(5000);
        worker.interrupt();
    }
}
```

### Q8: 什么是线程 Join？它的使用场景是什么？

**题目类型**：技术原理类

**问题描述**：Thread.join()方法是什么？它的使用场景是什么？

**答案要点**：

**join()方法原理：**

```java
// join()源码 - 等待该线程终止
public final void join() throws InterruptedException {
    wait();  // 调用wait()方法
}

// 实际使用
Thread t1 = new Thread(() -> {
    System.out.println("线程1开始");
    try { Thread.sleep(1000); } catch (Exception e) {}
    System.out.println("线程1结束");
});

Thread t2 = new Thread(() -> {
    System.out.println("线程2开始");
    t1.join();  // 等待t1结束
    System.out.println("线程2结束");
});

t2.start();
t1.start();
// 输出顺序: 线程2开始 -> 线程1开始 -> 线程1结束 -> 线程2结束
```

**join的使用场景：**

```java
// 场景1：等待多个线程完成
public class JoinDemo1 {
    public static void main(String[] args) throws InterruptedException {
        List<Thread> threads = new ArrayList<>();
        
        for (int i = 0; i < 5; i++) {
            final int num = i;
            Thread t = new Thread(() -> {
                System.out.println("线程" + num + "执行");
            });
            threads.add(t);
            t.start();
        }
        
        // 等待所有线程完成
        for (Thread t : threads) {
            t.join();
        }
        
        System.out.println("所有线程执行完毕");
    }
}

// 场景2：带超时的join
public class JoinDemo2 {
    public static void main(String[] args) {
        Thread t = new Thread(() -> {
            try {
                Thread.sleep(5000);
            } catch (InterruptedException e) {}
        });
        
        t.start();
        
        try {
            // 最多等待10秒
            t.join(10000);
            if (t.isAlive()) {
                System.out.println("线程还未结束");
            }
        } catch (InterruptedException e) {}
    }
}
```

**join vs wait：**
- join是Thread的方法，内部调用wait
- join会自动调用notifyAll
- join在Thread执行完毕后会自动唤醒

### Q9: synchronized关键字有哪些用法？

**题目类型**：基础概念类

**问题描述**：synchronized关键字可以修饰什么？它有哪些用法？

**答案要点**：

**四种用法：**

```java
public class SynchronizedDemo {
    
    // 1. 修饰实例方法 - 锁对象
    public synchronized void method1() {
        // 锁定当前this对象
    }
    
    // 2. 修饰静态方法 - 锁类
    public static synchronized void method2() {
        // 锁定当前类的Class对象
    }
    
    // 3. 修饰代码块 - 指定锁对象
    public void method3() {
        synchronized (this) {  // 锁当前对象
            // 临界区代码
        }
    }
    
    public void method4() {
        synchronized (SynchronizedDemo.class) {  // 锁类
            // 临界区代码
        }
    }
    
    // 4. 修饰代码块 - 锁任意对象
    private final Object lock = new Object();
    
    public void method5() {
        synchronized (lock) {
            // 锁定指定对象
        }
    }
}
```

**锁的范围对比：**

| 用法 | 锁对象 | 粒度 | 适用场景 |
|------|--------|------|----------|
| synchronized方法 | this/Class | 粗 | 整个方法 |
| synchronized块 | 指定对象 | 细 | 方法内部分代码 |
| 同步代码块(this) | this | 中 | 替代同步方法 |
| 同步代码块(对象) | 指定对象 | 细 | 多方法协调 |

**最佳实践：**

```java
// 减少锁的范围，提高并发
public class OptimizedDemo {
    private final Object lock = new Object();
    private int count;
    private List<String> list = new ArrayList<>();
    
    public void addToList(String item) {
        synchronized (lock) {
            list.add(item);
        }
    }
    
    public int getCount() {
        return count;  // 读操作可考虑无锁
    }
    
    public synchronized void increment() {
        count++;
    }
}
```

### Q10: synchronized的底层原理是什么？

**题目类型**：技术原理类

**问题描述**：synchronized的底层原理是什么？它是如何实现加锁的？

**答案要点**：

**对象头结构（JDK1.6+）：**

```
┌─────────────────────────────────────────────┐
│                   Mark Word                 │
│  ┌──┬──┬──┬──┬───────┬────────────────────┤
│  │GC│Lock│Age│偏向锁  │     哈希码         │
│  └──┴──┴──┴──┴───────┴────────────────────┤
└─────────────────────────────────────────────┘

锁状态：
- 无锁状态：对象的hashCode，分代年龄
- 偏向锁：线程ID，Epoch
- 轻量级锁：指向栈中锁记录的指针
- 重量级锁：指向互斥量的指针
```

**锁升级过程：**

```
无锁 → 偏向锁 → 轻量级锁 → 重量级锁
  │       │         │         │
  │    第一次       │         │
  │    访问         │         │
  │                 │         │
  │              多线程        │
  │              竞争          │
  │                           │
  │                        竞争激烈
  │                        或锁等待
```

**monitorenter/monitorexit：**

```java
// synchronized代码块编译后
public void method() {
    monitorenter  // 获取monitor
    try {
        // 临界区代码
    } finally {
        monitorexit   // 释放monitor
    }
}
```

**JDK1.6优化：**

| 优化 | 说明 |
|------|------|
| 偏向锁 | 无竞争时消除同步开销 |
| 轻量级锁 | 避免用户态/内核态切换 |
| 适应性自旋 | 自旋次数动态调整 |
| 锁消除 | JIT分析消除不必要的锁 |

### Q11: synchronized和ReentrantLock有什么区别？

**题目类型**：技术对比类

**问题描述**：synchronized和ReentrantLock有哪些区别？各自的使用场景是什么？

**答案要点**：

**核心区别对比：**

| 特性 | synchronized | ReentrantLock |
|------|--------------|---------------|
| 实现方式 | JVM内置 | JDK API |
| 锁类型 | 内置锁 | 可重入锁 |
| 灵活性 | 低 | 高 |
| 超时获取锁 | 不支持 | 支持tryLock() |
| 公平/非公平 | 非公平 | 可配置 |
| 条件变量 | 无 | 多个Condition |
| 中断响应 | 不可中断 | 可中断 |

**ReentrantLock示例：**

```java
public class ReentrantLockDemo {
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull = lock.newCondition();
    private final Object[] items = new Object[100];
    private int count;
    
    public void put(Object x) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length) {
                notFull.await();  // 等待不满
            }
            items[count++] = x;
            notEmpty.signal();   // 通知不空
        } finally {
            lock.unlock();
        }
    }
    
    public Object take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();  // 等待不空
            }
            Object x = items[--count];
            notFull.signal();      // 通知不满
            return x;
        } finally {
            lock.unlock();
        }
    }
}
```

### Q14: 什么是死锁？如何避免死锁？

| 条件 | 说明 |
|------|------|
| 互斥 | 资源只能被一个线程持有 |
| 占有且等待 | 持有资源同时等待其他资源 |
| 不可抢占 | 已获取的资源不能被强制释放 |

```java
    private static Object lockB = new Object();

        // 线程1：先获取lockA，再获取lockB

                System.out.println("线程1获取了lockA");
                try { Thread.sleep(1000); } catch (Exception e) {}
                synchronized (lockB) {
                    System.out.println("线程1获取了lockB");
                }
            }
        });

}
```

```java
    long start = System.currentTimeMillis();

        if (lockA.tryLock()) {

                if (lockB.tryLock()) {
                    return true;
                }
            } finally {
                lockA.unlock();
            }
        }
        if (System.currentTimeMillis() - start > 1000) {
            return false;
        }
        Thread.sleep(100);
    }
}

        }

    }
```

### Q16: 什么是生产者和消费者模式？如何实现？

**题目类型**：场景解决类

**问题描述**：什么是生产者-消费者模式？有哪些实现方式？

**答案要点**：

**模式概述：**

```
┌────────────┐     ┌────────────┐     ┌────────────┐
│  Producer  │────▶│   Queue    │────▶│  Consumer  │
│  生产者    │     │   缓冲区    │     │  消费者    │
└────────────┘     └────────────┘     └────────────┘
      │                                        │
      │         阻塞/唤醒                      │
      └────────────────────────────────────────┘
```

**实现方式：**

```java
// 方式1：wait/notify + synchronized
public class ProducerConsumer1 {
    private final int MAX_SIZE = 10;
    private Queue<Integer> queue = new LinkedList<>();
    
    public synchronized void produce(int item) throws InterruptedException {
        while (queue.size() >= MAX_SIZE) {
            wait();  // 队列满，等待消费
        }
        queue.offer(item);
        System.out.println("生产: " + item);
        notifyAll();  // 通知消费者
    }
    
    public synchronized int consume() throws InterruptedException {
        while (queue.isEmpty()) {
            wait();  // 队列空，等待生产
        }
        int item = queue.poll();
        System.out.println("消费: " + item);
        notifyAll();  // 通知生产者
        return item;
    }
}

// 方式2：BlockingQueue
public class ProducerConsumer2 {
    private final BlockingQueue<Integer> queue = 
        new LinkedBlockingQueue<>(10);
    
    public void produce(int item) throws InterruptedException {
        queue.put(item);  // 队列满则阻塞
        System.out.println("生产: " + item);
    }
    
    public int consume() throws InterruptedException {
        int item = queue.take();  // 队列空则阻塞
        System.out.println("消费: " + item);
        return item;
    }
}

// 方式3：Lock + Condition
public class ProducerConsumer3 {
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();
    private final int MAX_SIZE = 10;
    private Queue<Integer> queue = new LinkedList<>();
    
    public void produce(int item) throws InterruptedException {
        lock.lock();
        try {
            while (queue.size() >= MAX_SIZE) {
                notFull.await();
            }
            queue.offer(item);
            notEmpty.signal();
        } finally {
            lock.unlock();
        }
    }
    
    public int consume() throws InterruptedException {
        lock.lock();
        try {
            while (queue.isEmpty()) {
                notEmpty.await();
            }
            int item = queue.poll();
            notFull.signal();
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

```java
new ThreadPoolExecutor.CallerRunsPolicy() {

    public void rejectedExecution(Runnable r, 
            ThreadPoolExecutor e) {
        if (!e.isShutdown()) {
            r.run();  // 调用者线程执行
        }
    }
}

// 3. DiscardPolicy - 丢弃
// 静默丢弃任务

// 4. DiscardOldestPolicy - 丢弃最老

new ThreadPoolExecutor.DiscardOldestPolicy() {
    @Override
    public void rejectedExecution(Runnable r, 
            ThreadPoolExecutor e) {
        if (!e.isShutdown()) {
            e.getQueue().poll();  // 丢弃最老的
            e.execute(r);         // 执行新的
        }
    }
}
```

```java
    new LinkedBlockingQueue<>();  // 无界队列，可能内存溢出

LinkedBlockingQueue<Runnable> bounded = 
    new LinkedBlockingQueue<>(100);  // 有界队列，配合饱和策略

SynchronousQueue<Runnable> direct = 
    new SynchronousQueue<>();  // 直接交付，必须有空闲线程

ArrayBlockingQueue<Runnable> array = 
    new ArrayBlockingQueue<>(100);  // 有界，基于数组
```

### Q21: Future和CompletableFuture有什么区别？

**题目类型**：技术对比类

**问题描述**：Future和CompletableFuture有什么区别？CompletableFuture有哪些优势？

**答案要点**：

**Future示例：**

```java
// Future - 异步计算的结果
ExecutorService executor = Executors.newFixedThreadPool(2);

Future<Integer> future = executor.submit(() -> {
    Thread.sleep(2000);
    return 42;
});

// 阻塞获取结果
try {
    Integer result = future.get();  // 等待计算完成
    System.out.println(result);
} catch (Exception e) {
    e.printStackTrace();
}

// Future的局限性
// 1. 不能手动完成
// 2. 不能链式调用
// 3. 不能组合多个Future
// 4. 没有异常处理机制
```

**CompletableFuture优势：**

```java
// CompletableFuture - 强大的异步编程
CompletableFuture<String> cf = new CompletableFuture<>();

// 1. 链式编程
CompletableFuture.supplyAsync(() -> fetchUser())
    .thenApply(user -> user.getName())
    .thenApply(name -> name.toUpperCase())
    .thenAccept(System.out::println);

// 2. 组合多个Future
CompletableFuture<String> f1 = CompletableFuture
    .supplyAsync(() -> "Hello");
CompletableFuture<String> f2 = CompletableFuture
    .supplyAsync(() -> "World");

// 合并结果
CompletableFuture<String> combined = f1.thenCombine(f2, 
    (a, b) -> a + " " + b);

// 3. 异常处理
CompletableFuture.supplyAsync(() -> riskyOperation())
    .exceptionally(ex -> {
        log.error("Error", ex);
        return "fallback";
    })
    .thenAccept(System.out::println);

// 4. 超时控制
CompletableFuture<String> withTimeout = 
    CompletableFuture.supplyAsync(() -> slowOperation())
        .orTimeout(3, TimeUnit.SECONDS)
        .exceptionally(ex -> "timeout");

// 5. 手动完成
CompletableFuture<String> manualFuture = 
    new CompletableFuture<>();
// 其他线程调用
manualFuture.complete("result");
```

**对比总结：**

| 特性 | Future | CompletableFuture |
|------|--------|-------------------|
| 链式调用 | 不支持 | 支持 |
| 组合 | 不支持 | thenCombine |
| 异常处理 | 有限 | exceptionally |
| 超时控制 | get(timeout) | orTimeout |
| 手动完成 | 不支持 | complete |
| 回调通知 | 不支持 | thenAccept |

### Q22: 如何实现一个高效的缓存？

**题目类型**：场景解决类

**问题描述**：如何实现一个高效的并发缓存？考虑线程安全、性能和内存管理。

**答案要点**：

**实现方案：**

```java
// 方案1：ConcurrentHashMap + 过期清理
public class Cache<K, V> {
    private final ConcurrentHashMap<K, V> cache = 
        new ConcurrentHashMap<>();
    private final ConcurrentHashMap<K, Long> timestamps = 
        new ConcurrentHashMap<>();
    private final long expireMillis;
    
    public Cache(long expireSeconds) {
        this.expireMillis = expireSeconds * 1000;
    }
    
    public V get(K key) {
        V value = cache.get(key);
        if (value != null) {
            Long timestamp = timestamps.get(key);
            if (timestamp != null && 
                System.currentTimeMillis() - timestamp > expireMillis) {
                cache.remove(key);
                timestamps.remove(key);
                return null;
            }
        }
        return value;
    }
    
    public void put(K key, V value) {
        cache.put(key, value);
        timestamps.put(key, System.currentTimeMillis());
    }
}

// 方案2：Guava Cache
public class GuavaCacheExample {
    private static final LoadingCache<String, Object> cache = 
        CacheBuilder.newBuilder()
            .maximumSize(10000)           // 最大条目数
            .expireAfterWrite(10, TimeUnit.MINUTES)  // 写入后过期
            .removalListener(notification -> {
                System.out.println("移除: " + notification.getKey());
            })
            .build(new CacheLoader<String, Object>() {
                @Override
                public Object load(String key) {
                    return loadFromDB(key);
                }
            });
}

// 方案3：ConcurrentHashMap + 双检查锁
public class DoubleCheckedCache<K, V> {
    private final ConcurrentHashMap<K, V> cache = 
        new ConcurrentHashMap<>();
    
    public V get(K key, Function<K, V> loader) {
        V value = cache.get(key);
        if (value == null) {
            synchronized (this) {
                value = cache.get(key);
                if (value == null) {
                    value = loader.apply(key);
                    cache.put(key, value);
                }
            }
        }
        return value;
    }
}
```

**性能优化点：**

1. 使用ConcurrentHashMap保证线程安全
2. 分段锁减少竞争
3. 合理设置过期时间
4. 限制缓存大小，防止内存溢出
5. 使用弱引用/软引用

**答案要点**：

**超时处理方案：**

```java
// 1. Future超时
public Object getWithTimeout(Future<Object> future) {
    try {
        return future.get(3, TimeUnit.SECONDS);
    } catch (TimeoutException e) {
        future.cancel(true);  // 中断任务
        return null;
    } catch (Exception e) {
        throw new RuntimeException(e);
    }
}

// 2. CompletableFuture超时
public CompletableFuture<Object> asyncWithTimeout(
        Supplier<Object> supplier) {
    CompletableFuture<Object> future = 
        CompletableFuture.supplyAsync(supplier);
    
    return future.orTimeout(3, TimeUnit.SECONDS)
        .exceptionally(ex -> {
            if (ex instanceof TimeoutException) {
                return "timeout result";
            }
            throw new RuntimeException(ex);
        });
}

// 3. 带超时的Lock
public boolean tryLockWithTimeout(ReentrantLock lock, 
        long timeout, TimeUnit unit) {
    try {
        return lock.tryLock(timeout, unit);
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        return false;
    }
}

HttpClient client = HttpClient.newBuilder()

    .build();

Socket socket = new Socket();
```

### Q24: 如何实现一个线程安全的单例模式？

**题目类型**：场景解决类

**问题描述**：在多线程环境下，如何实现一个线程安全的单例模式？

**答案要点**：

**五种线程安全单例实现：**

```java
// 1. 饿汉式（立即加载）
public class Singleton1 {
    private static final Singleton1 INSTANCE = new Singleton1();
    
    private Singleton1() {}
    
    public static Singleton1 getInstance() {
        return INSTANCE;
    }
}

// 2. 懒汉式（同步方法）
public class Singleton2 {
    private static Singleton2 instance;
    
    private Singleton2() {}
    
    public synchronized static Singleton2 getInstance() {
        if (instance == null) {
            instance = new Singleton2();
        }
        return instance;
    }
}

// 3. 双重检查锁定
public class Singleton3 {
    private volatile static Singleton3 instance;
    
    private Singleton3() {}
    
    public static Singleton3 getInstance() {
        if (instance == null) {
            synchronized (Singleton3.class) {
                if (instance == null) {
                    instance = new Singleton3();  // volatile防止指令重排
                }
            }
        }
        return instance;
    }
}

// 4. 静态内部类
public class Singleton4 {
    private Singleton4() {}
    
    private static class Holder {
        private static final Singleton4 INSTANCE = new Singleton4();
    }
    
    public static Singleton4 getInstance() {
        return Holder.INSTANCE;  // 延迟加载
    }
}

// 5. 枚举单例（推荐）
public enum Singleton5 {
    INSTANCE;
    
    public void doSomething() {}
}

// 使用
public class Client {
    public void use() {
        Singleton5.INSTANCE.doSomething();
    }
}
```

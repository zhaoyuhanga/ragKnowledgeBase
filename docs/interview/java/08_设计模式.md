### Q1: 单例模式有哪些实现方式？

**题目类型**：基础概念类

**问题描述**：单例模式有哪些实现方式？各自的优缺点是什么？

**答案要点**：

**单例模式定义：**
保证一个类只有一个实例，并提供一个全局访问点。

**实现方式对比：**

| 方式 | 线程安全 | 延迟加载 | 性能 |
|------|----------|----------|------|
| 饿汉式 | 是 | 否 | 高 |
| 懒汉式 | 否（sync） | 是 | 低 |
| 双重检查 | 是 | 是 | 高 |
| 静态内部类 | 是 | 是 | 高 |
| 枚举 | 是 | 否 | 高 |

**代码实现：**

```java
// 1. 饿汉式（立即加载）
public class Singleton1 {
    private static final Singleton1 INSTANCE = new Singleton1();
    
    private Singleton1() {}
    
    public static Singleton1 getInstance() {
        return INSTANCE;
    }
}

// 2. 懒汉式（延迟加载，同步方法）
public class Singleton2 {
    private static Singleton2 instance;
    
    private Singleton2() {}
    
    public synchronized static Singleton2 getInstance() {
        if (instance == null) {
            instance = new Singleton2();
        }
        return instance;
    }
}

// 3. 双重检查锁定（延迟加载，高性能）
public class Singleton3 {
    // volatile防止指令重排序
    private static volatile Singleton3 instance;
    
    private Singleton3() {}
    
    public static Singleton3 getInstance() {
        if (instance == null) {
            synchronized (Singleton3.class) {
                if (instance == null) {
                    instance = new Singleton3();
                    // 编译优化可能导致：分配内存、赋默认值、赋初始值
                    // volatile保证：分配内存、赋初始值、赋默认值
                }
            }
        }
        return instance;
    }
}

// 4. 静态内部类（延迟加载，线程安全）
public class Singleton4 {
    private Singleton4() {}
    
    private static class Holder {
        private static final Singleton4 INSTANCE = new Singleton4();
    }
    
    public static Singleton4 getInstance() {
        return Holder.INSTANCE;
    }
}

// 5. 枚举单例（最佳实践）
public enum Singleton5 {
    INSTANCE;
    
    public void method() {}
}
```

**最佳实践：**
```java
// 推荐使用枚举或静态内部类
// 枚举天然防止反射和反序列化攻击
public enum DatabaseConnection {
    INSTANCE;
    
    private Connection connection;
    
    DatabaseConnection() {
        connection = DriverManager.getConnection("url");
    }
    
    public Connection getConnection() {
        return connection;
    }
}
```

### Q2: 工厂模式有哪些？它们有什么区别？

**题目类型**：技术对比类

**问题描述**：工厂模式有哪些？简单工厂、工厂方法、抽象工厂有什么区别？

**答案要点**：

**三种工厂模式对比：**

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 简单工厂 | 一个工厂生产所有产品 | 产品种类少 |
| 工厂方法 | 每个产品对应一个工厂 | 产品种类多，扩展性好 |
| 抽象工厂 | 生产一系列相关产品 | 产品族 |

**简单工厂（违反开闭原则）：**

```java
// 产品接口
interface Product {
    void create();
}

// 具体产品
class ProductA implements Product {
    @Override
    public void create() {
        System.out.println("生产产品A");
    }
}

class ProductB implements Product {
    @Override
    public void create() {
        System.out.println("生产产品B");
    }
}

// 简单工厂
class SimpleFactory {
    public Product create(String type) {
        switch (type) {
            case "A": return new ProductA();
            case "B": return new ProductB();
            default: throw new IllegalArgumentException();
        }
    }
}
```

**工厂方法（符合开闭原则）：**

```java
// 工厂接口
interface Factory {
    Product create();
}

// 具体工厂
class FactoryA implements Factory {
    @Override
    public Product create() {
        return new ProductA();
    }
}

class FactoryB implements Factory {
    @Override
    public Product create() {
        return new ProductB();
    }
}

// 使用
Factory factory = new FactoryA();
Product product = factory.create();
```

**抽象工厂（产品族）：**

```java
// 电脑配件族
interface ComputerParts {
    CPU createCPU();
    Memory createMemory();
}

class HighEndComputerParts implements ComputerParts {
    @Override
    public CPU createCPU() {
        return new HighEndCPU();
    }
    
    @Override
    public Memory createMemory() {
        return new HighEndMemory();
    }
}

class LowEndComputerParts implements ComputerParts {
    @Override
    public CPU createCPU() {
        return new LowEndCPU();
    }
    
    @Override
    public Memory createMemory() {
        return new LowEndMemory();
    }
}
```

### Q3: 建造者模式是什么？有什么优势？

**题目类型**：基础概念类

**问题描述**：建造者模式是什么？它相比构造函数有什么优势？

**答案要点**：

**建造者模式定义：**
将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。

**对比传统构造：**

```java
// 传统构造 - 参数过多，不易维护
public class User {
    private String name;
    private int age;
    private String address;
    private String phone;
    private String email;
    
    // 需要大量重载构造函数
    public User(String name, int age) {}
    public User(String name, int age, String address) {}
    // ...
}

// 建造者模式 - 清晰易读
public class User {
    private String name;
    private int age;
    private String address;
    
    private User(Builder builder) {
        this.name = builder.name;
        this.age = builder.age;
        this.address = builder.address;
    }
    
    public static class Builder {
        private String name;
        private int age;
        private String address;
        
        public Builder name(String name) {
            this.name = name;
            return this;
        }
        
        public Builder age(int age) {
            this.age = age;
            return this;
        }
        
        public Builder address(String address) {
            this.address = address;
            return this;
        }
        
        public User build() {
            return new User(this);
        }
    }
}

// 使用
User user = new User.Builder()
    .name("Tom")
    .age(25)
    .address("Beijing")
    .build();
```

**应用场景：**
```java
// 1. StringBuilder
StringBuilder sb = new StringBuilder()
    .append("Hello")
    .append(" ")
    .append("World");

// 2. Lombok @Builder
import lombok.Builder;

@Builder
public class Product {
    private String name;
    private int price;
}

// 3. OkHttp Request
Request request = new Request.Builder()
    .url("https://api.example.com")
    .post(body)
    .build();
```

### Q4: 原型模式是什么？如何实现深拷贝？

**题目类型**：基础概念类

**问题描述**：原型模式是什么？如何实现深拷贝？

**答案要点**：

**原型模式定义：**
通过复制现有对象来创建新对象，无需知道具体类型。

**实现方式：**

```java
// 1. 实现Cloneable接口
public class Prototype implements Cloneable {
    private String name;
    private List<String> items;
    
    @Override
    protected Prototype clone() throws CloneNotSupportedException {
        return (Prototype) super.clone();
    }
}

// 浅拷贝问题
Prototype p1 = new Prototype();
p1.items.add("item");
Prototype p2 = p1.clone();
// p2.items 和 p1.items 指向同一个List！

// 深拷贝实现
public class DeepPrototype implements Cloneable {
    private String name;
    private List<String> items;
    
    @Override
    public DeepPrototype clone() {
        try {
            DeepPrototype copy = (DeepPrototype) super.clone();
            // 手动拷贝引用类型
            copy.items = new ArrayList<>(this.items);
            return copy;
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}

// 序列化深拷贝
public class SerializationCopy implements Serializable {
    public <T extends Serializable> T deepCopy(T obj) {
        try {
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);
            oos.writeObject(obj);
            
            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            return (T) ois.readObject();
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}
```

### Q5: 什么是创建型模式？它们有什么共同点？

**题目类型**：基础概念类

**问题描述**：五种创建型模式（单例、工厂方法、抽象工厂、建造者、原型）有什么共同点？

**答案要点**：

**创建型模式核心目标：**
封装对象创建过程，降低耦合度。

| 模式 | 核心问题 | 解决方案 |
|------|----------|----------|
| 单例 | 只有一个实例 | 私有构造函数 |
| 工厂方法 | 产品对象创建 | 子类决定创建 |
| 抽象工厂 | 产品族创建 | 工厂簇 |
| 建造者 | 复杂对象构建 | 分离构建和表示 |
| 原型 | 复制已有对象 | clone()方法 |

**选择建议：**

```java
// 需要唯一实例 → 单例
// 需要工厂创建 → 工厂方法/抽象工厂
// 对象构造复杂 → 建造者
// 需要复制对象 → 原型
```

### Q7: 代理模式是什么？有哪些类型？

**题目类型**：基础概念类

**问题描述**：代理模式是什么？有哪些类型？各自的实现方式是什么？

**答案要点**：

**代理模式定义：**
为其他对象提供一种代理以控制对这个对象的访问。

**三种代理类型：**

| 类型 | 说明 | 用途 |
|------|------|------|
| 静态代理 | 编译时生成代理类 | 简单场景 |
| 动态代理 | 运行时生成代理类 | 通用场景 |
| CGLIB代理 | 继承方式生成代理 | 无接口类 |

**静态代理：**

```java
// 接口
interface UserService {
    void save(User user);
}

// 目标对象
class UserServiceImpl implements UserService {
    @Override
    public void save(User user) {
        System.out.println("保存用户");
    }
}

// 代理对象
class UserServiceProxy implements UserService {
    private UserService target;
    
    public UserServiceProxy(UserService target) {
        this.target = target;
    }
    
    @Override
    public void save(User user) {
        System.out.println("前置逻辑");
        target.save(user);
        System.out.println("后置逻辑");
    }
}
```

**JDK动态代理：**

```java
class JDKProxyFactory {
    public static <T> T createProxy(T target, Class<?> interfaceClass) {
        return (T) Proxy.newProxyInstance(
            interfaceClass.getClassLoader(),
            new Class[]{interfaceClass},
            (proxy, method, args) -> {
                System.out.println("前置逻辑");
                Object result = method.invoke(target, args);
                System.out.println("后置逻辑");
                return result;
            }
        );
    }
}

// 使用
UserService proxy = JDKProxyFactory.createProxy(
    new UserServiceImpl(), UserService.class);
```

**CGLIB代理：**

```java
class CglibProxyFactory {
    public static <T> T createProxy(T target) {
        return (T) Enhancer.create(
            target.getClass(),
            (MethodInterceptor) (obj, method, args, methodProxy) -> {
                System.out.println("前置逻辑");
                Object result = methodProxy.invokeSuper(obj, args);
                System.out.println("后置逻辑");
                return result;
            }
        );
    }
}
```

### Q8: 装饰器模式和代理模式有什么区别？

**题目类型**：技术对比类

**问题描述**：装饰器模式和代理模式有什么区别？各自的使用场景是什么？

**答案要点**：

**核心区别对比：**

| 特性 | 装饰器模式 | 代理模式 |
|------|-----------|----------|
| 目的 | 动态增加功能 | 控制访问 |
| 关系 | is-a | has-a |
| 继承层级 | 需要相同接口 | 不需要相同接口 |
| 创建方式 | 在构造时包装 | 外部创建 |
| 关注点 | 对象行为增强 | 访问控制 |

**装饰器模式（组合优于继承）：**

```java
// 咖啡组件接口
interface Coffee {
    String getDescription();
    double getCost();
}

// 基础咖啡
class SimpleCoffee implements Coffee {
    @Override
    public String getDescription() {
        return "Simple Coffee";
    }
    
    @Override
    public double getCost() {
        return 2.0;
    }
}

// 装饰器基类
abstract class CoffeeDecorator implements Coffee {
    protected Coffee coffee;
    
    public CoffeeDecorator(Coffee coffee) {
        this.coffee = coffee;
    }
}

// 具体装饰器
class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) {
        super(coffee);
    }
    
    @Override
    public String getDescription() {
        return coffee.getDescription() + ", Milk";
    }
    
    @Override
    public double getCost() {
        return coffee.getCost() + 0.5;
    }
}

// 使用 - 可叠加装饰
Coffee coffee = new MilkDecorator(
    new MilkDecorator(
        new SimpleCoffee()
    )
);
```

**代理模式（控制访问）：**

```java
// 代理模式 - 控制访问
class UserServiceProxy implements UserService {
    private UserService target;
    private boolean hasPermission;
    
    @Override
    public void save(User user) {
        if (!hasPermission) {
            throw new SecurityException("没有权限");
        }
        target.save(user);
    }
}
```

### Q9: 适配器模式是什么？

**题目类型**：基础概念类

**问题描述**：适配器模式是什么？有哪些类型？

**答案要点**：

**适配器模式定义：**
将一个类的接口转换成客户端期望的另一个接口，使原本不兼容的类可以合作。

**两种类型：**

| 类型 | 说明 | 特点 |
|------|------|------|
| 类适配器 | 通过继承实现 | 需要多重继承支持的语言 |
| 对象适配器 | 通过组合实现 | 更灵活，常用 |

**代码实现：**

```java
// 现有接口
interface Target {
    void request();
}

// 现有类（不兼容）
class Adaptee {
    public void specificRequest() {
        System.out.println("Adaptee的方法");
    }
}

// 类适配器（Java不直接支持）
// class ClassAdapter extends Adaptee implements Target {}

// 对象适配器
class ObjectAdapter implements Target {
    private Adaptee adaptee;
    
    public ObjectAdapter(Adaptee adaptee) {
        this.adaptee = adaptee;
    }
    
    @Override
    public void request() {
        adaptee.specificRequest();
    }
}

// 使用
Target target = new ObjectAdapter(new Adaptee());
target.request();
```

**实际应用：**

```java
// Spring AOP中的AdvisorAdapter
// 将Advisor适配为MethodInterceptor

// Spring MVC中的HandlerAdapter
// 适配不同类型的Handler

// Java IO中的InputStreamReader
// 将字节流适配为字符流
Reader reader = new InputStreamReader(new FileInputStream("file.txt"));
```

### Q10: 桥接模式是什么？

**题目类型**：基础概念类

**问题描述**：桥接模式是什么？它解决了什么问题？

**答案要点**：

**桥接模式定义：**
将抽象部分与实现部分分离，使它们可以独立变化。

**问题场景：**

```java
// 问题：多维度变化导致类爆炸
// 手机品牌：Apple, Samsung
// 手机类型：折叠、直板、曲面

// 传统方式：9个子类
class AppleFold {}
class AppleStraight {}
class AppleCurve {}
class SamsungFold {}
class SamsungStraight {}
class SamsungCurve {}

// 桥接模式：分离变化
// 品牌维度
interface Brand {
    void open();
}

class Apple implements Brand {
    @Override
    public void open() {
        System.out.println("Apple开机");
    }
}

// 手机类型维度
class Phone {
    protected Brand brand;  // 桥接
    
    public Phone(Brand brand) {
        this.brand = brand;
    }
}

class FoldPhone extends Phone {
    public FoldPhone(Brand brand) {
        super(brand);
    }
    
    public void open() {
        brand.open();
        System.out.println("折叠手机");
    }
}

// 使用 - 自由组合
Phone phone = new FoldPhone(new Apple());
phone.open();
```

### Q11: 外观模式是什么？

**题目类型**：基础概念类

**问题描述**：外观模式是什么？它有什么作用？

**答案要点**：

**外观模式定义：**
为子系统中的一组接口提供一个统一的入口，简化客户端调用。

**代码实现：**

```java
// 子系统 - 复杂内部实现
class CPU {
    public void freeze() { System.out.println("CPU冻结"); }
    public void jump(long pos) { System.out.println("跳转"); }
    public void execute() { System.out.println("执行"); }
}

class Memory {
    public void load(long pos, byte[] data) {
        System.out.println("加载内存");
    }
}

class HardDrive {
    public byte[] read(long lba, int size) {
        System.out.println("读取硬盘");
        return new byte[size];
    }
}

// 外观类 - 统一入口
class ComputerFacade {
    private CPU cpu;
    private Memory memory;
    private HardDrive drive;
    
    public ComputerFacade() {
        this.cpu = new CPU();
        this.memory = new Memory();
        this.drive = new HardDrive();
    }
    
    public void start() {
        cpu.freeze();
        memory.load(0, drive.read(0, 1024));
        cpu.jump(0);
        cpu.execute();
    }
}

// 客户端使用简化
ComputerFacade computer = new ComputerFacade();
computer.start();
```

**应用场景：**
```java
// Spring Boot对外暴露的统一接口
// MyBatis的SqlSession
// SLF4J日志门面
```

### Q12: 组合模式是什么？

**题目类型**：基础概念类

**问题描述**：组合模式是什么？它用于什么场景？

**答案要点**：

**组合模式定义：**
将对象组合成树形结构以表示"部分-整体"的层次结构。

**代码实现：**

```java
// 组件接口
interface FileSystemComponent {
    String getName();
    long getSize();
    void print(String prefix);
}

// 文件
class File implements FileSystemComponent {
    private String name;
    private long size;
    
    @Override
    public String getName() { return name; }
    
    @Override
    public long getSize() { return size; }
    
    @Override
    public void print(String prefix) {
        System.out.println(prefix + name + " (" + size + " KB)");
    }
}

// 文件夹
class Folder implements FileSystemComponent {
    private String name;
    private List<FileSystemComponent> children = new ArrayList<>();
    
    @Override
    public String getName() { return name; }
    
    @Override
    public long getSize() {
        return children.stream().mapToLong(FileSystemComponent::getSize).sum();
    }
    
    @Override
    public void print(String prefix) {
        System.out.println(prefix + name + "/");
        for (FileSystemComponent child : children) {
            child.print(prefix + "  ");
        }
    }
    
    public void add(FileSystemComponent component) {
        children.add(component);
    }
}

// 使用
Folder root = new Folder("root");
Folder docs = new Folder("docs");
docs.add(new File("readme.md", 10));
root.add(docs);
root.print("");
```

### Q13: 观察者模式是什么？

**题目类型**：基础概念类

**问题描述**：观察者模式是什么？如何实现？

**答案要点**：

**观察者模式定义：**
定义对象间的一种一对多依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知。

**代码实现：**

```java
// 主题接口
interface Subject {
    void attach(Observer observer);
    void detach(Observer observer);
    void notifyAll();
}

// 观察者接口
interface Observer {
    void update(String message);
}

// 具体主题
class NewsAgency implements Subject {
    private List<Observer> observers = new ArrayList<>();
    private String news;
    
    @Override
    public void attach(Observer observer) {
        observers.add(observer);
    }
    
    @Override
    public void detach(Observer observer) {
        observers.remove(observer);
    }
    
    @Override
    public void notifyAll() {
        for (Observer observer : observers) {
            observer.update(news);
        }
    }
    
    public void setNews(String news) {
        this.news = news;
        notifyAll();
    }
}

// 具体观察者
class NewsChannel implements Observer {
    private String name;
    
    public NewsChannel(String name) {
        this.name = name;
    }
    
    @Override
    public void update(String message) {
        System.out.println(name + " 收到新闻: " + message);
    }
}

// 使用
NewsAgency agency = new NewsAgency();
NewsChannel cctv = new NewsChannel("CCTV");
NewsChannel bbc = new NewsChannel("BBC");

agency.attach(cctv);
agency.attach(bbc);
agency.setNews("突发新闻！");
// cctv和bbc都会收到通知
```

**Java内置实现：**

```java
// Observable（已废弃）和 Observer
class NewsAgency extends Observable {
    public void setNews(String news) {
        setChanged();      // 标记已更改
        notifyObservers(news);
    }
}

class NewsChannel implements Observer {
    @Override
    public void update(Observable o, Object arg) {
        System.out.println("收到: " + arg);
    }
}

// Spring事件机制
// ApplicationEventPublisher + @EventListener
```

### Q14: 策略模式是什么？

**题目类型**：基础概念类

**问题描述**：策略模式是什么？它有什么优势？

**答案要点**：

**策略模式定义：**
定义一系列算法，把它们一个个封装起来，并且使它们可以相互替换。

**代码实现：**

```java
// 策略接口
interface PaymentStrategy {
    void pay(double amount);
}

// 具体策略
class AlipayStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("支付宝支付: " + amount);
    }
}

class WechatPayStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("微信支付: " + amount);
    }
}

class CreditCardStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("信用卡支付: " + amount);
    }
}

// 上下文
class PaymentContext {
    private PaymentStrategy strategy;
    
    public PaymentContext(PaymentStrategy strategy) {
        this.strategy = strategy;
    }
    
    public void setStrategy(PaymentStrategy strategy) {
        this.strategy = strategy;
    }
    
    public void pay(double amount) {
        strategy.pay(amount);
    }
}

// 使用
PaymentContext context = new PaymentContext(new AlipayStrategy());
context.pay(100);

context.setStrategy(new WechatPayStrategy());
context.pay(200);
```

**Spring应用：**

```java
// 使用@Bean注入多种策略
@Configuration
public class StrategyConfig {
    @Bean
    public Map<String, PaymentStrategy> strategies() {
        Map<String, PaymentStrategy> map = new HashMap<>();
        map.put("alipay", new AlipayStrategy());
        map.put("wechat", new WechatPayStrategy());
        return map;
    }
}

// 使用
@Service
public class PaymentService {
    @Autowired
    private Map<String, PaymentStrategy> strategies;
    
    public void pay(String type, double amount) {
        PaymentStrategy strategy = strategies.get(type);
        if (strategy != null) {
            strategy.pay(amount);
        }
    }
}
```

### Q16: 责任链模式是什么？

**题目类型**：基础概念类

**问题描述**：责任链模式是什么？有什么应用场景？

**答案要点**：

**责任链模式定义：**
将请求的发送者和接收者解耦，让多个对象有机会处理请求。

**代码实现：**

```java
// 处理器接口
abstract class Handler {
    protected Handler next;
    
    public Handler setNext(Handler next) {
        this.next = next;
        return next;
    }
    
    public final void handle(String request) {
        if (doHandle(request)) {
            return;
        }
        if (next != null) {
            next.handle(request);
        }
    }
    
    protected abstract boolean doHandle(String request);
}

// 具体处理器
class AuthHandler extends Handler {
    @Override
    protected boolean doHandle(String request) {
        if (!request.contains("token")) {
            System.out.println("未认证");
            return true;
        }
        return false; // 继续传递
    }
}

class ValidationHandler extends Handler {
    @Override
    protected boolean doHandle(String request) {
        if (request.contains("invalid")) {
            System.out.println("参数无效");
            return true;
        }
        return false;
    }
}

class BusinessHandler extends Handler {
    @Override
    protected boolean doHandle(String request) {
        System.out.println("处理业务: " + request);
        return true;
    }
}

// 使用
Handler auth = new AuthHandler();
Handler validation = new ValidationHandler();
Handler business = new BusinessHandler();

auth.setNext(validation).setNext(business);

auth.handle("token=abc"); // 完整处理链
auth.handle("invalid");   // 在验证阶段被拦截
```

### Q18: 迭代器模式是什么？

**题目类型**：基础概念类

**问题描述**：迭代器模式是什么？Java中是如何实现的？

**答案要点**：

**迭代器模式定义：**
提供一种方法顺序访问集合元素，而不暴露集合的内部表示。

**Java实现：**

```java
// 内置迭代器接口
public interface Iterator<E> {
    boolean hasNext();
    E next();
    default void remove() {}
    default void forEachRemaining(Consumer<? super E> action) {}
}

// 内置Iterable接口
public interface Iterable<T> {
    Iterator<T> iterator();
    default void forEach(Consumer<? super T> action) {}
    default Spliterator<T> spliterator() {}
}

// 自定义集合
class BookCollection implements Iterable<String> {
    private List<String> books = new ArrayList<>();
    
    public void addBook(String book) {
        books.add(book);
    }
    
    @Override
    public Iterator<String> iterator() {
        return books.iterator();
    }
}

// 使用
BookCollection collection = new BookCollection();
collection.addBook("Java");
collection.addBook("Python");

for (String book : collection) {
    System.out.println(book);
}

// 或使用lambda
```

### Q19: 设计模式的原则有哪些？

**题目类型**：基础概念类

**问题描述**：面向对象设计原则有哪些？SOLID原则是什么？

**答案要点**：

**SOLID原则：**

| 原则 | 说明 | 违反后果 |
|------|------|----------|
| S - 单一职责 | 一个类只负责一件事 | 臃肿、难以维护 |
| O - 开闭原则 | 对扩展开放，对修改关闭 | 频繁修改现有代码 |
| L - 里氏替换 | 子类可以替换父类 | 违反多态 |
| I - 接口隔离 | 接口要小而专 | 强迫实现不需要方法 |
| D - 依赖倒置 | 依赖抽象，不依赖具体 | 紧耦合 |

**其他原则：**

| 原则 | 说明 |
|------|------|
| DRY | 不要重复自己 |
| KISS | 保持简单 |
| YAGNI | 不需要就不实现 |
| LOD | 最小知识原则 |

### Q2: ArrayList和LinkedList的区别是什么？

**题目类型**：技术对比类

**问题描述**：ArrayList和LinkedList都是List接口的实现类，它们有什么区别？各自的使用场景是什么？

**答案要点**：

**核心区别对比：**

| 特性 | ArrayList | LinkedList |
|------|----------|------------|
| 底层数据结构 | 动态数组 | 双向链表 |
| 随机访问 | O(1) 效率高 | O(n) 效率低 |
| 插入/删除 | O(n) 需要移动元素 | O(1) 只需要修改指针 |
| 内存占用 | 连续内存，空间利用率高 | 每个节点存储前后指针 |
| 线程安全 | 不安全 | 不安全 |
| 适用场景 | 频繁随机访问 | 频繁插入删除 |

**源码分析：**

```java
// ArrayList - 基于数组
public class ArrayList<E> {
    transient Object[] elementData;  // 存储元素的数组
    private int size;  // 元素数量
    
    // 随机访问 - O(1)
    public E get(int index) {
        return (E) elementData[index];
    }
    
    // 指定位置插入 - O(n)
    public void add(int index, E element) {
        System.arraycopy(elementData, index, elementData, index + 1, size - index);
        elementData[index] = element;
        size++;
    }
}

// LinkedList - 基于双向链表
public class LinkedList<E> {
    private static class Node<E> {
        E item;
        Node<E> prev;
        Node<E> next;
    }
    
    transient Node<E> first;
    transient Node<E> last;
    private int size;
    
    // 插入操作 - O(1)
    public void add(int index, E element) {
        linkBefore(element, node(index));
    }
    
    // 获取节点 - O(n)
    Node<E> node(int index) {
        if (index < (size >> 1)) {
            Node<E> x = first;
            for (int i = 0; i < index; i++)
                x = x.next;
            return x;
        } else {
            Node<E> x = last;
            for (int i = size - 1; i > index; i--)
                x = x.prev;
            return x;
        }
    }
}
```

**使用建议：**
- 随机访问多 → ArrayList
- 插入删除多 → LinkedList
- 需要实现栈/队列 → LinkedList（实现了Deque）

### Q3: HashMap的底层实现原理是什么？

**题目类型**：技术原理类

**问题描述**：HashMap的底层数据结构是什么？它是如何解决哈希冲突的？

**答案要点**：

**JDK1.7 vs JDK1.8+ 区别：**

| 特性 | JDK 1.7 | JDK 1.8+ |
|------|---------|----------|
| 底层结构 | 数组 + 链表 | 数组 + 链表 + 红黑树 |
| 树化阈值 | 无 | 链表长度>8时转为红黑树 |
| 链表插入 | 头插法 | 尾插法 |
| 扩容时机 | 链表长度>阈值 | 元素数量>capacity×loadFactor |

**核心原理：**

```java
public class HashMap<K, V> {
    // 数组-链表节点
    static class Node<K, V> {
        final int hash;
        final K key;
        V value;
        Node<K, V> next;
    }
    
    // 数组
    transient Node<K, V>[] table;
    
    // 关键参数
    int threshold;           // 扩容阈值
    final float loadFactor;  // 负载因子，默认0.75
    
    // 计算hash值
    static final int hash(Object key) {
        int h;
        return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
    }
    
    // put操作流程
    public V put(K key, V value) {
        // 1. 计算hash
        int hash = hash(key);
        
        // 2. 确定数组位置
        int i = (n - 1) & hash;
        
        // 3. 遍历链表/红黑树
        for (Node<K, V> e = table[i]; e != null; e = e.next) {
            if (e.hash == hash && 
                (e.key == key || key.equals(e.key))) {
                V oldValue = e.value;
                e.value = value;
                return oldValue;
            }
        }
        
        // 4. 添加新节点
        addCount(1);
        return null;
    }
}
```

**哈希冲突解决方法：**
1. **链地址法**：相同hash的节点组成链表（JDK1.7头插、JDK1.8尾插）
2. **红黑树**：链表过长时转为红黑树（长度>8，数组长度>=64）

**容量和扩容：**
```java
// 默认初始容量 16
static final int DEFAULT_INITIAL_CAPACITY = 1 << 4;

// 容量必须是2的幂次方
// 扩容时容量翻倍
```

### Q5: HashSet的底层实现是什么？

**题目类型**：基础概念类

**问题描述**：HashSet的底层实现是什么？它如何保证元素的唯一性？

**答案要点**：

**HashSet实现原理：**
HashSet底层实际上是HashMap，存储的元素作为Map的key，value统一使用一个PRESENT对象。

```java
public class HashSet<E> {
    // 底层使用HashMap
    private transient HashMap<E, Object> map;
    
    // 统一的value
    private static final Object PRESENT = new Object();
    
    // 构造方法
    public HashSet() {
        map = new HashMap<>();
    }
    
    // 添加元素
    public boolean add(E e) {
        return map.put(e, PRESENT) == null;
    }
    
    // contains方法 - 实际调用HashMap的containsKey
    public boolean contains(Object o) {
        return map.containsKey(o);
    }
    
    // remove方法
    public boolean remove(Object o) {
        return map.remove(o) == PRESENT;
    }
}
```

**保证唯一性：**
1. 使用hashCode()确定存储位置
2. 使用equals()比较元素是否相同
3. 只有hashCode和equals都相等时才认为是重复元素

**LinkedHashSet vs HashSet：**
- LinkedHashSet：维护插入顺序（双向链表）
- HashSet：不保证顺序

### Q6: TreeSet和TreeMap的底层实现是什么？

**题目类型**：技术原理类

**问题描述**：TreeSet和TreeMap的底层数据结构是什么？它们是如何实现排序的？

**答案要点**：

**底层数据结构：红黑树**

```java
// TreeMap 核心结构 - 红黑树节点
public class TreeMap<K, V> {
    private transient Entry<K, V> root;
    private transient int size = 0;
    
    static final class Entry<K, V> {
        K key;
        V value;
        Entry<K, V> left;
        Entry<K, V> right;
        Entry<K, V> parent;
        boolean color = BLACK;
    }
    
    // 排序方式
    private final Comparator<? super K> comparator;
    
    // 插入操作 - O(log n)
    public V put(K key, V value) {
        Entry<K, V> t = root;
        if (t == null) {
            root = new Entry<>(key, value, null);
            size = 1;
            return null;
        }
        
        int cmp;
        Entry<K, V> parent;
        Comparator<? super K> cpr = comparator;
        
        if (cpr != null) {
            // 使用自定义比较器
            do {
                parent = t;
                cmp = cpr.compare(key, t.key);
                if (cmp < 0)
                    t = t.left;
                else if (cmp > 0)
                    t = t.right;
                else
                    return t.setValue(value);
            } while (t != null);
        } else {
            // 使用自然顺序
            Comparable<? super K> k = (Comparable<? super K>) key;
            do {
                parent = t;
                cmp = k.compareTo(t.key);
                if (cmp < 0)
                    t = t.left;
                else if (cmp > 0)
                    t = t.right;
                else
                    return t.setValue(value);
            } while (t != null);
        }
        // 插入并调整红黑树
        return null;
    }
}
```

**排序方式：**
1. **自然排序**：实现Comparable接口
2. **自定义比较器**：构造时传入Comparator

**TreeSet vs TreeMap：**
- TreeSet底层就是TreeMap
- TreeSet的add/remove/contains方法实际调用TreeMap的get/put/containsKey

### Q7: 什么是fail-fast和fail-safe？

**题目类型**：技术原理类

**问题描述**：Java集合中的fail-fast和fail-safe机制是什么？有什么区别？

**答案要点**：

**fail-fast（快速失败）：**

```java
// HashMap的迭代器 - fail-fast
private class HashIterator<T> implements Iterator<T> {
    int expectedModCount = modCount;  // 记录modCount
    
    public T next() {
        checkForComodification();  // 检查modCount是否改变
        // ...
    }
    
    final void checkForComodification() {
        if (modCount != expectedModCount)
            throw new ConcurrentModificationException();
    }
}

// 在结构修改（add/remove/clear）时modCount++
public void clear() {
    modCount++;
    // ...
}
```

**fail-safe（安全失败）：**

```java
// CopyOnWriteArrayList - fail-safe
public Iterator<E> iterator() {
    return new COWIterator<E>(array, 0);
}

// 在创建迭代器时复制一份数组
static class COWIterator<E> {
    private final Object[] snapshot;
    
    public E next() {
        return (E) snapshot[index++];
    }
}
```

**区别对比：**

| 特性 | fail-fast | fail-safe |
|------|------------|-----------|
| 实现方式 | 记录modCount | 复制原集合 |
| 内存开销 | 无额外开销 | 需要复制 |
| 是否允许并发修改 | 否 | 是 |
| 典型实现 | HashMap, ArrayList | CopyOnWriteArrayList, ConcurrentHashMap |
| 异常 | ConcurrentModificationException | 无异常 |

### Q9: HashMap的扩容机制是怎样的？

**题目类型**：技术原理类

**问题描述**：HashMap的扩容机制是什么？扩容时发生了什么？有哪些优化？

**答案要点**：

**扩容触发条件：**
- 元素数量 > capacity × loadFactor（默认16 × 0.75 = 12）

**扩容过程：**

```java
// JDK1.8扩容逻辑
final Node<K, V>[] resize() {
    Node<K, V>[] oldTab = table;
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    int oldThr = threshold;
    
    int newCap, newThr = 0;
    
    // 1. 计算新容量和新阈值
    if (oldCap > 0) {
        // 超过最大容量，不再扩容
        if (oldCap >= MAXIMUM_CAPACITY) {
            threshold = Integer.MAX_VALUE;
            return oldTab;
        }
        // 容量和阈值翻倍
        newCap = oldCap << 1;
        newThr = oldThr << 1;
    } else if (oldThr > 0) {
        newCap = oldThr;
    } else {
        newCap = DEFAULT_INITIAL_CAPACITY;  // 16
        newThr = DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY;  // 12
    }
    
    // 2. 创建新数组
    Node<K, V>[] newTab = (Node<K, V>[]) new Node[newCap];
    table = newTab;
    
    // 3. 迁移数据
    if (oldTab != null) {
        for (int j = 0; j < oldCap; ++j) {
            Node<K, V> e;
            if ((e = oldTab[j]) != null) {
                oldTab[j] = null;
                
                if (e.next == null) {
                    // 单节点，直接移动
                    newTab[e.hash & (newCap - 1)] = e;
                } else if (e instanceof TreeNode) {
                    // 红黑树节点
                    ((TreeNode<K, V>) e).split(this, newTab, j, oldCap);
                } else {
                    // 链表节点 - JDK1.8优化
                    Node<K, V> loHead = null, loTail = null;
                    Node<K, V> hiHead = null, hiTail = null;
                    Node<K, V> next;
                    
                    do {
                        next = e.next;
                        // 利用hash高位判断新位置
                        if ((e.hash & oldCap) == 0) {
                            if (loTail == null)
                                loHead = e;
                            else
                                loTail.next = e;
                            loTail = e;
                        } else {
                            if (hiTail == null)
                                hiHead = e;
                            else
                                hiTail.next = e;
                            hiTail = e;
                        }
                    } while ((e = next) != null);
                    
                    // 原位置
                    if (loTail != null) {
                        loTail.next = null;
                        newTab[j] = loHead;
                    }
                    // 新位置 = 旧位置 + oldCap
                    if (hiTail != null) {
                        hiTail.next = null;
                        newTab[j + oldCap] = hiHead;
                    }
                }
            }
        }
    }
    
    return newTab;
}
```

**JDK1.8优化 - 链表迁移：**
- 不再重新计算hash
- 利用hash & oldCap 判断是否需要移动到新位置
- 分为两队：原位置队、新位置队（原位置+oldCap）

**性能影响：**
1. 扩容是耗时操作
2. 预估容量可以减少扩容
3. 负载因子影响空间和时间权衡

### Q10: HashMap的hash算法是怎样的？为什么要这样设计？

**题目类型**：技术原理类

**问题描述**：HashMap的hash算法是怎样的？为什么要使用这种复杂的hash计算方式？

**答案要点**：

**HashMap的hash计算：**

```java
// JDK1.8 hash算法
static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

// 计算数组位置
int index = (n - 1) & hash;
```

**为什么要扰动：**

```
原始hashCode: 1111 1111 1111 1111 1111 1111 1111 1111 (key1)
             1010 1010 1010 1010 1010 1010 1010 1010 (key2)

与16异或后:   0000 0000 0000 0000 1111 1111 1111 1111 (低16位混合)

效果: key1和key2的hash分布更均匀
     减少碰撞概率
```

**为什么不用key.hashCode()直接作为hash：**

```java
// 问题示例：数组容量为16
int index = (n - 1) & key.hashCode();
// n-1 = 15 = 0000 0000 0000 0000 0000 0000 0000 1111

// 只用了hash的低4位，大量碰撞！
```

**优化后的好处：**
1. 高16位与低16位异或，让高位参与运算
2. 数组容量通常较小，高位信息丢失
3. 扰动后碰撞更少，分布更均匀

### Q11: 什么是HashMap的死循环问题？

**题目类型**：技术原理类

**问题描述**：HashMap在并发环境下可能出现什么问题？为什么JDK1.8解决了死循环问题？

**答案要点**：

**JDK1.7的死循环问题：**

```java
// JDK1.7 扩容时的链表迁移 - 头插法
void transfer(Entry[] newTable, boolean rehash) {
    Entry[] src = table;
    int newCapacity = newTable.length;
    
    for (int j = 0; j < src.length; j++) {
        Entry<K,V> e = src[j];
        if (e != null) {
            src[j] = null;
            do {
                Entry<K,V> next = e.next;  // 记录下一个节点
                
                // 重新计算位置
                int i = indexFor(e.hash, newCapacity);
                
                // 头插法 - 问题所在！
                e.next = newTable[i];
                newTable[i] = e;
                
                e = next;
            } while (e != null);
        }
    }
}
```

**死循环产生的过程：**

```
原链表: A -> B -> C
       (同一个桶)

并发扩容时，线程A和线程B可能产生:
线程A: A.next = null, B.next = A
线程B: B.next = null, A.next = B

结果形成: A -> B -> A (环形链表)
```

**JDK1.8的解决方案：**

```java
// JDK1.8改用尾插法
do {
    next = e.next;
    if ((e.hash & oldCap) == 0) {
        // 保持原位置
        if (loTail == null)
            loHead = e;
        else
            loTail.next = e;
        loTail = e;
    } else {
        // 移动到新位置
        if (hiTail == null)
            hiHead = e;
        else
            hiTail.next = e;
        hiTail = e;
    }
} while ((e = next) != null);
```

**JDK1.8的改进：**
1. 尾插法保证顺序不变
2. 不会产生环形链表
3. 但仍有数据覆盖风险

**线程安全替代：**
- ConcurrentHashMap
- Collections.synchronizedMap()
- Hashtable

### Q12: LinkedHashMap是如何实现LRU缓存的？

**题目类型**：技术原理类

**问题描述**：LinkedHashMap如何实现LRU（最近最少使用）缓存？

**答案要点**：

**LinkedHashMap结构：**

```java
public class LinkedHashMap<K, V> {
    // 双向链表维护顺序
    private static class Entry<K, V> extends HashMap.Node<K, V> {
        Entry<K, V> before, after;
    }
    
    transient LinkedHashMap.Entry<K, V> head;
    transient LinkedHashMap.Entry<K, V> tail;
    
    // accessOrder: true表示访问顺序，false表示插入顺序
    final boolean accessOrder;
    
    // 访问后调用
    void afterNodeAccess(Node<K, V> p) { }
    void afterNodeInsertion(boolean evict) { }

```

**实现LRU缓存：**

```java
public class LRUCache<K, V> extends LinkedHashMap<K, V> {
    private final int capacity;
    
    public LRUCache(int capacity) {
        // accessOrder=true 表示按访问顺序
        super(capacity, 0.75f, true);
        this.capacity = capacity;
    }
    
    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        // 当size大于容量时，删除最老的元素
        return size() > capacity;
    }
    
    public static void main(String[] args) {
        LRUCache<String, Integer> cache = new LRUCache<>(3);
        cache.put("A", 1);
        cache.put("B", 2);
        cache.put("C", 3);
        
        cache.get("A");  // 访问A，刷新为最新
        
        cache.put("D", 4);  // 淘汰B
        
        System.out.println(cache.keySet());  // [A, C, D]
    }
}
```

**LinkedHashMap特性：**
- 继承HashMap，使用哈希表+双向链表
- 默认按插入顺序排列
- accessOrder=true时按访问顺序排列（适合LRU）

### Q14: ConcurrentHashMap是如何保证线程安全的？

**题目类型**：技术原理类

**问题描述**：ConcurrentHashMap是如何实现线程安全的？JDK1.7和JDK1.8有什么区别？

**答案要点**：

**JDK1.7 - Segment分段锁：**

```java
// 原理：数组+Segment数组，每个Segment一把锁
public class ConcurrentHashMap<K, V> {
    final Segment<K, V>[] segments;
    
    static final class Segment<K, V> extends ReentrantLock {
        transient volatile HashEntry<K, V>[] table;
    }
    
    // put操作
    public V put(K key, V value) {
        int hash = hash(key);
        int j = (hash >>> segmentShift) & segmentMask;
        
        // 只锁定对应的Segment
        Segment<K, V> s = segments[j];
        s.lock();
        try {
            // 操作
        } finally {
            s.unlock();
        }
    }
}
```

**JDK1.8 - CAS + synchronized：**

```java
// JDK1.8 改进：取消Segment，使用CAS+synchronized
public class ConcurrentHashMap<K, V> {
    transient volatile Node<K, V>[] table;
    
    // CAS + synchronized
    final V putVal(K key, V value, boolean onlyIfAbsent) {
        int hash = spread(key.hashCode());
        
        for (Node<K, V>[] tab = table; ...) {
            int n, i;
            
            // 1. 初始化数组 - CAS
            if ((tab = table) == null || (n = tab.length) == 0) {
                tab = initTable();
            }
            
            // 2. 当前位置空 - CAS插入
            else if ((f = tabAt(tab, i = (n - 1) & hash)) == null) {
                if (casTabAt(tab, i, null, new Node<K, V>(hash, key, value)))
                    break;
            }
            
            // 3. 发生hash碰撞 - synchronized锁定
            else {
                synchronized (f) {
                    // 链表/红黑树操作
                }
            }
        }
    }
    
    // 使用CAS保证原子性
    static final <K, V> boolean casTabAt(...) {
        return U.compareAndSwapObject(tab, i, c, v);
    }
}
```

**JDK1.8优化点：**
1. 取消分段锁，降低锁粒度
2. 使用CAS处理空槽插入
3. 链表过长时转为红黑树
4. 锁住单个桶而非整个表

### Q15: CopyOnWriteArrayList的原理是什么？

**题目类型**：技术原理类

**问题描述**：CopyOnWriteArrayList是如何实现线程安全的？它的优缺点是什么？

**答案要点**：

**Copy-On-Write模式：**

```java
public class CopyOnWriteArrayList<E> {
    // volatile数组，保证可见性
    private transient volatile Object[] array;
    
    final Object[] getArray() {
        return array;
    }
    
    // 读操作无需加锁
    public E get(int index) {
        return (E) getArray()[index];
    }
    
    // 写操作先复制
    public boolean add(E e) {
        synchronized (lock) {
            Object[] elements = getArray();
            int len = elements.length;
            
            // 复制新数组
            Object[] newElements = Arrays.copyOf(elements, len + 1);
            newElements[len] = e;
            
            // 原子替换
            array = newElements;
            return true;
        }
    }
    
    // 迭代器是快照，不受影响
    public Iterator<E> iterator() {
        return new COWIterator<E>(getArray(), 0);
    }
}
```

**特点分析：**

| 特性 | 说明 |
|------|------|
| 读操作 | 无锁，高并发读 |
| 写操作 | 加锁，复制整个数组 |
| 迭代器 | 快照遍历，不支持修改 |
| 内存开销 | 写操作需要复制数组 |
| 数据一致性 | 最终一致，不是强一致 |

**使用场景：**
- 读多写少的场景
- 不需要实时一致性的场景
- 如：配置信息、缓存等

**缺点：**
1. 内存占用大（每次写都复制）
2. 不适合大数据量
3. 迭代器只能遍历，不能修改

### Q16: Queue和Deque的区别是什么？

**题目类型**：技术对比类

**问题描述**：Java中的Queue和Deque接口有什么区别？各自有哪些实现类？

**答案要点**：

**Queue vs Deque：**

```java
// Queue - 单端队列，先进先出
public interface Queue<E> extends Collection<E> {
    boolean add(E e);           // 添加失败抛异常
    boolean offer(E e);         // 添加失败返回false
    E remove();                 // 移除失败抛异常
    E poll();                   // 移除失败返回null
    E element();                // 获取头部失败抛异常
    E peek();                   // 获取头部失败返回null
}

// Deque - 双端队列
public interface Deque<E> extends Queue<E> {
    // 两端操作
    void addFirst(E e);
    void addLast(E e);
    boolean offerFirst(E e);
    boolean offerLast(E e);
    E removeFirst();
    E removeLast();
    E pollFirst();
    E pollLast();
    E getFirst();
    E getLast();
    E peekFirst();
    E peekLast();
    
    // 栈操作
    void push(E e);   // 等价于addFirst
    E pop();          // 等价于removeFirst
}
```

**使用示例：**

```java
// Queue - 先进先出
Queue<String> queue = new LinkedList<>();
queue.offer("A");
queue.offer("B");
queue.poll();  // A

// Deque - 双端队列
Deque<String> deque = new ArrayDeque<>();
deque.offerFirst("A");
deque.offerLast("B");
deque.pollFirst();  // A
deque.pollLast();   // B

// Deque实现栈
Deque<String> stack = new ArrayDeque<>();
stack.push("A");
stack.push("B");
stack.pop();  // B (后进先出)

// ArrayDeque vs LinkedList
// ArrayDeque: 基于数组，性能更好
// LinkedList: 基于链表，支持null元素
```

**实现类对比：**

| 类 | 特点 |
|------|------|
| LinkedList | 双向链表实现，支持null |
| ArrayDeque | 循环数组实现，性能更好，不能有null |
| PriorityQueue | 基于堆结构，元素按优先级排序 |

### Q18: 如何正确遍历集合？foreach和iterator的区别是什么？

**题目类型**：技术对比类

**问题描述**：Java集合的遍历方式有哪些？foreach和iterator有什么区别？

**答案要点**：

**遍历方式对比：**

```java
List<String> list = Arrays.asList("A", "B", "C");

// 1. for循环 - 可获取索引
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}

// 2. 增强for循环(foreach) - 语法糖，编译为iterator
for (String s : list) {
    System.out.println(s);
}

// 3. Iterator - 支持安全删除
Iterator<String> iterator = list.iterator();
while (iterator.hasNext()) {
    String s = iterator.next();
    if (s.equals("B")) {
        iterator.remove();  // 安全删除
    }
}

// 4. ListIterator - 双向遍历
ListIterator<String> listIterator = list.listIterator();
while (listIterator.hasNext()) {
    listIterator.next();
}
while (listIterator.hasPrevious()) {
    listIterator.previous();
}

// 5. forEach方法 - JDK8+
list.forEach(System.out::println);

// 6. Stream API - 函数式
list.stream().forEach(System.out::println);

// Map遍历
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);

// 遍历entry
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + ":" + entry.getValue());
}

// JDK8+ forEach
map.forEach((k, v) -> System.out.println(k + ":" + v));
```

**区别：**

| 特性 | foreach | Iterator |
|------|---------|----------|
| 实现方式 | 语法糖 | 迭代器模式 |
| 编译后 | 转换为Iterator | 原始实现 |
| 安全删除 | 不支持 | 支持remove() |
| 并发修改 | 可能异常 | fail-fast检测 |
| 适用场景 | 简单遍历 | 需要删除操作 |

### Q19: HashMap的key有什么要求？如何自定义key？

**题目类型**：场景解决类

**问题描述**：HashMap的key需要满足什么条件？如果使用自定义对象作为key，需要重写哪些方法？

**答案要点**：

**HashMap对key的要求：**

1. 必须实现hashCode()和equals()方法
2. key应该是不可变的
3. 建议实现Comparable（TreeMap需要）

**自定义Key示例：**

```java
public class OrderKey {
    private String orderId;
    private String userId;
    
    public OrderKey(String orderId, String userId) {
        this.orderId = orderId;
        this.userId = userId;
    }
    
    @Override
    public int hashCode() {
        // 使用Objects.hash保证一致性
        return Objects.hash(orderId, userId);
    }
    
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) 
            return false;
        OrderKey other = (OrderKey) obj;
        return Objects.equals(orderId, other.orderId) &&
               Objects.equals(userId, other.userId);
    }
}

// 使用
Map<OrderKey, Order> orderMap = new HashMap<>();
```

**最佳实践：**

```java
// 1. 常用key类型
Map<String, Integer> map = new HashMap<>();  // String
Map<Integer, String> map2 = new HashMap<>(); // Integer

// 2. 使用不可变对象作为key
public class ImmutableKey {
    private final String id;
    private final String name;
    
    // 省略getter和构造函数
}

// 3. 避免使用可变对象
class MutableKey {
    private String value;  // 可变！
    
    // 如果value改变，hashCode会变，导致找不到
}

// 4. hashCode和equals的契约
// - 同一对象多次调用hashCode必须返回相同值
// - equals返回true的两个对象，hashCode必须相同
// - equals返回false的对象，hashCode可以相同（但会影响性能）
```

### Q2: RAG系统的数据流是怎样的？

**题目类型**：架构设计类

**问题描述**：请描述RAG系统中数据是如何流转的，从文档上传到答案生成的完整流程。

**答案要点：**

**数据流详解：**

```
【文档导入流程】
用户上传文档
    ↓
文件哈希检测 → 重复检测
    ↓
保存原始文件 → 创建文档记录
    ↓
发布解析任务 → RabbitMQ
    ↓
Parser Worker消费
    ↓
文档解析（提取文本、表格、图片）
    ↓
Clean Worker消费
    ↓
文本清洗（去噪、脱敏）
    ↓
Chunk Worker消费
    ↓
智能切分（语义切分）
    ↓
Embedding Worker消费
    ↓
向量生成 + 索引构建
    ↓
更新文档状态为"已就绪"

【查询问答流程】
用户提问
    ↓
Query改写（规范化、多查询、HyDE）
    ↓
混合检索（向量+关键词）
    ↓
RRF融合
    ↓
Rerank重排序
    ↓
上下文组装
    ↓
LLM生成答案
    ↓
返回答案和引用
```

**关键技术点：**
- 异步处理：保证用户体验
- 状态追踪：每个步骤可追溯
- 幂等设计：支持失败重试
- 版本管理：支持文档更新

### Q3: RAG系统的模块之间如何解耦？

**题目类型**：架构设计类

**问题描述**：RAG系统包含多个模块（解析、切分、检索等），模块之间如何解耦？为什么要解耦？

**答案要点：**

**解耦方式：**

1. **消息队列解耦**
```python
# 生产者
await rabbitmq.publish('parse_tasks', {
    'document_id': doc.id,
    'file_path': doc.path
})

# 消费者
@rabbitmq.consumer('parse_tasks')
async def parse_document(task):
    result = await parse_service.parse(task['file_path'])
    await rabbitmq.publish('clean_tasks', result)
```

2. **服务间API解耦**
```
解析服务 → HTTP/gRPC → 清洗服务
清洗服务 → HTTP/gRPC → 切分服务
```

3. **事件驱动解耦**
```
文档解析完成 → 发布事件 → 所有监听该事件的模块响应
```

**解耦的好处：**

| 好处 | 说明 |
|------|------|
| 独立演进 | 各模块可独立迭代 |
| 容错隔离 | 单模块故障不影响其他 |
| 弹性扩展 | 根据负载独立扩缩容 |
| 技术灵活 | 各模块可用不同技术栈 |

### Q6: RAG系统的容错设计如何实现？

**题目类型**：架构设计类

**问题描述**：分布式系统中故障不可避免。RAG系统如何实现容错？

**答案要点：**

**容错策略：**

1. **服务冗余**
```
API服务：至少2个实例
Worker服务：多实例并行
数据库：主从复制
```

2. **消息队列特性**
```python
# RabbitMQ消息持久化
channel.basic_publish(
    exchange='',
    routing_key='parse_tasks',
    body=message,
    properties=pika.BasicProperties(
        delivery_mode=2  # 消息持久化
    )
)

# 消费者确认
def callback(ch, method, properties, body):
    try:
        process_message(body)
        ch.basic_ack(delivery_tag=method.delivery_tag)
    except Exception as e:
        ch.basic_nack(delivery_tag=method.delivery_tag, requeue=True)
```

3. **重试机制**
```python
class RetryPolicy:
    def __init__(self):
        self.max_retries = 3
        self.backoff = [1, 5, 30]  # 秒
    
    def execute(self, func, *args):
        for attempt in range(self.max_retries):
            try:
                return func(*args)
            except RetryableError as e:
                if attempt == self.max_retries - 1:
                    raise
                time.sleep(self.backoff[attempt])
```

4. **熔断降级**
```
向量检索失败 → 降级到关键词检索
LLM生成失败 → 返回预设回复
全链路失败 → 返回友好错误
```

### Q17: RAG系统的缓存策略如何设计？

**题目类型**：项目设计类

**问题描述**：RAG系统涉及多种缓存场景。如何设计完整的缓存策略？

**答案要点：**

**缓存分层设计：**

```
┌─────────────────────────────────────┐
│ L1: 本地缓存 (进程内存)              │
│ - Embedding向量缓存                  │
│ - 热点文档缓存                       │
│ - 配置缓存                           │
│ - TTL: 1-10分钟                     │
└─────────────────────────────────────┘
              ↓ 未命中
┌─────────────────────────────────────┐
│ L2: 分布式缓存 (Redis)               │
│ - Query向量缓存                      │
│ - 会话缓存                           │
│ - 限流计数器                         │
│ - TTL: 1-24小时                     │
└─────────────────────────────────────┘
              ↓ 未命中
┌─────────────────────────────────────┐
│ L3: 数据库 (MySQL/Milvus)           │
│ - 持久化存储                          │
│ - 最终数据源                          │
└─────────────────────────────────────┘
```

**缓存实现：**

```python
class RetrievalCache:
    def __init__(self, redis_client, local_cache):
        self.redis = redis_client
        self.local = local_cache
    
    async def get_vector(self, query: str) -> Optional[np.array]:
        # 1. 先查本地缓存
        cache_key = self._make_key(query)
        local_result = self.local.get(cache_key)
        if local_result:
            return local_result
        
        # 2. 再查Redis
        redis_result = self.redis.get(cache_key)
        if redis_result:
            vector = np.frombuffer(redis_result)
            self.local.set(cache_key, vector, ttl=600)
            return vector
        
        return None
    
    async def set_vector(self, query: str, vector: np.array):
        cache_key = self._make_key(query)
        # 同时写入本地和Redis
        self.local.set(cache_key, vector, ttl=600)
        self.redis.setex(cache_key, vector.tobytes(), ttl=86400)
```

    # 计算存储
    vector_count = doc_count * avg_chunks_per_doc
    storage_gb = (

```python
class DataMigration:
    def __init__(self, source_db, target_db):
        self.source = source_db
        self.target = target_db
    
    async def migrate_vectors(self, batch_size=1000):
        """向量数据迁移"""
        total_migrated = 0
        
        # 1. 创建目标索引
        await self.target.create_index()
        
        # 2. 批量迁移
        cursor = self.source.get_chunks_cursor()
        
        while True:
            batch = await cursor.fetch(batch_size)
            if not batch:
                break
            
            # 3. 转换格式
            vectors = [
                {
                    'id': chunk.id,
                    'vector': chunk.vector,
                    'metadata': chunk.metadata
                }
                for chunk in batch
            ]
            
            # 4. 写入目标
            await self.target.insert(vectors)
            total_migrated += len(vectors)
            
            logger.info(f"已迁移 {total_migrated} 条记录")
        
        # 5. 验证
        source_count = await self.source.count()
        target_count = await self.target.count()
        
        if source_count != target_count:
            raise MigrationError(f"数据量不匹配: {source_count} vs {target_count}")
        
        logger.info(f"迁移完成: {total_migrated} 条记录")
```

1. 双写：新旧系统同时写入

**备份策略：**

| 备份类型 | 频率 | 保存周期 | 恢复时间 |
|----------|------|----------|----------|

| 增量备份 | 每小时 | 7天 | 小时级 |
| 全量备份 | 每天 | 30天 | 天级 |

**灾备切换流程：**

```python
async def disaster_recovery():
    """灾难恢复流程"""
    # 1. 检测主站点故障
    if not await health_check('primary'):
        logger.critical("主站点不可用，触发灾备切换")
    
    # 2. 通知相关人员
    await notify_oncall("灾备切换开始")
    
    # 3. DNS切换
    await switch_dns('primary', 'disaster_recovery')
    
    # 4. 验证服务
    if await health_check('disaster_recovery'):
        logger.info("灾备站点服务正常")
    else:
        # 启动备用方案
        await activate_cold_standby()
    
    # 5. 通知恢复完成
    await notify_oncall("灾备切换完成")
```

class ExternalIntegrationAPI:
    def __init__(self, rag_service):
        self.rag = rag_service
    
    @router.post("/webhook/document_uploaded")
    async def handle_webhook(self, payload: WebhookPayload):
        """接收外部系统的文档上传事件"""

            raise Unauthorized()

        await self.rag.import_document(
            source='external',
            url=doc_info.url,
            metadata=doc_info.metadata
        )
        
        return {"status": "accepted"}

### Q26: 如何设计RAG系统的A/B测试框架？

**题目类型**：综合设计类

**问题描述**：需要测试不同策略的效果。如何设计A/B测试框架？

**答案要点：**

**A/B测试架构：**

```python
class ABTestFramework:
    def __init__(self):
        self.experiments = {}
    
    def create_experiment(self, name, variants: dict):
        """创建实验"""
        self.experiments[name] = {
            'variants': variants,
            'metrics': {k: [] for k in variants.keys()}
        }
    
    def assign(self, user_id: str, experiment_name: str) -> str:
        """分配实验组"""
        experiment = self.experiments[experiment_name]
        bucket = hash(user_id) % 100
        
        cumulative = 0
        for variant, config in experiment['variants'].items():
            cumulative += config.get('traffic', 50)
            if bucket < cumulative:
                return variant
        
        return list(experiment['variants'].keys())[0]
    
    def record(self, experiment_name: str, variant: str, 
               user_id: str, metrics: dict):
        """记录指标"""
        self.experiments[experiment_name]['metrics'][variant].append(metrics)
    
    def analyze(self, experiment_name: str) -> dict:
        """分析实验结果"""
        exp = self.experiments[experiment_name]
        results = {}
        
        for variant, metrics in exp['metrics'].items():
            results[variant] = {
                'count': len(metrics),
                'avg_latency': mean(m.get('latency', 0) for m in metrics),
                'satisfaction': mean(m.get('satisfaction', 0) for m in metrics)
            }
        
        return results

# 使用示例
framework = ABTestFramework()
framework.create_experiment('retrieval_strategy', {
    'control': {'traffic': 50, 'vector_weight': 0.5},
    'treatment': {'traffic': 50, 'vector_weight': 0.7}
})
```

### Q1: 为什么RAG系统需要混合检索而不是单一检索方式？

**题目类型**：技术原理类

**问题描述**：RAG系统通常采用混合检索（向量检索+关键词检索），而不是单一检索。为什么需要混合检索？单一检索有什么局限性？

**答案要点：**

**单一检索的局限性：**

| 检索方式 | 优势 | 局限性 |
|----------|------|--------|
| 向量检索 | 语义理解、模糊匹配 | 对专有名词、编号不敏感 |
| 关键词检索 | 精确匹配、数字、术语 | 无法理解语义、歧义问题 |

**问题场景：**

```
用户Query: "RAG系统的embedding维度是多少？"

向量检索问题：
- "维度"可能被误解为"维度攻击"中的维度
- embedding和"嵌入"、"向量表示"可能匹配不上

关键词检索问题：
- 只能匹配完全相同的词
- 无法理解"维度是多少"问的是数值
```

**混合检索的优势：**

| 场景 | 向量检索 | 关键词检索 | 混合效果 |
|------|----------|------------|----------|
| 语义问题 | 强 | 弱 | 强强联合 |
| 精确术语 | 弱 | 强 | 互补 |
| 口语化表达 | 强 | 弱 | 语义补充 |
| 数字查询 | 弱 | 强 | 精确匹配 |

**RAG系统推荐架构：**
- 向量检索：捕获语义相似性
- 关键词检索：精确匹配专有名词
- 融合：综合两者优势

### Q2: 什么是RRF（Reciprocal Rank Fusion）融合算法？

**题目类型**：技术原理类

**问题描述**：RRF是混合检索中常用的融合算法。什么是RRF？它的原理是什么？如何实现？

**答案要点：**

**RRF算法原理：**

RRF通过计算每个结果在多个检索结果列表中的排名，给出综合评分：

```
RRF_score(d) = Σ 1/(k + rank_i(d))

其中：
- d: 文档
- rank_i(d): 文档d在第i个检索结果中的排名
- k: 常数（通常为60）
```

**示例计算：**

```
向量检索结果: [A, B, C, D, E]  (A排第1, B排第2...)
关键词检索结果: [B, D, A, F, G]

计算RRF分数 (k=60)：
- A: 1/(60+1) + 1/(60+3) = 0.0164 + 0.0159 = 0.0323
- B: 1/(60+2) + 1/(60+1) = 0.0161 + 0.0164 = 0.0325
- C: 1/(60+3) + 0 = 0.0159
- D: 1/(60+4) + 1/(60+2) = 0.0156 + 0.0161 = 0.0317

最终排序: B > A > C > D
```

**实现代码：**

```python
def rrf_fusion(results_list: list[list], k=60) -> list:
    """
    RRF融合多个检索结果列表
    """
    scores = defaultdict(float)
    
    for results in results_list:
        for rank, doc_id in enumerate(results):
            scores[doc_id] += 1 / (k + rank + 1)
    
    # 按分数排序
    sorted_docs = sorted(scores.items(), key=lambda x: x[1], reverse=True)
    return [doc_id for doc_id, _ in sorted_docs]
```

**RRF的优点：**
- 简单高效
- 不需要调参（k一般用默认值60）
- 对排名敏感，不对分数敏感
- 适合不同量级的检索结果融合

### Q3: 如何设计混合检索的权重分配策略？

**题目类型**：技术选型类

**问题描述**：混合检索中，向量检索和关键词检索的权重如何分配？不同场景下应该如何调整？

**答案要点：**

**权重分配原则：**

| 场景 | 向量权重 | 关键词权重 | 原因 |
|------|----------|------------|------|
| 语义问题为主 | 0.7 | 0.3 | 需要语义理解 |
| 精确查询为主 | 0.3 | 0.7 | 需要精确匹配 |
| 通用场景 | 0.5 | 0.5 | 平衡 |
| 专业术语多 | 0.4 | 0.6 | 术语精确匹配重要 |

**动态权重调整：**

```python
def calculate_dynamic_weights(query: str) -> tuple[float, float]:
    """
    根据Query特征动态调整权重
    """
    vector_weight = 0.5
    keyword_weight = 0.5
    
    # 特征检测
    has_semantic_keywords = any(kw in query for kw in [
        '怎么', '如何', '为什么', '是什么', '什么意思'
    ])
    has_exact_keywords = any(kw in query for kw in [
        '版本', '版本号', '参数', '配置', 'API', 'SDK'
    ])
    has_numbers = bool(re.search(r'\d+', query))
    
    # 动态调整
    if has_semantic_keywords:
        vector_weight += 0.2
        keyword_weight -= 0.2
    elif has_exact_keywords or has_numbers:
        keyword_weight += 0.2
        vector_weight -= 0.2
    
    return vector_weight, keyword_weight

def weighted_fusion(vector_results, keyword_results, 
                   vector_weight=0.5, keyword_weight=0.5):
    """
    加权融合
    """
    scores = defaultdict(float)
    
    # 向量检索结果打分
    for rank, (doc_id, score) in enumerate(vector_results):
        scores[doc_id] += vector_weight * (1 / (rank + 1))
    
    # 关键词检索结果打分
    for rank, (doc_id, score) in enumerate(keyword_results):
        scores[doc_id] += keyword_weight * (1 / (rank + 1))
    
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

**实验调优建议：**
- 从0.5/0.5开始
- 根据召回率调整
- A/B测试验证

### Q5: 什么是BM25算法？它与向量检索有什么区别？

**题目类型**：技术原理类

**问题描述**：BM25是关键词检索常用的算法。BM25与向量检索有什么本质区别？各自适合什么场景？

**答案要点：**

**BM25算法原理：**

BM25（Best Matching 25）是一种基于词频的检索算法，核心思想：

```
BM25(d, q) = Σ IDF(qi) × (tf(ti,d) × (k+1)) / (tf(ti,d) + k × (1-b+b × |d|/avgdl))

其中：
- tf: 词频
- |d|: 文档长度
- avgdl: 平均文档长度
- k, b: 调节参数
- IDF: 逆文档频率
```

**与向量检索的对比：**

| 维度 | BM25 | 向量检索 |
|------|------|----------|
| 原理 | 词频统计 | 语义表示 |
| 匹配方式 | 精确匹配 | 语义相似 |
| 歧义处理 | 无法处理 | 可以处理 |
| 同义词 | 无法处理 | 可以处理 |
| 专有名词 | 精确 | 可能不精确 |
| 计算速度 | 快 | 中等 |
| 需要索引 | 是 | 是 |

**互补场景：**

```
Query: "深度学习的优化方法有哪些"

BM25适合：
- 匹配包含"深度学习"、"优化方法"的文档
- 如果文档说"机器学习调参技巧"，BM25可能匹配不上

向量检索适合：
- "深度学习"和"机器学习"语义相似，可以匹配
- "优化方法"和"调参"语义相近，可以匹配

混合检索：
- 综合两者结果
- 既能精确匹配"深度学习"
- 也能语义匹配"机器学习调参"
```

### Q6: 如何处理混合检索中的结果冲突？

**题目类型**：场景解决类

**问题描述**：向量检索和关键词检索可能返回不同的结果，甚至冲突。如何处理这种冲突？

**答案要点：**

**冲突类型及处理：**

1. **排序不一致**
   - 向量检索认为A排第一
   - 关键词检索认为B排第一
   - 处理：RRF融合综合排序

2. **一方结果为空**
   - 向量检索有结果，关键词检索无结果
   - 处理：以有结果的一方为主

3. **结果差异大**
   - 两个检索返回完全不同的文档
   - 处理：检查Query是否有歧义

**处理策略：**

```python
def handle_conflicts(vector_results, keyword_results, threshold=0.3):
    """
    处理混合检索结果冲突
    """
    # 1. 计算结果重叠度
    vector_ids = set(r.id for r in vector_results)
    keyword_ids = set(r.id for r in keyword_results)
    overlap = len(vector_ids & keyword_ids) / len(vector_ids | keyword_ids)
    
    # 2. 根据重叠度调整策略
    if overlap < threshold:
        # 结果差异大，可能Query有歧义
        # 尝试Query改写或返回双通道结果
        return {
            'strategy': 'dual_channel',
            'vector_results': vector_results[:10],
            'keyword_results': keyword_results[:10]
        }
    else:
        # 结果重叠度高，正常融合
        return {
            'strategy': 'fusion',
            'results': rrf_fusion(vector_results, keyword_results)
        }

def smart_fallback(vector_results, keyword_results):
    """
    智能降级策略
    """
    # 如果一方结果为空，以另一方为主
    if not vector_results:
        return keyword_results
    if not keyword_results:
        return vector_results
    
    # 正常情况RRF融合
    return rrf_fusion(vector_results, keyword_results)
```

### Q7: 如何设计混合检索的候选集大小？

**题目类型**：技术选型类

**问题描述**：混合检索时，向量检索和关键词检索各自召回多少候选结果？如何确定？

**答案要点：**

**候选集大小设计：**

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 向量TopN | 50-100 | 语义相似候选 |
| 关键词TopN | 50-100 | 关键词匹配候选 |
| 融合后TopK | 20-50 | 输入重排序 |
| 最终输出 | 5-10 | 输入LLM |

**设计考虑：**

1. **召回率 vs 效率**
   - 候选越多，召回率越高
   - 但后续处理成本也越高

2. **重复率影响**
   - 两个通道结果重叠多 → 可以减少候选
   - 结果差异大 → 需要更多候选

3. **重排序成本**
   - Cross-Encoder重排序成本高
   - 候选过多会拖慢速度

**自适应候选大小：**

```python
def adaptive_topn(query: str, base_topn=50) -> tuple[int, int]:
    """
    根据Query特征自适应调整候选集大小
    """
    # 复杂查询需要更多候选
    if len(query) > 50 or has_complex_structure(query):
        vector_topn = base_topn * 2  # 100
        keyword_topn = base_topn * 2  # 100
    else:
        vector_topn = base_topn
        keyword_topn = base_topn
    
    # 专有名词多可以减少候选
    if count_proper_nouns(query) > 3:
        vector_topn = int(vector_topn * 0.7)
    
    return vector_topn, keyword_topn
```

### Q8: 混合检索中如何实现去重？

**题目类型**：场景解决类

**问题描述**：向量检索和关键词检索可能返回重复的文档。如何在融合时有效去重？

**答案要点：**

**去重策略：**

1. **ID去重**
```python
def deduplicate_by_id(results):
    """基于ID去重"""
    seen = set()
    unique_results = []
    
    for result in results:
        if result.id not in seen:
            seen.add(result.id)
            unique_results.append(result)
    
    return unique_results
```

2. **内容去重**
```python
def deduplicate_by_content(results, similarity_threshold=0.95):
    """基于内容相似度去重"""
    unique = []
    
    for result in results:
        is_duplicate = False
        for unique_result in unique:
            if calculate_similarity(result.content, 
                                   unique_result.content) > similarity_threshold:
                is_duplicate = True
                break
        
        if not is_duplicate:
            unique.append(result)
    
    return unique
```

3. **文档级别去重**
```python
def deduplicate_by_document(results, max_per_doc=3):
    """同一文档保留多个Chunk"""
    doc_chunks = defaultdict(list)
    
    for result in results:
        doc_chunks[result.document_id].append(result)
    
    unique = []
    for doc_id, chunks in doc_chunks.items():
        unique.extend(chunks[:max_per_doc])
    
    return unique
```

**去重时机：**
- 融合前去重：减少融合计算量
- 融合后去重：保证最终结果唯一
- 通常两者结合

### Q9: 如何评估混合检索的效果？

**题目类型**：技术原理类

**问题描述**：混合检索的效果如何评估？有哪些指标？

**答案要点：**

**评估指标体系：**

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| Recall@K | (命中数)/(总相关数) | 召回率 |
| MRR | Σ(1/第一个命中位置)/N | 平均倒数排名 |
| NDCG | 归一化折损累计增益 | 排序质量 |
| Precision@K | 命中数/K | 精确率 |

**分离评估 vs 联合评估：**

```python
class HybridRetrievalEvaluator:
    def evaluate(self, query, ground_truth, hybrid_results):
        # 分别评估向量检索和关键词检索
        vector_recall = self.calculate_recall(
            [r for r in hybrid_results if r.source == 'vector'],
            ground_truth
        )
        keyword_recall = self.calculate_recall(
            [r for r in hybrid_results if r.source == 'keyword'],
            ground_truth
        )
        
        # 评估混合检索
        hybrid_recall = self.calculate_recall(
            hybrid_results,
            ground_truth
        )
        
        # 评估融合效果
        fusion_gain = hybrid_recall - max(vector_recall, keyword_recall)
        
        return {
            'vector_recall': vector_recall,
            'keyword_recall': keyword_recall,
            'hybrid_recall': hybrid_recall,
            'fusion_gain': fusion_gain,  # 混合检索提升了多少
            'coverage': len(set(r.id for r in hybrid_results)) / len(hybrid_results)
        }
```

**融合效果评估：**

| 指标 | 理想值 | 说明 |
|------|--------|------|
| fusion_gain | >5% | 混合比单一检索提升 |
| coverage | <1.5 | 去重效率 |
| overlap_rate | 30-50% | 两通道重叠率 |

### Q10: 混合检索与重排序的关系是什么？

**题目类型**：技术原理类

**问题描述**：混合检索和重排序（Rerank）是什么关系？它们在RAG流程中的顺序是什么？

**答案要点：**

**流程顺序：**

```
Query → 混合检索（向量+关键词）→ RRF融合 → 重排序 → 最终结果
```

**各自职责：**

| 阶段 | 作用 | 输入 | 输出 |
|------|------|------|------|
| 混合检索 | 多通道召回 | Query | 各通道TopN候选 |
| RRF融合 | 综合排序 | 各通道结果 | 融合后TopM |
| 重排序 | 精细排序 | TopM候选 | 最终TopK |

**关系理解：**

1. **混合检索是召回**
   - 扩大候选集
   - 多种方式互补

2. **重排序是精排**
   - 对候选集精细排序
   - 考虑更多特征

3. **协同工作**
```python
def full_retrieval_pipeline(query, query_vector):
    # 1. 多通道检索
    vector_results = vector_search(query_vector, top_n=100)
    keyword_results = keyword_search(query, top_n=100)
    
    # 2. RRF融合
    fused_results = rrf_fusion([vector_results, keyword_results])
    
    # 3. 业务过滤
    filtered = apply_filters(fused_results)
    
    # 4. 重排序
    reranked = rerank(query, filtered[:20])
    
    return reranked[:10]
```

### Q11: 如何实现基于业务规则的检索过滤？

**题目类型**：场景解决类

**问题描述**：实际业务中需要根据权限、部门、时间等条件过滤检索结果。如何实现？

**答案要点：**

**过滤规则类型：**

| 规则类型 | 示例 | 实现方式 |
|----------|------|----------|
| 权限过滤 | 只能看自己部门文档 | expr过滤 |
| 时间过滤 | 只看近一年的文档 | 时间范围过滤 |
| 文档状态 | 只看已发布的文档 | 状态过滤 |
| 类型过滤 | 只检索某类文档 | 类型过滤 |

**Milvus过滤实现：**

```python
def filtered_search(query_vector, user_context, collection):
    """
    带业务过滤的检索
    """
    # 构建过滤表达式
    filters = []
    
    # 1. 权限过滤
    if not user_context.is_admin:
        filters.append(f"department in {user_context.allowed_departments}")
        filters.append(f"visibility_level <= {user_context.clearance_level}")
    
    # 2. 时间过滤
    if user_context.filter_time_range:
        filters.append(f"create_time > '{user_context.start_date}'")
        filters.append(f"create_time < '{user_context.end_date}'")
    
    # 3. 文档状态过滤
    filters.append("status in ['published', 'archived']")
    
    # 4. 组合过滤表达式
    expr = " and ".join(filters)
    
    # 5. 执行过滤检索
    results = collection.search(
        data=[query_vector],
        anns_field="embedding",
        param={"metric_type": "COSINE", "params": {"nprobe": 32}},
        expr=expr,
        limit=50
    )
    
    return results
```

**过滤时机：**
- 向量检索后过滤：数据量大时效率低
- 向量检索前过滤：减少搜索范围
- 两阶段过滤：先用粗粒度过滤，再用细粒度

### Q12: 如何实现基于查询意图的检索策略切换？

### Q15: 如何设计混合检索的结果缓存策略？

**题目类型**：性能优化类

**问题描述**：混合检索计算量大，如何缓存结果提升性能？

**答案要点：**

**缓存策略设计：**

```python
class HybridSearchCache:
    def __init__(self, redis_client):
        self.redis = redis_client
    
    def generate_cache_key(self, query, vector, filters):
        """生成缓存key"""
        # 对向量进行量化，减少精度
        vector_key = np.round(vector, decimals=4).tobytes()
        vector_hash = hashlib.md5(vector_key).hexdigest()
        
        filter_hash = hashlib.md5(str(filters).encode()).hexdigest()
        
        return f"hybrid:{vector_hash}:{filter_hash}"
    
    def get(self, query, vector, filters):
        """获取缓存"""
        key = self.generate_cache_key(query, vector, filters)
        cached = self.redis.get(key)
        
        if cached:
            return json.loads(cached)
        return None
    
    def set(self, query, vector, filters, results):
        """设置缓存"""
        key = self.generate_cache_key(query, vector, filters)
        
        # 缓存结果（限制大小）
        cache_data = {
            'results': results[:20],  # 只缓存Top20
            'timestamp': time.time()
        }
        
        self.redis.setex(key, ttl=3600, value=json.dumps(cache_data))
```

**多级缓存：**

```
L1: 本地内存缓存 (ms级)
    - 热点Query结果
    
L2: Redis分布式缓存 (ms级)
    - 跨实例共享
    - TTL过期
    
L3: 数据库缓存 (s级)
    - 长尾Query结果
    - 持久化
```

**缓存失效：**

| 事件 | 失效策略 |
|------|----------|
| 文档更新 | 清除相关Query缓存 |
| Embedding模型更新 | 全量清除 |
| 定时 | TTL自动过期 |

### Q16: 如何实现混合检索的结果多样性保证？

**题目类型**：技术原理类

**问题描述**：混合检索可能返回内容高度相似的结果。如何保证结果多样性？

**答案要点：**

**MMR（最大边际相关）算法：**

```python
def mmr_selection(query_vector, candidates, k=10, lambda_param=0.5):
    """
    MMR: 平衡相关性和多样性
    """
    selected = []
    remaining = candidates.copy()
    
    while len(selected) < k and remaining:
        best_score = -float('inf')
        best_item = None
        
        for item in remaining:
            # 相关性分数（从混合检索结果）
            relevance = item.hybrid_score
            
            # 多样性分数（与已选结果的最大相似度）
            if selected:
                max_similarity = max(
                    cosine_similarity(item.vector, s.vector) 
                    for s in selected
                )
            else:
                max_similarity = 0
            
            # MMR公式
            mmr_score = lambda_param * relevance - (1 - lambda_param) * max_similarity
            
            if mmr_score > best_score:
                best_score = mmr_score
                best_item = item
        
        selected.append(best_item)
        remaining.remove(best_item)
    
    return selected
```

**多样性保证策略：**

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 文档级别去重 | 同一文档只保留多个Chunk | 避免重复 |
| 语义去重 | 相似度>阈值只保留一个 | 去除冗余 |
| MMR | 平衡相关性和多样性 | 综合场景 |
| 类别分布 | 保证结果来自不同类别 | 分类知识库 |

**组合策略：**

        return self.fuse_and_return(results)

### Q20: 如何设计混合检索的结果排序算法？

**题目类型**：架构设计类

**问题描述**：混合检索需要综合多个维度对结果排序。如何设计一个灵活的排序算法？

**答案要点：**

**多维度排序因素：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 相关性 | 0.5 | 混合检索综合分数 |
| 时效性 | 0.2 | 新文档优先 |
| 权威性 | 0.2 | 官方文档、高等级文档 |
| 质量分 | 0.1 | 解析质量、完整性 |

**排序算法实现：**

```python
def multi_factor_ranking(results, query_context):
    """
    多因素综合排序
    """
    scored_results = []
    
    for result in results:
        # 1. 相关性分数（已归一化到0-1）
        relevance = result.hybrid_score
        
        # 2. 时效性分数
        days_old = (now() - result.create_time).days
        recency = 1 / (1 + days_old / 365)  # 越新越高
        
        # 3. 权威性分数
        authority = {
            'official': 1.0,
            'internal': 0.8,
            'user': 0.5
        }.get(result.source_type, 0.5)
        
        # 4. 质量分数
        quality = result.quality_score  # 解析质量
        
        # 5. 综合分数
        final_score = (
            0.5 * relevance +
            0.2 * recency +
            0.2 * authority +
            0.1 * quality
        )
        
        scored_results.append({
            'result': result,
            'score': final_score,
            'breakdown': {
                'relevance': relevance,
                'recency': recency,
                'authority': authority,
                'quality': quality
            }
        })
    
    # 按综合分数排序
    scored_results.sort(key=lambda x: x['score'], reverse=True)
    
    return scored_results
```

class ABTestFramework:
    def __init__(self):
        self.experiments = {}
    
    def create_experiment(self, name, variants):
        """创建实验"""
        self.experiments[name] = {
            'variants': variants,
            'metrics': {},
            'assignments': {}  # user_id -> variant
        }
    
    def assign_variant(self, user_id, experiment_name):
        """分配实验组"""
        experiment = self.experiments[experiment_name]
        
        if user_id in experiment['assignments']:
            return experiment['assignments'][user_id]
        
        # 随机分配
        variant = random.choice(experiment['variants'])
        experiment['assignments'][user_id] = variant
        return variant
    
    def record_metric(self, user_id, experiment_name, metric_name, value):
        """记录实验指标"""
        variant = self.assignments[experiment_name][user_id]
        self.experiments[experiment_name]['metrics'].setdefault(variant, {})
        self.experiments[experiment_name]['metrics'][variant].setdefault(metric_name, [])
        self.experiments[experiment_name]['metrics'][variant][metric_name].append(value)
    
    def analyze_results(self, experiment_name):
        """分析实验结果"""
        experiment = self.experiments[experiment_name]
        results = {}
        
        for variant, metrics in experiment['metrics'].items():
            results[variant] = {
                'sample_size': len(metrics.get('conversion', [])),
                'conversion_rate': sum(metrics.get('conversion', [])) / max(len(metrics.get('conversion', [])), 1),
                'avg_latency': statistics.mean(metrics.get('latency', [0]))
            }
        
        return results

```python
class IncrementalLearningRetrieval:
    def __init__(self):
        self.model = RetrievalModel()
        self.feedback_buffer = []
        self.update_interval = 1000  # 每1000条反馈更新一次
    
    def record_interaction(self, query, query_vector, clicked_result, 
                          all_results, session_context):
        """记录交互数据"""
        interaction = {
            'query': query,
            'query_vector': query_vector,
            'clicked': clicked_result,
            'shown': [r.id for r in all_results],
            'position': next((i for i, r in enumerate(all_results) 
                              if r.id == clicked_result.id), -1),
            'session': session_context
        }
        self.feedback_buffer.append(interaction)
        
        # 达到阈值则增量更新
        if len(self.feedback_buffer) >= self.update_interval:
            self.incremental_update()
    
    def incremental_update(self):
        """增量更新模型"""
        # 1. 从正反馈学习
        positive_samples = [i for i in self.feedback_buffer if i['position'] < 5]
        
        # 2. 从负反馈学习
        negative_samples = [i for i in self.feedback_buffer if i['position'] >= 10]
        
        # 3. 更新排序模型
        self.model.incremental_fit(
            positive=positive_samples,
            negative=negative_samples
        )
        
        # 4. 清空缓冲区
        self.feedback_buffer = []
        
        logger.info(f"增量更新完成，正样本{len(positive_samples)}，负样本{len(negative_samples)}")
```

### Q26: 如何设计混合检索的熔断机制？

**题目类型**：架构设计类

**问题描述**：某个检索通道故障时，如何快速熔断避免影响整体服务？

**答案要点：**

**熔断器模式：**

```python
class CircuitBreaker:
    def __init__(self, failure_threshold=5, timeout=60):
        self.failure_count = 0
        self.failure_threshold = failure_threshold
        self.timeout = timeout
        self.last_failure_time = None
        self.state = 'CLOSED'  # CLOSED, OPEN, HALF_OPEN
    
    def call(self, func, *args, **kwargs):
        if self.state == 'OPEN':
            if time.time() - self.last_failure_time > self.timeout:
                self.state = 'HALF_OPEN'
            else:
                raise CircuitOpenError("Circuit is OPEN")
        
        try:
            result = func(*args, **kwargs)
            if self.state == 'HALF_OPEN':
                self.state = 'CLOSED'
                self.failure_count = 0
            return result
        except Exception as e:
            self.failure_count += 1
            self.last_failure_time = time.time()
            
            if self.failure_count >= self.failure_threshold:
                self.state = 'OPEN'
            
            raise

class HybridSearchWithCircuitBreaker:
    def __init__(self):
        self.vector_circuit = CircuitBreaker()
        self.keyword_circuit = CircuitBreaker()
    
    async def search(self, query, query_vector):
        results = []
        
        # 向量检索通道
        try:
            vector_results = self.vector_circuit.call(
                self.vector_search, query_vector
            )
            results.extend(vector_results)
        except CircuitOpenError:
            logger.warning("向量检索通道熔断，使用降级策略")
        
        # 关键词检索通道
        try:
            keyword_results = self.keyword_circuit.call(
                self.keyword_search, query
            )
            results.extend(keyword_results)
        except CircuitOpenError:
            logger.warning("关键词检索通道熔断")
        
        # 两通道都失败，降级
        if not results:
            return self.fallback_search()
        
        return self.fuse(results)
```

**熔断策略：**

| 状态 | 行为 | 触发条件 |
|------|------|----------|
| CLOSED | 正常调用 | - |
| OPEN | 快速失败 | 连续5次失败 |
| HALF_OPEN | 允许部分调用 | 熔断超时后 |

class TokenBucketRateLimiter:
    """令牌桶限流"""
    def __init__(self, rate=100, capacity=200):
        self.rate = rate
        self.capacity = capacity
        self.tokens = capacity
        self.last_update = time.time()
    
    def acquire(self, tokens=1):
        now = time.time()
        # 补充令牌
        self.tokens = min(
            self.capacity,
            self.tokens + (now - self.last_update) * self.rate
        )
        self.last_update = now
        
        if self.tokens >= tokens:
            self.tokens -= tokens
            return True
        return False

# 使用

### Q28: 如何设计混合检索的结果新鲜度策略？

**题目类型**：场景解决类

**问题描述**：知识库需要同时展示最新文档和经典文档。如何平衡结果的新鲜度和经典性？

**答案要点：**

**新鲜度策略设计：**

```python
class FreshnessAwareRetrieval:
    def __init__(self):
        self.decay_base = 0.99  # 每天衰减系数
    
    def freshness_score(self, doc, query_time):
        """计算新鲜度分数"""
        age_days = (query_time - doc.create_time).days
        
        if age_days == 0:
            return 1.0  # 今天发布，满分
        elif age_days < 7:
            return self.decay_base ** (age_days * 2)  # 近7天衰减较慢
        else:
            return self.decay_base ** (age_days * 0.5)  # 旧文档缓慢衰减
    
    def balanced_search(self, query, query_vector, freshness_weight=0.2):
        """平衡新鲜度和相关性的检索"""
        # 1. 混合检索
        hybrid_results = self.hybrid_search(query, query_vector, top_n=100)
        
        # 2. 注入新鲜度分数
        now = datetime.now()
        for result in hybrid_results:
            result.relevance_score = result.hybrid_score
            result.freshness_score = self.freshness_score(result, now)
            
            # 3. 综合分数
            result.final_score = (
                (1 - freshness_weight) * result.relevance_score +
                freshness_weight * result.freshness_score
            )
        
        # 4. 按综合分数排序
        hybrid_results.sort(key=lambda x: x.final_score, reverse=True)
        
        return hybrid_results[:10]
```

| 场景 | 新鲜度权重 | 说明 |
|------|------------|------|
| 新闻资讯 | 0.5 | 最新优先 |
| 技术文档 | 0.2 | 相关性优先 |
| 产品说明 | 0.1 | 稳定内容优先 |
| 动态配置 | 0.4 | 版本更新相关 |

class HybridRetrievalEvaluator:
    def comprehensive_evaluation(self, test_queries, ground_truth):
        """
        综合评估混合检索系统
        """
        results = {
            'recall': self.evaluate_recall(test_queries, ground_truth),
            'mrr': self.evaluate_mrr(test_queries, ground_truth),
            'diversity': self.evaluate_diversity(test_queries),
            'latency': self.evaluate_latency(test_queries),
            'coverage': self.evaluate_channel_coverage(test_queries)
        }
        
        # 综合得分
        results['overall'] = (
            0.4 * results['recall'] +
            0.2 * results['mrr'] +
            0.1 * results['diversity'] +
            0.2 * self.normalize_latency(results['latency']) +
            0.1 * results['coverage']
        )
        
        return results
    
    def evaluate_channel_coverage(self, queries):
        """
        评估两通道覆盖率
        """
        vector_only = 0
        keyword_only = 0
        both = 0
        
        for query in queries:
            vector_hits = query.vector_results & query.ground_truth
            keyword_hits = query.keyword_results & query.ground_truth
            
            if vector_hits and not keyword_hits:
                vector_only += 1
            elif keyword_hits and not vector_hits:
                keyword_only += 1
            elif vector_hits and keyword_hits:
                both += 1
        
        # 两通道互补率
        complement_rate = (vector_only + keyword_only) / len(queries)
        
        return {
            'vector_only_rate': vector_only / len(queries),
            'keyword_only_rate': keyword_only / len(queries),
            'both_rate': both / len(queries),
            'complement_rate': complement_rate  # 互补率越高说明混合检索价值越大
        }

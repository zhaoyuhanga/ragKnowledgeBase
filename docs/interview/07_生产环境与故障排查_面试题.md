### Q1: Milvus生产环境的配置建议是什么？

**题目类型**：生产实践类

**问题描述**：Milvus在生产环境部署时，有哪些关键配置需要关注？

**答案要点：**

**关键配置参数：**

| 配置项 | 生产建议值 | 说明 |
|--------|------------|------|
| etcd.endpoints | 3节点集群 | 高可用 |
| minio.bucket | 独立bucket | 数据隔离 |
| pulsar.metadataTenant | 独立tenant | 资源隔离 |
| queryNode.replicas | 根据QPS配置 | 水平扩展 |
| indexNode.replicas | 2-3个 | 索引构建 |

**推荐生产配置：**

```yaml
# milvus.yaml
etcd:
  endpoints:
    - etcd-1:2379
    - etcd-2:2379
    - etcd-3:2379
  rootPath: milvus
  metaSubPath: meta

storage:
  type: minio
  endpoint: minio:9000
  accessKeyID: minioadmin
  secretAccessKey: minioadmin
  useSSL: true
  bucketName: milvus-bucket
  rootPath: files

pulsar:
  address: pulsar://pulsar:6650
  webport: 8080
  tenant: public
  namespace: default

queryNode:
  replicas: 3
  resources:
    limits:
      cpu: 2
      memory: 8Gi
    requests:
      cpu: 1
      memory: 4Gi

indexNode:
  replicas: 2
  resources:
    limits:
      cpu: 4
      memory: 16Gi
```

**资源规划建议：**
- 向量数量 < 1000万：单节点Milvus
- 1000万 ~ 1亿：3节点集群
- 1亿以上：分片集群

```python
async def get_cached_embedding(query: str):
    key = f"embedding:{hashlib.md5(query.encode()).hexdigest()}"
    cached = await redis.get(key)
    if cached:
        return np.frombuffer(cached)
    return None

# 2. 限流
async def rate_limit(user_id: str, limit: int = 100):
    key = f"ratelimit:{user_id}"
    current = await redis.incr(key)
    if current == 1:
        await redis.expire(key, 60)
    return current <= limit

# 3. 分布式锁
async def acquire_lock(task_id: str):
    key = f"lock:task:{task_id}"
    return await redis.set(key, "1", nx=True, ex=30)
```

**问题描述**：生产环境中Milvus查询出现超时，如何系统性地排查？

**答案要点：**

**排查流程：**

```
1. 确认超时范围
   ├─ 单个查询超时 → 检查索引状态
   └─ 批量超时 → 检查系统负载

2. 检查Milvus状态
   ├─ 健康检查接口
   └─ 日志分析

3. 检查资源使用
   ├─ 内存是否不足
   ├─ CPU是否打满
   └─ 网络是否延迟

4. 检查查询参数
   ├─ TopK是否过大
   ├─ 过滤条件是否复杂
   └─ Collection是否需要优化
```

**排查命令：**

```bash
# 1. 检查Milvus健康状态
curl http://milvus-proxy:9091/healthz

# 2. 查看Milvus日志
kubectl logs -f milvus-querynode-xxx | grep "timeout"

# 3. 检查资源使用
docker stats milvus-querynode

# 4. 查看QueryNode状态
curl http://milvus-rootcoord:8081/nodes

# 5. 检查Collection状态

```

**常见原因及解决方案：**

| 原因 | 表现 | 解决方案 |
|------|------|----------|
| 内存不足 | OOM Kill | 增加内存或减少并发 |
| 索引未构建 | FLAT索引很慢 | 触发索引构建 |
| 数据倾斜 | 部分节点负载高 | 重新平衡数据 |
| 网络延迟 | 偶发超时 | 检查网络配置 |

```sql
-- 优化后：
```

### Q9: 生产环境如何进行容量评估和扩容？

**题目类型**：生产实践类

**问题描述**：如何评估RAG系统的容量需求？何时需要扩容？

**答案要点：**

**容量评估模型：**

```python
def capacity_assessment(
    doc_count: int,
    avg_chunks_per_doc: int,
    daily_qa_count: int,
    peak_qps: int
) -> dict:
    """
    容量评估
    """
    # 向量数量
    vector_count = doc_count * avg_chunks_per_doc
    
    # 存储计算
    vector_storage_gb = (
        vector_count * 1024 * 4 / (1024**3)  # float32向量
        * 1.5  # 索引开销
    )
    
    # Milvus内存需求
    milvus_memory_gb = vector_storage_gb * 1.2  # 索引+缓冲
    
    # MySQL存储
    mysql_storage_gb = doc_count * 0.01  # 每文档约10KB
    
    # API服务器
    api_servers = max(2, peak_qps / 200)  # 每台200 QPS
    
    # Worker数量
    workers = max(3, daily_qa_count / 10000)  # 每Worker日处理1万
    
    return {
        'vector_count': vector_count,
        'milvus_memory_gb': milvus_memory_gb,
        'mysql_storage_gb': mysql_storage_gb,
        'api_servers': api_servers,
        'workers': workers
    }
```

**扩容指标：**

| 指标 | 扩容阈值 | 扩容动作 |
|------|----------|----------|
| Milvus内存使用 | >75% | 增加节点 |
| 队列积压 | >5000 | 增加Worker |
| API延迟P99 | >500ms | 增加API实例 |
| 磁盘使用 | >80% | 扩容存储 |

**扩容操作：**

```bash
# Milvus扩容

# API扩容
kubectl scale deployment rag-api --replicas=5
```

```python
# 1. 查询导入任务状态
async def check_import_status(task_id: str):

    # 打印任务详情
```

**降级策略：**

```python
class DegradationService:
    async def qa_with_degradation(self, query: str):
        """
        降级策略：保证系统可用
        """
        try:
            # 尝试完整流程
            return await self.full_qa_pipeline(query)
        except VectorDBTimeout:
            # 降级：只用关键词检索
            logger.warning("向量检索超时，降级到关键词检索")
            return await self.keyword_only_qa(query)
        except LLMTimeout:
            # 降级：返回预设回复
            logger.warning("LLM超时，返回默认回复")
            return self.get_default_answer(query)
        except Exception as e:
            # 最终降级：返回友好错误
            logger.error(f"所有降级失败: {e}")
            return {
                "answer": "系统繁忙，请稍后重试",
                "status": "degraded"
            }
```

**降级级别：**

| 级别 | 策略 | 影响 |
|------|------|------|
| L1 | 关闭查询改写 | 召回略降 |
| L2 | 关闭重排序 | 精度略降 |
| L3 | 只用关键词检索 | 语义召回降 |
| L4 | 返回FAQ缓存 | 无实时生成 |
| L5 | 返回系统繁忙 | 无法服务 |

### Q29: 如何设计系统的健康检查机制？

**题目类型**：应急处理类

**问题描述**：如何设计完善的健康检查机制？

**答案要点：**

**健康检查设计：**

```python
class HealthCheck:
    async def check(self):
        """综合健康检查"""
        results = await asyncio.gather(
            self.check_dependencies(),
            self.check_services(),
            self.check_resources(),
            return_exceptions=True
        )
        
        status = "healthy" if all(r is True for r in results) else "degraded"
        
        return {
            "status": status,
            "checks": {
                "dependencies": results[0],
                "services": results[1],
                "resources": results[2]
            },
            "timestamp": datetime.now().isoformat()
        }
    
    async def check_dependencies(self):
        """检查外部依赖"""
        try:
            # Milvus
            milvus_healthy = await self.check_milvus()
            # MySQL
            mysql_healthy = await self.check_mysql()
            # Redis
            redis_healthy = await self.check_redis()
            # LLM API
            llm_healthy = await self.check_llm()
            
            return milvus_healthy and mysql_healthy and redis_healthy and llm_healthy
        except Exception as e:
            logger.error(f"依赖检查失败: {e}")
            return False
    
    async def check_milvus(self):
        """检查Milvus"""
        try:
            result = await milvus_client.query(
                "SELECT 1",
                collection_name="health_check"
            )
            return True
        except Exception:
            return False
```

**健康检查接口：**

```python
@router.get("/health")
async def health():
    checker = HealthCheck()
    result = await checker.check()
    
    status_code = 200 if result["status"] == "healthy" else 503
    return JSONResponse(result, status_code=status_code)

@router.get("/health/live")
async def liveness():
    """K8s liveness probe"""
    return {"status": "alive"}

@router.get("/health/ready")
async def readiness():
    """K8s readiness probe"""
    checker = HealthCheck()
    result = await checker.check()
    if result["status"] == "healthy":
        return {"status": "ready"}
    return JSONResponse({"status": "not ready"}, status_code=503)
```

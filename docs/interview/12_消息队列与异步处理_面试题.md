### Q1: RAG系统中为什么需要消息队列？

**题目类型**：技术原理类

**问题描述**：RAG系统包含文档解析、切分、向量化等多个步骤，为什么需要使用消息队列？

**答案要点：**

**使用消息队列的原因：**

| 原因 | 说明 | 好处 |
|------|------|------|
| 解耦 | 各模块通过消息通信 | 独立演进 |
| 异步 | 非阻塞处理 | 快速响应 |
| 削峰 | 缓冲流量高峰 | 保护系统 |
| 可靠 | 消息持久化 | 保证不丢失 |

**RAG系统中的消息流：**

```
用户上传文档
    ↓
发布消息 → parse_queue
    ↓
Parser Worker消费
    ↓
发布消息 → clean_queue
    ↓
Clean Worker消费
    ↓
发布消息 → chunk_queue
    ↓
Chunk Worker消费
    ↓
发布消息 → embedding_queue
    ↓
Embedding Worker消费
```

**对比：无队列 vs 有队列**

| 维度 | 同步处理 | 异步队列 |
|------|----------|----------|
| 用户体验 | 等待完成 | 立即返回 |
| 峰值处理 | 易崩溃 | 缓冲处理 |
| 模块耦合 | 紧耦合 | 松耦合 |
| 失败处理 | 困难 | 重试机制 |

### Q2: RabbitMQ和Kafka有什么区别？RAG系统如何选择？

**题目类型**：技术对比类

**问题描述**：RabbitMQ和Kafka都是常用的消息队列，它们有什么区别？RAG系统应该如何选择？

**答案要点：**

**核心对比：**

| 特性 | RabbitMQ | Kafka |
|------|----------|-------|
| 消息模型 | 队列+交换机 | 分区+Topic |
| 消息顺序 | 单队列有序 | 分区内有序 |
| 消息持久化 | 支持 | 支持 |
| 吞吐量 | ~10万/秒 | ~百万/秒 |
| 延迟 | 低(亚毫秒) | 毫秒级 |
| 消息确认 | ACK机制 | 偏移量提交 |
| 适用场景 | 任务队列 | 日志流 |

**RAG系统选择建议：**

| 场景 | 推荐 | 原因 |
|------|------|------|
| 文档导入任务 | RabbitMQ | 任务量可控，需要确认机制 |
| 日志采集 | Kafka | 高吞吐 |
| 事件驱动 | RabbitMQ | 灵活的路由 |
| 大数据流处理 | Kafka | 百万级吞吐 |

**RAG系统推荐架构：**

### Q4: RabbitMQ的消息确认机制是怎样的？

**题目类型**：技术原理类

**问题描述**：RabbitMQ的消息确认机制是什么？不同确认模式的区别是什么？

**答案要点：**

**确认模式：**

| 模式 | 说明 | 可靠性 | 性能 |
|------|------|--------|------|
| 自动ACK | 消息投递给消费者即确认 | 低 | 高 |
| 手动ACK | 消费者显式确认 | 高 | 稍低 |

**手动ACK示例：**

```python
def callback(ch, method, properties, body):
    try:
        # 处理消息
        process_document(body)
        
        # 成功处理，手动确认
        ch.basic_ack(delivery_tag=method.delivery_tag)
    
    except Exception as e:
        # 处理失败，消息重新入队
        ch.basic_nack(
            delivery_tag=method.delivery_tag,
            requeue=True  # 重新入队
        )

# 消费消息
channel.basic_qos(prefetch_count=10)  # 最多预取10条
channel.basic_consume(
    queue='parse_queue',
    on_message_callback=callback,
    auto_ack=False  # 手动确认
)
```

**ACK时机选择：**

| 场景 | 建议 | 原因 |
|------|------|------|
| 耗时任务 | 处理完成后ACK | 确保处理成功 |
| 幂等操作 | 可自动ACK | 处理是幂等的 |
| 关键任务 | 手动ACK | 不容丢失 |

**题目类型**：技术原理类

**问题描述**：RabbitMQ的死信队列（Dead Letter Exchange）是什么？RAG系统中如何使用？

**答案要点：**

**死信队列场景：**

| 场景 | 说明 | 处理 |
|------|------|------|
| 消息被拒绝 | 消费者reject且requeue=False | 死信 |
| 消息超时 | TTL过期 | 死信 |
| 队列满 | 超出容量 | 死信 |

**死信队列配置：**

```python

channel.queue_declare(queue='parse_dlx', durable=True)

# 创建主队列，配置死信
channel.queue_declare(
    queue='parse_queue',
    durable=True,
    arguments={
        'x-dead-letter-exchange': 'dlx',  # 死信交换机
        'x-dead-letter-routing-key': 'parse.failed',  # 死信路由键
        'x-message-ttl': 3600000  # 消息1小时过期
    }
)
```

**RAG系统中的应用：**

```python
# 解析任务处理流程
async def process_parse_task(task):
    try:
        result = await parse_document(task)
        return result
    except ParseError as e:
        # 解析失败，进入死信队列待人工处理
        await publish_to_dlx({
            'task': task,
            'error': str(e),
            'failed_at': datetime.now()
        })
        raise

# 死信队列消费者（告警/人工处理）
async def handle_failed_tasks():
    while True:
        task = await dlx_consumer.get()
        await send_alert(task)  # 发送告警
        await mark_for_review(task)  # 标记待人工处理
```

### Q7: RabbitMQ如何实现延迟消息？

**TTL+死信方案：**

```python
    durable=True,

        'x-dead-letter-routing-key': 'document.parse',
        'x-message-ttl': 5000  # 5秒延迟
    }
)

# 发送延迟消息

    exchange='',
    routing_key='delay_parse_queue',
    body=message
)

# 5秒后消息自动进入parse_queue
```

**重试场景应用：**

```python
async def process_with_retry(task, max_retries=3):
    for attempt in range(max_retries):
        try:
            return await process_task(task)
        except RetryableError as e:
            if attempt == max_retries - 1:
                raise
            
            # 延迟重试（指数退避）
            delay = 2 ** attempt
            await publish_with_delay('retry_queue', task, delay)
```

### Q13: 如何设计RAG系统的异步任务处理框架？

**题目类型**：架构设计类

**问题描述**：RAG系统有多个异步任务，如何设计一个统一的任务处理框架？

**答案要点：**

**任务处理框架设计：**

```python
class TaskProcessor:
    def __init__(self):
        self.handlers = {}
        self.rabbitmq = RabbitMQConnection()
        self.retry_policy = RetryPolicy()
    
    def register(self, task_type: str, handler: Callable):
        """注册任务处理器"""
        self.handlers[task_type] = handler
    
    async def start(self):
        """启动任务处理器"""
        for task_type, handler in self.handlers.items():
            await self.rabbitmq.consume(
                queue=f'{task_type}_queue',
                callback=lambda msg: self.process(task_type, msg)
            )
    
    async def process(self, task_type: str, message):
        """处理任务"""
        handler = self.handlers[task_type]
        
        try:
            result = await handler(message.payload)
            await self.on_success(message, result)
        except RetryableError as e:
            await self.retry_policy.retry(message)
        except Exception as e:
            await self.on_failure(message, e)

# 使用示例
processor = TaskProcessor()

@processor.register('parse')
async def handle_parse(message):
    return await parse_document(message['file_path'])

@processor.register('embed')
async def handle_embed(message):
    return await embed_chunks(message['chunks'])
```

### Q14: 如何处理异步任务中的异常？

**题目类型**：场景解决类

**问题描述**：异步任务执行中可能出现异常，如何处理？

**答案要点：**

**异常处理模式：**

```python
class TaskError(Exception):
    """可重试错误"""
    pass

class FatalError(Exception):
    """不可重试错误"""
    pass

class RetryPolicy:
    def __init__(self, max_retries=3, backoff=[1, 5, 30]):
        self.max_retries = max_retries
        self.backoff = backoff
    
    async def retry(self, task):
        """带退避的重试"""
        if task.retry_count >= self.max_retries:
            # 进入死信队列
            await task.send_to_dlx()
            return
        
        delay = self.backoff[task.retry_count]
        await asyncio.sleep(delay)
        
        task.retry_count += 1
        await task.republish()
```

**异常分类处理：**

```python
async def safe_process(task):
    try:
        return await process_task(task)
    
    except NetworkError as e:
        # 网络错误，可重试
        await task.retry_with_delay(delay=5)
    
    except InvalidFormatError as e:
        # 格式错误，不可重试
        await task.mark_failed(str(e))
```

```python
class BatchProcessor:
    def __init__(self, batch_size=100, flush_interval=5):
        self.batch_size = batch_size
        self.flush_interval = flush_interval
        self.buffer = []
        self.last_flush = time.time()
    
    async def add(self, task):
        """添加任务到批次"""
        self.buffer.append(task)
        
        # 达到批次大小，触发处理
        if len(self.buffer) >= self.batch_size:
            await self.flush()
        
        # 超时触发
        elif time.time() - self.last_flush > self.flush_interval:
            await self.flush()
    
    async def flush(self):
        """处理当前批次"""
        if not self.buffer:
            return
        
        tasks = self.buffer
        self.buffer = []
        self.last_flush = time.time()
        
        # 批量处理
        await self.process_batch(tasks)

# 使用示例
```

```python
import asyncio

class RateLimiter:
    """令牌桶限流器"""
    
    def __init__(self, rate: int, capacity: int):
        self.rate = rate  # 每秒生成的令牌数
        self.capacity = capacity  # 令牌桶容量
        self.tokens = capacity
        self.last_update = time.time()
    
    async def acquire(self):
        """获取令牌"""
        while True:
            now = time.time()
            # 补充令牌
            self.tokens = min(
                self.capacity,
                self.tokens + (now - self.last_update) * self.rate
            )
            self.last_update = now
            
            if self.tokens >= 1:
                self.tokens -= 1
                return
            
            await asyncio.sleep(0.1)

# 使用限流器

    await rate_limiter.acquire()
    return await embedding_service.encode(chunks)
```

### Q24: 如何设计异步任务的公平调度？

**题目类型**：技术原理类

**问题描述**：多个任务需要公平调度，避免某些任务一直等待，如何设计？

**答案要点：**

**公平调度策略：**

```python
class FairScheduler:
    """公平调度器"""
    
    def __init__(self, queues: dict):
        self.queues = queues  # {tenant_id: queue}
        self.current = 0  # 当前调度到的租户索引
        self.tenants = list(queues.keys())
    
    async def get_next_task(self):
        """获取下一个任务（轮询）"""
        attempts = 0
        while attempts < len(self.tenants):
            tenant = self.tenants[self.current]
            self.current = (self.current + 1) % len(self.tenants)
            
            task = await self.queues[tenant].get()
            if task:
                return task
            
            attempts += 1
        
        # 所有队列都为空，等待
        return None
```

**加权公平调度：**

```python
class WeightedFairScheduler:
    """加权公平调度"""
    
    def __init__(self, weights: dict):
        # weights = {'tenant1': 3, 'tenant2': 1}
        self.weights = weights
        self.current_counts = {k: 0 for k in weights}
    
    async def get_next_task(self):
        """获取下一个任务（加权轮询）"""
        # 选择权重余额最高的租户
        candidates = [
            (tenant, self.weights[tenant] - self.current_counts[tenant])
            for tenant in self.weights
        ]
        
        tenant, _ = max(candidates, key=lambda x: x[1])
        self.current_counts[tenant] += 1
        
        task = await self.queues[tenant].get()
        return task
```

### Q25: 如何实现异步任务的依赖调度？

**题目类型**：技术原理类

**问题描述**：某些任务有依赖关系（如A完成后才能执行B），如何实现依赖调度？

**答案要点：**

**依赖调度设计：**

```python
class DependencyScheduler:
    def __init__(self):
        self.task_graph = {}  # {task_id: [dependency_ids]}
        self.pending_tasks = {}  # {task_id: task}
        self.running_tasks = set()
    
    def add_task(self, task_id: str, dependencies: list):
        """添加带依赖的任务"""
        self.task_graph[task_id] = dependencies
        self.pending_tasks[task_id] = task
    
    def get_ready_tasks(self) -> list:
        """获取可执行的任务"""
        ready = []
        
        for task_id in list(self.pending_tasks.keys()):
            deps = self.task_graph[task_id]
            
            # 检查依赖是否都完成
            if all(dep not in self.pending_tasks for dep in deps):
                ready.append(self.pending_tasks.pop(task_id))
        
        return ready
    
    def on_task_complete(self, task_id: str):
        """任务完成回调"""
        self.running_tasks.discard(task_id)
        
        # 检查是否有任务现在可执行
        ready_tasks = self.get_ready_tasks()
        for task in ready_tasks:
            self.execute_task(task)
```

```python
class OrderedProcessor:
    """顺序处理器"""
    
    def __init__(self):
        self.processors = {}  # {sequence_id: processing}
        self.completed = {}  # {sequence_id: result}
    
    async def process_ordered(self, task):
        """顺序处理消息"""
        seq_id = task.sequence_id

        if seq_id == self.get_first_pending(seq_id):

        while not self.is_ready(seq_id):

    def is_ready(self, seq_id):

        prev_id = self.get_previous(seq_id)
        if prev_id is None:
            return True
        return prev_id in self.completed
```

**问题描述**：异步任务失败需要及时告警，如何设计告警机制？

**答案要点：**

**告警机制设计：**

```python
class TaskAlertManager:
    def __init__(self):
        self.alert_channels = {
            'email': EmailAlertChannel(),
            'sms': SMSAlertChannel(),
            'dingtalk': DingTalkAlertChannel()
        }
    
    async def on_task_failed(self, task):
        """任务失败告警"""
        alert_level = self.determine_alert_level(task)
        
        if alert_level == 'critical':
            await self.send_critical_alert(task)
        elif alert_level == 'warning':
            await self.send_warning_alert(task)
    
    def determine_alert_level(self, task):
        """判断告警级别"""
        if task.retry_count >= 3:
            return 'critical'  # 重试3次失败
        elif task.error_type == 'OutOfMemoryError':
            return 'critical'  # 内存错误
        elif task.is_business_critical:
            return 'warning'  # 业务关键任务
        return 'info'
```

### Q29: 如何设计异步任务的补偿机制？

**题目类型**：架构设计类

**问题描述**：任务部分成功需要补偿（如部分数据写入后失败），如何设计补偿机制？

**答案要点：**

**补偿事务模式：**

```python
class CompensatingTransaction:
    """补偿事务"""
    
    async def execute_with_compensation(self, steps: list):
        """
        执行带补偿的操作
        steps = [
            {'action': 'save_document', 'compensate': 'delete_document'},
            {'action': 'create_chunks', 'compensate': 'delete_chunks'},
            {'action': 'embed_vectors', 'compensate': 'delete_vectors'}
        ]
        """
        completed = []
        
        try:
            for step in steps:
                result = await self.execute_step(step['action'])
                completed.append((step['action'], result))
        
        except Exception as e:
            # 执行补偿
            for action, result in reversed(completed):
                compensate_fn = self.get_compensate_fn(action)
                await compensate_fn(result)
            
            raise
```

**Saga模式实现：**

```python
class SagaOrchestrator:
    """Saga编排器"""
    
    async def execute_saga(self, saga_id: str, steps: list):
        """执行Saga"""
        saga = {
            'id': saga_id,
            'steps': steps,
            'completed': [],
            'status': 'running'
        }
        
        try:
            for step in steps:
                result = await self.execute_step(step)
                saga['completed'].append(step)
            
            saga['status'] = 'completed'
        
        except Exception as e:
            # 补偿已完成的步骤
            await self.compensate(saga)
            saga['status'] = 'compensated'
        
        return saga
```

### Q30: 如何设计异步任务的可观测性？

**题目类型**：运维实践类

**问题描述**：异步任务需要完整可观测，如何设计可观测性体系？

**答案要点：**

**可观测性设计：**

```python
class ObservableTask:
    """可观测任务"""
    
    def __init__(self, task_id: str):
        self.task_id = task_id
        self.start_time = None
        self.end_time = None
        self.events = []
        self.spans = []
    
    def start_span(self, name: str):
        """开始追踪Span"""
        span = {
            'name': name,
            'start': time.time()
        }
        self.spans.append(span)
        return len(self.spans) - 1
    
    def end_span(self, span_index: int):
        """结束Span"""
        self.spans[span_index]['end'] = time.time()
        self.spans[span_index]['duration'] = (
            self.spans[span_index]['end'] - self.spans[span_index]['start']
        )
    
    def log_event(self, event_type: str, data: dict):
        """记录事件"""
        self.events.append({
            'type': event_type,
            'timestamp': time.time(),
            'data': data
        })
    
    def to_trace(self):
        """转换为分布式追踪格式"""
        return {
            'task_id': self.task_id,
            'duration': self.end_time - self.start_time if self.end_time else None,
            'spans': self.spans,
            'events': self.events
        }
```

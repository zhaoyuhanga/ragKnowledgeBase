## 一、数据库选型与架构

### 1.1 数据库选型矩阵

| 数据库类型 | 选型 | 版本 | 用途 | 数据量预估 |
|-----------|------|------|------|-----------|
| **关系型** | MySQL 8.0 | 8.0.35+ | 用户、权限、配置等结构化数据 | < 1TB |
| **向量数据库** | Milvus | 2.4+ | 文档向量存储与检索 | < 10TB |
| **全文搜索** | Elasticsearch | 8.x | 关键词检索、日志分析 | < 5TB |
| **缓存** | Redis Cluster | 7.0+ | 会话、热点数据、队列 | < 100GB |
| **对象存储** | MinIO | RELEASE | 文档文件、二进制数据 | 无限制 |
| **时序存储** | MongoDB | 6.0+ | 审计日志、访问日志 | < 50TB |

## 二、命名规范

### 2.1 通用命名规范

```sql
-- 数据库命名
{ratabase_name}_{module}_{environment}

示例：
- rag_qa_user_prod        -- 用户模块生产库
- rag_qa_document_prod    -- 文档模块生产库
- rag_qa_analytics_prod   -- 分析模块生产库

-- 表命名
{module}_{entity}

示例：
- user_account           -- 用户账号表
- user_role              -- 用户角色表
- document_info          -- 文档信息表
- knowledge_base         -- 知识库表

-- 字段命名（蛇形命名）
{entity}_id, {entity}_name, {entity}_type, created_at, updated_at

-- 索引命名
idx_{table}_{column1}_{column2}...
idx_{table}_{column}_unique        -- 唯一索引

-- 外键命名
fk_{table}_{ref_table}

-- 视图命名
v_{view_name}
v_{module}_{view_name}
```

### 2.2 字段类型规范

| 数据类型 | 存储空间 | 使用场景 |
|---------|---------|---------|
| **TINYINT** | 1 byte | 状态、布尔值 |
| **SMALLINT** | 2 bytes | 小范围数值 |
| **INT** | 4 bytes | 主键、计数值 |
| **BIGINT** | 8 bytes | 大数值、文件大小 |
| **VARCHAR(n)** | 变长 | 文本字段（< 65535字符） |
| **TEXT** | 变长 | 大文本（> 65535字符） |
| **DECIMAL(p,s)** | 变长 | 金额、精确数值 |
| **DATETIME** | 8 bytes | 时间戳 |
| **JSON** | 变长 | 灵活结构数据 |
| **UUID** | 36 bytes | 分布式ID |

## 三、核心表结构设计

### 3.1 用户模块 (user)

```sql
-- ================================================================
-- 用户模块表结构
-- ================================================================

-- 用户账号表
CREATE TABLE user_account (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '用户ID',
    uuid            CHAR(36) NOT NULL UNIQUE COMMENT 'UUID标识',
    username        VARCHAR(50) NOT NULL COMMENT '用户名',
    email           VARCHAR(255) NOT NULL COMMENT '邮箱',
    phone           VARCHAR(20) DEFAULT NULL COMMENT '手机号',
    password_hash    VARCHAR(255) NOT NULL COMMENT '密码哈希',
    avatar_url      VARCHAR(500) DEFAULT NULL COMMENT '头像URL',
    status          TINYINT NOT NULL DEFAULT 1 COMMENT '状态: 1-正常 2-禁用 3-删除',
    email_verified  TINYINT NOT NULL DEFAULT 0 COMMENT '邮箱已验证',
    phone_verified  TINYINT NOT NULL DEFAULT 0 COMMENT '手机已验证',
    mfa_enabled     TINYINT NOT NULL DEFAULT 0 COMMENT 'MFA已启用',
    mfa_secret      VARCHAR(255) DEFAULT NULL COMMENT 'MFA密钥',
    last_login_at   DATETIME DEFAULT NULL COMMENT '最后登录时间',
    last_login_ip   VARCHAR(45) DEFAULT NULL COMMENT '最后登录IP',
    login_count     INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '登录次数',
    created_by      BIGINT UNSIGNED DEFAULT NULL COMMENT '创建人',
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    UNIQUE KEY uk_username (username),
    UNIQUE KEY uk_email (email),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户账号表';

-- 用户角色表
CREATE TABLE user_role (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY COMMENT '角色ID',
    name            VARCHAR(50) NOT NULL COMMENT '角色名称',
    code            VARCHAR(50) NOT NULL COMMENT '角色代码',
    description     VARCHAR(255) DEFAULT NULL COMMENT '描述',
    permissions     JSON NOT NULL COMMENT '权限列表',
    is_system       TINYINT NOT NULL DEFAULT 0 COMMENT '系统内置: 0-否 1-是',
    sort_order      INT NOT NULL DEFAULT 0 COMMENT '排序',
    status          TINYINT NOT NULL DEFAULT 1 COMMENT '状态: 1-启用 2-禁用',
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    UNIQUE KEY uk_code (code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户角色表';

-- 用户-角色关联表
CREATE TABLE user_role_relation (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id         BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
    role_id         BIGINT UNSIGNED NOT NULL COMMENT '角色ID',
    granted_by      BIGINT UNSIGNED DEFAULT NULL COMMENT '授权人',
    granted_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '授权时间',
    
    UNIQUE KEY uk_user_role (user_id, role_id),
    INDEX idx_role_id (role_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户角色关联表';

-- 用户配额表
CREATE TABLE user_quota (
    id                  BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id            BIGINT UNSIGNED NOT NULL UNIQUE COMMENT '用户ID',
    tier                VARCHAR(20) NOT NULL DEFAULT 'free' COMMENT '套餐: free/pro/enterprise',
    
    -- 文档配额
    documents_limit     INT NOT NULL DEFAULT 50 COMMENT '文档数量上限',
    documents_used     INT NOT NULL DEFAULT 0 COMMENT '已用文档数',
    
    -- 存储配额
    storage_mb_limit    BIGINT NOT NULL DEFAULT 512 COMMENT '存储上限(MB)',
    storage_mb_used    BIGINT NOT NULL DEFAULT 0 COMMENT '已用存储(MB)',
    
    -- API配额
    api_calls_day_limit INT NOT NULL DEFAULT 100 COMMENT '日API调用上限',
    api_calls_day_used  INT NOT NULL DEFAULT 0 COMMENT '当日API调用',
    
    -- Token配额
    token_month_limit   BIGINT NOT NULL DEFAULT 100000 COMMENT '月Token上限',
    token_month_used    BIGINT NOT NULL DEFAULT 0 COMMENT '当月Token消耗',
    
    -- 时间戳
    quota_reset_at      DATE NOT NULL COMMENT '配额重置日期',
    updated_at          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_quota_reset_at (quota_reset_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户配额表';

-- API密钥表
CREATE TABLE api_key (
    id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id         BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
    key_name        VARCHAR(100) NOT NULL COMMENT '密钥名称',
    key_prefix      VARCHAR(20) NOT NULL COMMENT '密钥前缀(rag_xxx)',
    key_hash        VARCHAR(255) NOT NULL COMMENT '密钥哈希',
    scopes          JSON NOT NULL COMMENT '权限范围',
    rate_limit      INT DEFAULT NULL COMMENT '速率限制(次/分钟)',
    expires_at      DATETIME DEFAULT NULL COMMENT '过期时间',
    last_used_at    DATETIME DEFAULT NULL COMMENT '最后使用时间',
    is_active       TINYINT NOT NULL DEFAULT 1 COMMENT '状态: 1-启用 2-禁用',
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_user_id (user_id),
    INDEX idx_key_prefix (key_prefix),
    INDEX idx_expires_at (expires_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='API密钥表';
```

## 四、索引设计规范

### 4.1 索引创建原则

| 场景 | 索引类型 | 说明 |
|------|----------|------|
| 主键 | PRIMARY KEY | 自增 BIGINT |
| 唯一约束 | UNIQUE KEY | 确保字段唯一性 |
| 外键关联 | INDEX | 加速关联查询 |
| 高频查询条件 | INDEX | 常见 WHERE 条件 |
| 排序字段 | INDEX | 加速 ORDER BY |
| 分页查询 | INDEX | 配合 OFFSET 使用 |

### 4.2 索引命名规范

```sql
-- 普通索引
idx_{table}_{column}
idx_document_info_knowledge_base_id

-- 唯一索引
uk_{table}_{column}
uk_user_account_email

-- 联合索引
idx_{table}_{column1}_{column2}_{column3}
idx_qa_log_user_kb_created (user_id, knowledge_base_id, created_at)

-- 外键索引
idx_{table}_{foreign_table}_id
idx_document_chunk_document_id
```

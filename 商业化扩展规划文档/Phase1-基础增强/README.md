**日期：** 2026-05-17  
**预计周期：** 2-3 周

## 一、阶段概述

### 1.1 阶段目标

从 Demo 转向可交付产品的基础能力建设，实现多用户隔离、知识库分类管理和文档处理增强。

### 1.2 核心功能

| 模块 | 功能点 | 优先级 | 工作量 |
|------|--------|--------|--------|
| 多用户权限 | 用户管理、RBAC 权限 | P0 | 3 天 |
| 知识库分类 | 多知识库、分类、标签 | P0 | 2 天 |
| 文档管理增强 | 批量上传、版本管理 | P1 | 2 天 |
| API 密钥 | API Key 生成与管理 | P1 | 2 天 |

## 二、功能需求详述

### 2.1 多用户权限系统

#### 2.1.1 用户管理

**功能描述**：
- 用户注册（邮箱/手机）
- 用户登录（用户名密码、JWT Token）
- 密码找回与重置
- 用户信息管理
- 用户状态管理（启用/禁用）

**需求规格**：

| 需求ID | 需求描述 | 验收条件 | 验证方法 |
|--------|----------|----------|----------|
| U-001 | 用户注册 | 支持邮箱/用户名注册，密码加密存储 | 注册后登录验证 |
| U-002 | 用户登录 | 支持用户名密码登录，返回 JWT Token | 调用登录接口验证 |
| U-003 | Token 刷新 | 支持刷新过期 Token | Token 过期后刷新 |
| U-004 | 密码重置 | 支持邮箱验证码重置密码 | 发送重置邮件并重置 |
| U-005 | 用户信息更新 | 用户可修改基本信息 | 修改后验证 |
| U-006 | 用户列表 | 管理员可查看所有用户 | 管理员登录查看 |
| U-007 | 用户禁用 | 管理员可禁用/启用用户 | 禁用后无法登录 |

**接口设计**：

```
POST   /api/v1/auth/register         # 用户注册
POST   /api/v1/auth/login           # 用户登录
POST   /api/v1/auth/logout          # 用户登出
POST   /api/v1/auth/refresh        # 刷新Token
POST   /api/v1/auth/forgot-password # 忘记密码
POST   /api/v1/auth/reset-password # 重置密码

GET    /api/v1/users               # 用户列表
POST   /api/v1/users               # 创建用户
GET    /api/v1/users/{id}          # 用户详情
PUT    /api/v1/users/{id}          # 更新用户
PATCH  /api/v1/users/{id}/status   # 修改用户状态
DELETE /api/v1/users/{id}          # 删除用户
```

| R-004 | 分配权限 | 可为角色分配权限 | 分配后用户继承 |

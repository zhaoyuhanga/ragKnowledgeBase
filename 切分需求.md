新切分器要求：

1. 新增或重写为 SemanticChunker / StructuredChunker。
2. 支持中文文档。
3. 解析并保留结构：
  - Markdown 标题层级：# / ## / ###
    - 自然段
    - 列表
    - Markdown 表格
    - 代码块
    - PDF 页码字段，如果上游 parser 有提供
4. 按标题层级构建 section。
5. 每个 section 生成 parent section。
6. section 内生成 child chunks。
7. 检索时使用 child chunk。
8. 生成答案时可通过 parent_section_id 回填上下文。

切分策略：

1. 优先按结构切，不按固定字数硬切。
2. 段落按语义合并到目标 token 范围。
3. 超长段落按中文标点二次切分。
4. 表格优先整表保留。
5. 超长表格按“表头 + 多行”拆分。
6. 列表尽量整体保留。
7. 超长列表按列表项拆分。
8. 代码块整体保留。
9. 超长代码块按行数拆分。
10. 每个 chunk 支持 overlap。

默认配置：

- CHUNK_TARGET_TOKENS=600
- CHUNK_MAX_TOKENS=900
- CHUNK_MIN_TOKENS=120
- CHUNK_OVERLAP_TOKENS=100
- CHUNK_VERSION=semantic-v1

每个 chunk 必须包含：

- id 或 chunk_index
- document_id
- parent_section_id
- title_path
- section_level
- block_type: title / paragraph / list / table / code / mixed
- content
- enhanced_content
- page_no，如果没有则为 null
- token_count
- char_count
- content_hash
- chunk_version

enhanced_content 规则：

- 用于 embedding 和检索。
- 内容格式包含标题路径 + 正文。
- 不要污染原始 content。
- LLM 回答引用时优先展示原始 content、title_path、page_no。

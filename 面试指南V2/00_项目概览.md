# RAG 问答系统 - 项目概览

## 1. 项目简介

本项目是一个**基于 RAG（Retrieval-Augmented Generation，检索增强生成）技术的智能问答系统**，旨在为企业和个人提供私有化知识库问答解决方案。用户可以上传文档，系统自动解析、切分、向量化存储，当用户提问时，系统通过向量检索找到相关内容，结合大语言模型生成准确的回答。

## 2. 核心功能

| 功能模块 | 功能点 | 描述 |
|---------|--------|------|
| **文档管理** | 多格式支持 | 支持 PDF、DOCX、Markdown、TXT 等格式文档上传 |
| | 文档解析 | 自动提取文档文本内容 |
| | 知识库管理 | 文档列表查看、状态追踪、删除管理 |
| **智能问答** | 向量检索 | 基于语义相似度的文档检索 |
| | LLM 生成 | 结合检索内容生成准确回答 |
| | 多轮对话 | 支持会话上下文关联 |
| | 流式输出 | SSE 流式响应，提升用户体验 |
| **系统配置** | 参数调优 | 运行时配置检索参数 |
| | 健康检查 | 多组件状态监控 |
| | 缓存管理 | 问答结果缓存与清理 |

## 6. 核心流程

### 6.1 文档处理流程

```
用户上传文档
     │
     ▼
┌──────────────┐
│ 文件类型检测  │ ←─ 根据文件扩展名判断
└──────────────┘
     │
     ▼
┌──────────────┐
│  文档内容解析  │ ←─ PDF/DOCX/MD/TXT 提取文本
└──────────────┘
     │
     ▼
┌──────────────┐
│   文本切分    │ ←─ 按段落、按句子切分
└──────────────┘
     │
     ▼
┌──────────────┐
│   向量化      │ ←─ sentence-transformers
└──────────────┘
     │
     ▼
┌──────────────┐
│  Milvus 存储  │ ←─ 批量插入向量数据
└──────────────┘
     │
     ▼
┌──────────────┐
│  元数据存储   │ ←─ MySQL 存储文档信息
└──────────────┘
```

## 7. 项目亮点

### 7.1 技术亮点

1. **RAG 架构完整实现**
   - 文档解析 → 文本切分 → 向量化 → 检索 → 生成
   - 全流程自动化处理

2. **向量数据库选型**
   - 使用 Milvus 企业级向量数据库
   - 支持 ANN 索引（HNSW、IVF 等）
   - 支持分布式部署

3. **缓存优化**
   - Redis 多级缓存策略
   - 基于 MD5 的问答缓存
   - 缓存命中率统计

4. **异步架构**
   - FastAPI 异步框架
   - 异步文件处理
   - SSE 流式响应

5. **MMR 多样性检索**
   - 最大边际相关性重排序
   - 平衡相关性与多样性
   - 减少检索结果冗余

### 7.2 工程亮点

1. **单例模式全局管理**
   - Embedding 模型单例加载
   - LLM 客户端单例复用
   - Redis 缓存连接池

2. **生命周期管理**
   - 应用启动初始化
   - 关闭时资源清理
   - 配置热加载

3. **健康检查机制**
   - MySQL 连接检测
   - Redis 可用性检测
   - Milvus 状态监控

4. **结构化日志**
   - loguru 日志系统
   - 问答日志追踪
   - 文档操作审计

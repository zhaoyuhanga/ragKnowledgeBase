### Q9: 为什么需要 Redis 缓存？缓存策略是什么？

**参考答案：**

**缓存必要性：**

| 场景 | 无缓存 | 有缓存 |
|------|--------|--------|
| 重复问题 | 每次调用 LLM（耗时 2-5s） | 毫秒级响应 |
| LLM 调用成本 | 100% | 按命中率节省 |
| 系统负载 | 高 | 低 |

**缓存策略：**

```python
# 1. 缓存键设计：基于问题内容 Hash
cache_key = f"qa:{hashlib.md5(question.encode()).hexdigest()}"

# 2. 缓存有效期：1 小时
TTL = 3600

# 3. 缓存命中处理
cached = redis_cache.get(cache_key)
if cached:
    logger.info(f"缓存命中: {question[:50]}...")
    return json.loads(cached)

# 4. 缓存未命中处理
result = await llm.generate_with_context(...)
redis_cache.set(cache_key, json.dumps(result), ttl=TTL)
```

### Q10: 系统的性能瓶颈在哪里？如何优化？

**参考答案：**

**性能瓶颈分析：**

| 瓶颈 | 影响 | 优化方案 |
|------|------|----------|
| **LLM 调用** | 延迟 2-5s | 缓存、流式输出 |
| **向量检索** | 延迟 100-500ms | 索引优化、预加载 |
| **文档解析** | 大文件耗时 | 异步处理、限流 |
| **数据库查询** | 延迟 10-50ms | 索引、连接池 |

**优化措施：**

```python
# 1. LLM 优化：流式输出
async def ask_stream(question: str):
    async def generate():
        # SSE 流式返回
        for token in llm.generate_stream(question, contexts):
            yield f"data: {token}\n\n"
    return StreamingResponse(generate(), media_type="text/event-stream")

# 2. 向量检索优化：HNSW 参数调优
search_params = {
    "metric_type": "IP",
    "params": {"ef": 64}  # 调高提高精度，降低提高速度
}

# 3. 连接池优化
engine = create_engine(
    DATABASE_URL,
    pool_size=10,
    max_overflow=20,
    pool_pre_ping=True
)
```

### Q15: 为什么需要运行时配置管理？

**参考答案：**

**运行时配置的作用：**

```python
# rag-qa-system/app/services/runtime_config.py

class RuntimeConfigManager:
    """运行时配置管理器"""
    
    def __init__(self, db: Session):
        self.db = db
        self._config = {}
        self._load_from_db()
    
    def _load_from_db(self):
        """从数据库加载配置"""
        configs = self.db.query(SystemConfig).all()
        self._config = {c.key: c.value for c in configs}
    
    @property
    def retrieval_top_k(self) -> int:
        """检索数量"""
        return int(self._config.get("retrieval_top_k", 5))
    
    @property
    def similarity_threshold(self) -> float:
        """相似度阈值"""
        return float(self._config.get("similarity_threshold", 0.5))

# 优势：
# 1. 无需重启服务即可修改配置
# 2. 可通过管理界面调整参数
# 3. 配置变更可审计
```

### Q20: 如何实现请求限流？

**参考答案：**

**限流实现：**

```python
from fastapi import Request
import time

# 基于 Redis 的滑动窗口限流
async def rate_limit(request: Request, max_requests: int = 100, window: int = 60):
    """请求限流"""
    client_ip = request.client.host
    key = f"rate:{client_ip}:{int(time.time() / window)}"
    
    count = await redis_client.incr(key)
    if count == 1:
        await redis_client.expire(key, window)
    
    if count > max_requests:
        raise HTTPException(
            status_code=429,
            detail="请求过于频繁，请稍后再试"
        )

# 使用限流中间件
@app.middleware("http")
async def rate_limit_middleware(request: Request, call_next):
    if request.url.path.startswith("/api/"):
        await rate_limit(request)
    return await call_next(request)
```

```python
def mmr_rerank(self, results, fetch_k=100, lambda_mult=0.3):
    """MMR 多样性重排序"""
    selected = []
    candidates = results[:fetch_k]
    
    while len(selected) < len(results):
        mmr_score = -float('inf')
        mmr_item = None
        
        for candidate in candidates:
            relevance = candidate.distance
            
            # 计算与已选文档的最大相似度
            diversity = max(
                cosine_sim(candidate.embedding, s.embedding)
                for s in selected
            ) if selected else 0
            
            # MMR 分数
            score = relevance - lambda_mult * diversity
            
            if score > mmr_score:
                mmr_score = score
                mmr_item = candidate
        
        if mmr_item:
            selected.append(mmr_item)
            candidates.remove(mmr_item)
```

## 1. 项目概述

### 1.1 项目简介

RAG（Retrieval-Augmented Generation）问答系统是一个基于检索增强生成技术的智能问答平台。系统支持文档上传、智能切分、向量检索和大语言模型生成，为用户提供精准的问答服务。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 文档管理 | 支持 PDF、Markdown、TXT、DOCX 格式文档上传和解析 |
| 知识库构建 | 智能文本切分、向量化存储到 Milvus 向量数据库 |
| 智能问答 | 基于向量检索和 LLM 生成准确、专业的回答 |
| 缓存管理 | Redis 缓存热点问答，降低 API 调用成本 |
| AI 扩展 | 本地检索为空时，调用 LLM 生成内容并回存向量库 |

### 3.2 数据流架构

```
用户提问
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 1. 缓存检查 (Redis)                                          │
│    - 检查是否有相同问题的缓存                                  │
│    - 有 → 直接返回缓存结果                                     │
└──────────────────────────────────────────────────────────────┘
    │ 无缓存
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 2. 向量检索 (Milvus)                                         │
│    - 用户问题 → Embedding → 向量                              │
│    - ANN 检索找到 Top-K 相关文档                              │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 3. 重排序 (Reranker - 可选)                                  │
│    - Cross-Encoder 对候选结果重新打分                          │
│    - 选择 Top-K 最终结果                                      │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 4. LLM 生成 (DeepSeek)                                      │
│    - 将检索结果作为上下文                                      │
│    - 调用 LLM 生成回答                                        │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 5. 结果处理                                                  │
│    - 保存到 Redis 缓存                                       │
│    - 保存到数据库历史记录                                      │
│    - 返回给用户                                               │
└──────────────────────────────────────────────────────────────┘
```

    │

└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 3. 重排序 (可选)                                             │
│    - Cross-Encoder 重新打分                                   │
│    - 选择 Top-K 最终结果                                      │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 4. LLM 生成                                                  │
│    - 构造 Prompt（系统提示 + 上下文 + 问题）                    │

    │
    ▼

│    - 保存生成内容到向量库                                      │

    ▼
返回结果 + 来源信息

## 5. 核心模块说明

### 5.1 SemanticChunker (语义切分器)

**功能**：将长文档智能切分为适合检索的小块

**切分策略**：
1. **结构优先**：按 Markdown 标题、段落、列表、表格、代码块切分
2. **Token 约束**：每个 Chunk 目标 600 tokens，最大 900 tokens
3. **Overlap**：相邻 Chunk 之间保留 100 tokens 重叠
4. **Parent-Child**：保留章节层级，支持层级检索

**关键类**：
- `ChunkMetadata`: Chunk 元数据
- `Chunk`: 切分结果
- `Section`: 文档章节

### 5.2 EmbeddingService (向量化服务)

**功能**：将文本转换为向量表示

**配置**：
- Provider: Ollama
- Model: Qwen3-Embedding (2560 维)
- 归一化: L2 归一化

**接口**：
- `encode(texts)`: 批量编码
- `encode_single(text)`: 单条编码
- `check_health()`: 健康检查

### 5.3 VectorStore (向量存储)

**功能**：Milvus 向量数据库封装

**特性**：
- 单例模式
- 自动创建 Collection
- 支持 MMR（最大边际相关）重排序
- 支持按 source_type 过滤

**接口**：
- `add_vectors()`: 添加向量
- `search_vectors()`: 向量检索
- `delete_vectors()`: 删除向量

### 5.4 RerankerService (重排序服务)

**功能**：使用 Cross-Encoder 对检索结果重新排序

**配置**：
- Model: Qwen3-Reranker
- recall_k: 召回数量 (默认 50)
- top_k: 最终返回数量 (默认 10)

**接口**：
- `rerank(query, candidates)`: 重排序

### 5.5 QAService (问答服务)

**功能**：核心问答业务逻辑

**流程**：
1. 缓存检查
2. 向量检索
3. Rerank 重排序
4. LLM 生成
5. 结果缓存

### 5.6 Cache (缓存服务)

**功能**：Redis 缓存封装

**特性**：
- QA 缓存: TTL 可配置
- 缓存键前缀
- 序列化: JSON
